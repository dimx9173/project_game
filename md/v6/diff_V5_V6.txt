# V5 vs V6 功能與規格差異比較

## 1. 系統功能與架構
**功能精簡，資料庫明確化**
經過重新評估 MVP 開發階段需求，V6 版本在業務功能上**退回至 V5 的基礎**，但**調整了資料庫規格**。
*   移除原先 V6 初期提議的實體機台「硬體指紋綁定認證」(Device Fingerprinting) 等複雜的資安機制。
*   **集中式資料庫**：由 V5 的 MySQL 升級改用 **PostgreSQL**。
*   **單機本地資料庫**：由 V5 的 SQLite / MySQL 雙軌並行，明確限縮並統一代管為 **SQLite**，確保單機端具備最穩定的輕量化運行環境。
*   保留所有既有 V5 的雙層後台架構、連線邏輯及非功能需求。

## 2. 文件與工程標準化
**Markdown 語法格式標準化**
V6 版本全面導入底層 Markdown 語法格式標準化重構 (基於 `.markdownlint.json` 與 Prettier 規範)，確保技術規格書、架構文件與 UI 說明文件在呈現上具備高度的工程規範與一致性，以利後續的 CI/CD 流程或跨平台排版處理。
