# V5 vs V6 功能與規格差異比較

## 1. 系統功能與安全防護差異
**新增：機台硬體特徵綁定認證 (Device Fingerprinting)**
V6 版本在安全性架構上首次導入了針對實體機台的硬體指紋綁定認證機制。
*   新機台初次連線時，本地程式會讀取底層硬體資訊（主機板序號、CPU ID 等）產生獨一無二的「硬體指紋」，並交由總台管理員人工審核綁定。
*   後續所有的 API 請求皆需攜帶 Token 並與此硬體指紋進行交叉驗證，防止偽造身分。
*   遇到異常或遭竊時，總台支援「一鍵註銷」立即撤銷該機台的連線權限。

*(除了上述資安機制的追加外，V6 版其餘文件內容重點為底層 Markdown 語法格式的標準化重構，舊有功能流程皆與 V5 一致。)*

## 2. 規格與技術選型演進
在架構的資料庫選型上，V6 版進行了更為明確的技術收斂與現代化升級：
*   **集中式資料庫**：由 V5 的 `MySQL` 全面升級改用 **`PostgreSQL`**。
*   **單機本地資料庫**：由 V5 的 `SQLite / MySQL` 雙軌並行，明確限縮並統一代管為 **`SQLite`**，以確保單機端具備最穩定的輕量化運行環境。
