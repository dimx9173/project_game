# V5 vs V6 功能與規格差異比較

## 1. 系統功能與架構
**功能完全一致**
經過重新評估 MVP 開發階段需求，V6 版本在功能與架構上**完全退回至 V5 的基礎**。
*   移除原先 V6 提議的實體機台「硬體指紋綁定認證」(Device Fingerprinting) 等複雜的資安機制。
*   保留 V5 的 MySQL 集中式資料庫與 SQLite/MySQL 單機雙軌並行等設計，不再強制要求升級為 PostgreSQL。
*   保留所有既有 V5 的雙層後台架構、連線邏輯及非功能需求。

## 2. 文件與工程標準化
**Markdown 語法格式標準化**
V6 版本唯一的變更為針對底層 Markdown 語法格式引入標準化重構 (基於 `.markdownlint.json` 與 Prettier 規範)，確保技術規格書、架構文件與 UI 說明文件在呈現上具備高度的工程規範與一致性，以利後續的 CI/CD 流程或跨平台排版處理。
