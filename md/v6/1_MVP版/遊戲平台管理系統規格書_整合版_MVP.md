## 1. 文件概述

本文件詳細闡述遊戲平台管理系統的完整軟體需求，包含集中式後台與單機本地後台兩個層次的功能需求、非功能需求、使用案例、資料模型等內容。

### 1.1 文件目的

本文件旨在為開發、測試、部署人員提供清晰的軟體需求指導，確保系統開發符合業務需求。

---

## 2. 系統概述

### 2.1 系統架構

遊戲平台採用雙層後台架構。

- **集中式後台**：負責全局資料監控、遠端機台管理、全平台交易統計、玩家集中管理等功能。
- **單機本地後台**：負責本地機台管理、本地遊戲配置、本地交易結算、本地統計分析等功能。
  兩個後台相互獨立，但在連線時可進行資料同步與對帳。

### 2.2 系統範圍

本系統涵蓋遊戲機台的集中式管理與本地管理兩個層次，支援遊戲、機台、玩家、交易、串接、多人遊戲、使用者權限、版本更新、介面設定等完整功能模組。

---

## 3. 集中式後台功能需求

### 3.1 儀表板模組

實時顯示全平台的關鍵 KPI（總營收、總下注額、總派彩額、淨收益、線上玩家數、線上機台數）。提供簡單的數據清單與狀態列表。

### 3.2 機台管理模組

集中式管理所有遊戲機台。支援機台列表檢視（含篩選搜尋）、機台連線狀態監控、遠端操作（開分、洗分、重啟）、以及基本的機台群組分類。

### 3.3 遊戲管理模組

本地遊戲庫管理。支援遊戲列表檢視、遊戲新增/編輯/刪除、遊戲啟用/停用。

### 3.4 玩家管理模組

集中式玩家資料管理。支援玩家列表檢視、玩家狀態管理（啟用/停權）、基本錢包餘額檢視。

### 3.5 錢包交易模組

全平台交易紀錄管理。支援交易列表檢視、簡單的日/週/月統計、歷史流水表導出。

### 3.6 使用者權限模組

後台雙層帳號與角色管理（超級管理員與一般操作員）。支援帳號新增/編輯/刪除、操作日誌基礎紀錄。

### 3.7 介面設定模組

玩家端遊戲大廳基礎配置推播。支援大廳背景設定、遊戲分類排序。

---

## 4. 單機本地後台功能需求

### 4.1 儀表板模組

顯示本機的關鍵 KPI（今日營收、今日下注額、今日派彩額、淨收益、線上玩家數）。提供機台基本資訊、硬體狀態、網路狀態、快速操作按鈕等資訊。

### 4.2 機台管理模組

本地機台管理。支援機台資訊檢視、硬體狀態監控、開分、洗分、重啟、關機、進入測試模式、機台設定等功能。

### 4.3 遊戲管理模組

本地遊戲庫管理。支援遊戲列表檢視、遊戲啟用/停用、遊戲配置、遊戲測試等功能。

### 4.4 交易管理模組

本地交易紀錄管理。支援交易列表檢視（多維度篩選）、交易詳情檢視、交易統計（日/週/月）、交易操作（回滾/確認/導出）等功能。

### 4.5 玩家管理模組

本地玩家資料管理。支援玩家列表檢視、玩家詳情檢視、玩家操作（啟用/停權/重設密碼/調整餘額）、玩家統計等功能。

### 4.6 統計報表模組

本地統計資料收集與呈現。支援日/週/月報表、遊戲排行、玩家排行等多維度統計分析。

### 4.7 系統設定模組

單機本地後台配置管理。支援基本設定、與中央連線的網路設定、安全加密與金鑰設定。

### 4.8 同步對帳模組

本地資料與後台資料的同步上傳。遇到斷線時會以列隊 (Queue) 在本地存檔，並在網路恢復後自動依序上傳。

## 5. 非功能需求

### 5.1 效能需求

- **集中式後台**：支援 1000+ 台機台的集中管理，API 響應時間 < 200ms，資料庫查詢時間 < 500ms。
- **單機本地後台**：本地操作響應時間 < 100ms，資料庫查詢時間 < 200ms。

### 5.2 可用性需求

- **集中式後台**：系統可用性 ≥ 99.5%，支援高可用部署。
- **單機本地後台**：離線運作時仍可提供完整功能，連線恢復時自動同步資料。

### 5.3 安全性需求

敏感資料加密儲存（AES-256），交易簽名驗證（RSA），基本的雙層存取控制（RBAC），支援 HTTPS 加密通訊與本地單機鎖定。

### 5.4 可擴展性需求

系統架構支援水平擴展，資料庫支援分片，API 支援版本管理，支援第三方遊戲商接入。

### 5.5 相容性需求

- **集中式後台**：支援 Chrome、Firefox、Safari、Edge 等主流瀏覽器。
- **單機本地後台**：支援 Windows、Linux、macOS 等作業系統。

## 6. 使用案例

### 6.1 機台管理員的使用案例

機台管理員登入單機本地後台，檢視機台狀態，進行開分、洗分、重啟等操作。查詢本機的交易紀錄與統計報表。配置遊戲大廳與系統參數。在連線時自動同步資料到集中式後台。

### 6.2 平台管理員的使用案例

平台管理員登入集中式後台，觀看全台營收總表。新增本地遊戲並推播給特定群組的機台。審核機台傳送上來的離線斷點交易紀錄。

### 6.3 財務人員的使用案例

財務人員登入集中式後台，查詢全平台的交易紀錄。生成日結或週結財務報表並導出 Excel 進行人工對帳。

### 6.4 遊戲商的使用案例

遊戲商通過 API 接入平台，提供遊戲與交易資料。平台通過串接遊戲管理模組進行 API 連線監控與交易對帳。

## 8. 工時評估

### 8.1 開發時程

**開發時程：MVP（約 48 工作天/Man-Days，2.5 個月）**  
**預算規模：44.2 萬**

### 8.2 功能重點

本方案定位為**「快速落地、安全可靠的第一線商用版本」**。相比於傳統粗糙的 POC，本方案去除了華而不實的進階圖表與複雜權限，但**堅守資料不遺失、設定可視化**的底線。

### 8.3 詳細功能規格與系統升級點

#### 8.3.1 核心資料與斷線保單機制（預估工時：12 MD）

- **資料不遺失承諾**：
  - 機台端 (SQLite) 開/洗分後，會將跨網路上傳的任務存入本地的「背景佇列 (Background Queue)」。
  - **斷線重試**：若網路斷線，前台操作完全不受阻礙。背景 Queue 會持續每 3 分鐘自動重試，直到中央伺服器 (PostgreSQL) 成功回應為止。
- **內建第三方遊戲串接（5 家內）**：
  - 除了本地自製遊戲引擎外，亦支援至多 5 家外部遊戲供應商的 API 對接。透過標準化的 PaymentRouter 抽象層，提供彈性的串接能力。

#### 8.3.2 集中式後台（預估工時：20 MD）

- **雙層實體權限 (RBAC)**
  - 提供「超級管理員 (Admin)」與「一般操作員 (Operator)」兩層登入。
  - 操作員只能查看報表與基礎操作，無法匯出名單或更改機台全域設定。
- **正規化數據列表 (DataGrids)**
  - 不提供複雜圖表 (無 Pie Chart / Bar Chart)，但提供具備「分頁、日期區間過濾、欄位排序」的現代化 DataGrid 列表。
  - 包含：系統交易總表、玩家(Guest)餘額速查表、機台參數列表。
- **手動機台狀態**
  - 沒有 1 分鐘心跳監控燈號。機台清單會顯示「最後一次同步帳單的時間」，供管理員人工判斷該機台是否活躍。

#### 8.3.3 單機本地後台（預估工時：18 MD）

- **視覺化本機設定介面 (Local Settings UI)**
  - 完全抹除修改 `.env` 的需求。機台管理者能在本機開啟專屬設定網頁，透過 UI 介面設定：「中央 API IP 位置、機台綁定序號」。
- **現場安全開洗分通道**
  - 提供專屬的輸入介面，並以 PIN 碼保護，防止玩家誤觸。開洗分完成後會顯示「已排入同步佇列」。

### 8.4 工時與資源分配

| 項目分類 | 內容重點 | 預估工時 (Man-Days) | 佔比 |
| :--- | :--- | :--- | :--- |
| **系統基建與單機 Queue** | PostgreSQL + SQLite + 斷線重試佇列 | 9 MD | 19% |
| **集中式後台 (前後端)** | 兩層實體驗證、現代化過濾大表單、無圖表 | 13 MD | 27% |
| **單機本地後/前端** | 視覺化設定檔介面、現場 PIN 碼打分功能 | 8 MD | 17% |
| **遊戲串接與測試** | 5 家 API 串接與前後端單元與 QA 測試 | 18 MD | 37% |
| **總計** | **預計由 1 名資深全端工程師獨立開發** | **48 MD** | **100%** |

## 9. 附錄

### 8.1 術語表

| 術語             | 定義                                              |
| ---------------- | ------------------------------------------------- |
| **集中式後台**   | 遊戲平台的中央管理系統                            |
