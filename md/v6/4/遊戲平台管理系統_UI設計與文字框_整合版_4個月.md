## 1. 系統架構詳細規劃 (企業旗艦版)

### 1.1 具備容錯、對帳與 OTA 派發的雙向流動
* **OTA (Over-The-Air) 靜態檔案派發架構**：
  * 中央透過 Nginx 或 S3 提供 `.zip` 的靜態檔案高速下載路徑 (依據頻寬可能需引入 Cloudflare CDN caching)。
  * 單機端 Node.js 使用 `axios` 或 `node-fetch` stream 下載檔案，存入 `/temp` 資料夾，下載完畢驗證 MD5，利用 `child_process.exec` 開啟獨立的更新腳本，並 `process.exit()` 自我中止以便覆蓋原始檔。

### 1.2 高規非同步佇列與對帳 (Message Queue Extension)
* 單機端的 `TaskQueue` (SQLite) 與 中央的 `sync_logs` 雙向同步。
* 若中央發生嚴重資料庫塞車（超過 10 秒無法消化單機倒過來的 500 筆資料），單機端會觸發「Exponential Backoff (指數退避)」機制，以 `1分鐘 -> 3分鐘 -> 10分鐘 -> 30分鐘` 的間隙逐漸拉開重試，保護中央伺服器不被 DDoS。

### 1.3 泛用型的外部串接橋樑 (Payment Router)
* 比起 標準進階版「寫死一家遊戲商」，本版本在程式碼結構上導入了 `Strategy Pattern (策略模式)` 撰寫 `PaymentRouter`。
* 未來第二期想要接新的一家捕魚機廠商，前後端不需要動核心轉帳架構，只要由工程師實作一個新的 `class ProviderB implements VendorWallet` 即可上線。
