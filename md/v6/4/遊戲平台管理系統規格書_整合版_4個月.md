## 1. 文件概述

本文件詳細闡述遊戲平台管理系統的完整軟體需求，包含集中式後台與單機本地後台兩個層次的功能需求、非功能需求、使用案例、資料模型等內容。

### 1.1 文件目的

本文件旨在為開發、測試、部署人員提供清晰的軟體需求指導，確保系統開發符合業務需求。

---

## 2. 系統概述

### 2.1 系統架構

遊戲平台採用雙層後台架構。

- **集中式後台**：負責全局資料監控、遠端機台管理、全平台交易統計、玩家集中管理等功能。
- **單機本地後台**：負責本地機台管理、本地遊戲配置、本地交易結算、本地統計分析等功能。
  兩個後台相互獨立，但在連線時可進行資料同步與對帳。

### 2.2 系統範圍

本系統涵蓋遊戲機台的集中式管理與本地管理兩個層次，支援遊戲、機台、玩家、交易、串接、多人遊戲、使用者權限、版本更新、介面設定等完整功能模組。

---

## 3. 集中式後台功能需求

### 3.1 儀表板模組

實時顯示全平台的關鍵 KPI，提供高度客製化的 Widget 儀表板面板，可自訂圓餅圖、長條圖與熱門遊戲排行分佈。

### 3.2 深度機台管理與圖表監控

遠端機台管理。支援機台以 1 分鐘為單位的即時心跳、CPU 與記憶體佔用的資源監測。包含遠端指令發送、極嚴格的分類與分區檢視。

### 3.3 遊戲庫與版本管理模組

全局遊戲庫管理。支援遊戲上下架、分類推薦標籤、控制第三方遊戲的供應商開關，以及全台軟體版本版號維護。

### 3.4 玩家屬性管理模組

集中式玩家資料庫。支援玩家等級與 VIP 標籤管理、中央級距的玩家錢包活動檢測、高頻率打水防護機制設定。

### 3.5 錢包交易與多維度報表模組

全平台營收與抽水報表。具備深度的多維交叉篩選器（Filter），可拉出日/週/月/年，甚至依據代理商與機台的立體營運曲線與交叉比對圖表。

### 3.6 高度抽象 PaymentRouter 模組

負責對接外部無上限數量的電子遊戲供應商。透過解耦的 PaymentRouter 架構，能將第三方錢包操作標準化，單機端無縫呼叫、核銷與跨站點轉水。

### 3.7 靜態與動態 RBAC 模組

最高權限之動態角色自定義引擎。業主可手動勾選高達上百個細部 API 權限，指派給特定的「審計員」、「查帳會計」或「維修外包」。

### 3.8 全球 OTA 無感熱更新發布中心

包含腳本上傳驗證、MD5 雜湊審查、排程更新（預計深夜）、與失敗自動還原機制。玩家端不受干擾的背景靜默熱更新。

### 3.9 爭議對帳處理專區

專為處理網路斷線、封包遺失、人工現場補發獎勵的核銷特區。提供雙面對帳與會計差異除錯，嚴格控管每一筆人工干預的資金流向。

### 3.10 前端版面廣播模組

玩家端前台版面配置中心。支援遠端推播大廳排版與活動橫幅 (Banner)。

## 4. 單機本地後台功能需求

### 4.1 儀表板模組

顯示本機的關鍵 KPI（今日營收、今日下注額、今日派彩額、淨收益、線上玩家數）。提供機台基本資訊、硬體狀態、網路狀態、快速操作按鈕等資訊。

### 4.2 機台管理模組

本地機台管理。支援機台資訊檢視、硬體狀態監控、開分、洗分、重啟、關機、進入測試模式、機台設定等功能。

### 4.3 遊戲管理模組

本地遊戲庫管理。支援遊戲列表檢視、遊戲啟用/停用、遊戲配置、遊戲測試等功能。

### 4.4 交易管理模組

本地交易紀錄管理。支援交易列表檢視（多維度篩選）、交易詳情檢視、交易統計（日/週/月）、交易操作（回滾/確認/導出）等功能。

### 4.5 玩家管理模組

本地玩家資料管理。支援玩家列表檢視、玩家詳情檢視、玩家操作（啟用/停權/重設密碼/調整餘額）、玩家統計等功能。

### 4.6 統計報表模組

本地統計資料收集與呈現。支援日/週/月報表、遊戲排行、玩家排行等多維度統計分析。

### 4.7 底層通訊與金鑰加密模組

單機本地網路防綁架設定。支援中央通訊的動態憑證核發、雙面驗證 (Mutual TLS)、與硬體特徵綁定的安全檢測。

### 4.8 雙向佇列與交易補償機制

支援指數退避演算法 (Exponential Backoff) 與補償機制。確保斷線期間即使產生異常中斷，系統亦能以本地的 SQL 交易紀錄為主要權威進行修補與強制覆寫。

### 4.9 監控與日誌模組

系統狀態監控與日誌管理。支援硬體監控、軟體監控、日誌查詢等功能。

## 5. 非功能需求

### 5.1 效能需求

- **集中式後台**：支援 1000+ 台機台的集中管理，API 響應時間 < 200ms，資料庫查詢時間 < 500ms。
- **單機本地後台**：本地操作響應時間 < 100ms，資料庫查詢時間 < 200ms。

### 5.2 可用性需求

- **集中式後台**：系統可用性 ≥ 99.5%，支援高可用部署。
- **單機本地後台**：離線運作時仍可提供完整功能，連線恢復時自動同步資料。

### 5.3 企業級防護與稽核需求

實施極端嚴格的資安標準（AES-256 + 鹽值）、交易簽名驗證（HMAC/RSA）。每個單一調用包含完整的 HTTP Request, IP, 裝置特徵等記錄做為稽核依據。

### 5.4 可服務性與自動擴充架構

系統架構支援水平擴展，資料庫支援分片，API 支援版本管理，支援第三方遊戲商接入。

### 5.5 相容性需求

- **集中式後台**：支援 Chrome、Firefox、Safari、Edge 等主流瀏覽器。
- **單機本地後台**：支援 Windows、Linux、macOS 等作業系統。

## 6. 使用案例

### 6.1 高階硬體的機台監控

駐點工程師登入單機後台，看見 CPU 因為更新下載突然飆漲到了 90%，但他知道這是背景 OTA 的正常現象，並安心地讓系統執行熱更新而不需要讓玩家起身離開。

### 6.2 動態會計審核的情境

財務人員登入集中式後台的 `[爭議對帳處理專區]`，系統自動標記了一筆因昨天雷擊重啟導致「中央與單機不對等」的 1500 分交易。會計人員按下「承認單機數據」後，系統補上人工核銷戳記，結束爭議。

### 6.3 超級擴充第三方的情境

工程師接到指令掛載第 4 間國外的捕魚機公司。因此他不用修改 V6 主架構與核心流水帳邏輯，只要運用 PaymentRouter 的 Standard Interface 實作，即刻就讓所有台灣的機台都能跑國外的 API 遊戲。

## 7. 資料字典

### 7.1 核心資料模型

**機台（Machine）**

| 欄位名稱    | 資料類型 | 說明                                   |
| ----------- | -------- | -------------------------------------- |
| `id`        | String   | 機台編號（主鍵）                       |
| `name`      | String   | 機台名稱                               |
| `ip`        | String   | 機台 IP 位址                           |
| `mac`       | String   | 機台 MAC 位址                          |
| `status`    | Enum     | 機台狀態（online/offline/maintenance） |
| `mode`      | Enum     | 運作模式（network/standalone）         |
| `version`   | String   | 軟體版本                               |
| `createdAt` | DateTime | 建立時間                               |

**遊戲（Game）**

| 欄位名稱       | 資料類型 | 說明                        |
| -------------- | -------- | --------------------------- |
| `id`           | String   | 遊戲編號（主鍵）            |
| `name`         | String   | 遊戲名稱                    |
| `provider`     | String   | 遊戲供應商                  |
| `iconUrl`      | String   | 遊戲圖示 URL                |
| `status`       | Enum     | 遊戲狀態（active/inactive） |
| `displayOrder` | Integer  | 顯示順序                    |
| `createdAt`    | DateTime | 建立時間                    |

**玩家（Player）**

| 欄位名稱      | 資料類型 | 說明                                |
| ------------- | -------- | ----------------------------------- |
| `id`          | String   | 玩家編號（主鍵）                    |
| `username`    | String   | 玩家帳號                            |
| `nickname`    | String   | 玩家暱稱                            |
| `balance`     | Decimal  | 錢包餘額                            |
| `status`      | Enum     | 玩家狀態（active/suspended/frozen） |
| `totalBet`    | Decimal  | 累計下注額                          |
| `totalPayout` | Decimal  | 累計派彩額                          |
| `createdAt`   | DateTime | 註冊時間                            |

**交易（Transaction）**

| 欄位名稱        | 資料類型 | 說明                                    |
| --------------- | -------- | --------------------------------------- |
| `id`            | String   | 交易編號（主鍵）                        |
| `playerId`      | String   | 玩家編號（外鍵）                        |
| `gameId`        | String   | 遊戲編號（外鍵）                        |
| `type`          | Enum     | 交易類型（bet/payout/deposit/withdraw） |
| `amount`        | Decimal  | 交易金額                                |
| `balanceBefore` | Decimal  | 交易前餘額                              |
| `balanceAfter`  | Decimal  | 交易後餘額                              |
| `status`        | Enum     | 交易狀態（success/failed/pending）      |
| `createdAt`     | DateTime | 交易時間                                |

## 8. 附錄

### 8.1 術語表

| 術語             | 定義                                              |
| ---------------- | ------------------------------------------------- |
| **集中式後台**   | 遊戲平台的中央管理系統                            |

| **RBAC**         | 基於角色的存取控制（Role-Based Access Control）   |

| **WebSocket**    | 實時雙向通訊協議                                  |
