## 1. 文件概述

本文件詳細闡述遊戲平台管理系統的完整軟體需求，包含集中式後台與單機本地後台兩個層次的功能需求、非功能需求、使用案例、資料模型等內容。

### 1.1 文件目的

本文件旨在為開發、測試、部署人員提供清晰的軟體需求指導，確保系統開發符合業務需求。

---

## 2. 系統概述

### 2.1 系統架構

遊戲平台採用雙層後台架構。

- **集中式後台**：負責全局資料監控、遠端機台管理、全平台交易統計、玩家集中管理等功能。
- **單機本地後台**：負責本地機台管理、本地遊戲配置、本地交易結算、本地統計分析等功能。
  兩個後台相互獨立，但在連線時可進行資料同步與對帳。

### 2.2 系統範圍

本系統涵蓋遊戲機台的集中式管理與本地管理兩個層次，支援遊戲、機台、玩家、交易、串接、多人遊戲、使用者權限、版本更新、介面設定等完整功能模組。

## 3. 集中式後台功能需求

### 3.1 儀表板模組

實時顯示全平台的關鍵 KPI（總營收、總下注額、總派彩額、淨收益、線上玩家數、線上機台數）。提供營收趨勢圖、機台狀態圓餅圖、熱門遊戲排行、系統通知等視覺化資訊。

### 3.2 機台管理模組

集中式管理所有遊戲機台。支援機台列表檢視（含篩選搜尋）、機台狀態監控（連線/離線/單機模式）、遠端操作（開分、洗分、重啟）、機台配置下發、機台軟體版本管理、機台分組管理等功能。

### 3.3 遊戲管理模組

全局遊戲庫管理。支援遊戲列表檢視、遊戲新增/編輯/刪除、遊戲啟用/停用、遊戲版本管理、遊戲分類管理、遊戲供應商管理等功能。

### 3.4 玩家管理模組

集中式玩家資料管理。支援玩家列表檢視、玩家詳情檢視、玩家狀態管理（啟用/停權）、玩家錢包管理、玩家交易歷史查詢、批量玩家操作等功能。

### 3.5 錢包交易模組

全平台交易紀錄管理。支援交易列表檢視（多維度篩選）、交易詳情檢視、交易統計（日/週/月/年）、交易對帳、異常交易處理等功能。

### 3.6 串接遊戲管理模組

遊戲商 API 串接管理。支援遊戲商列表檢視、API 連線狀態監控、遊戲同步管理、交易對帳、API 日誌查詢等功能。

### 3.7 多人遊戲模組

多人遊戲（如百家樂）的集中式管理。支援牌桌列表檢視、牌桌狀態監控、牌桌配置管理、牌局歷史查詢等功能。

### 3.8 使用者權限模組

後台帳號與角色管理。支援帳號新增/編輯/刪除、角色權限設定、操作日誌審計等功能。

### 3.9 版本更新模組

應用程式版本與遠端更新管理。支援版本列表管理、更新任務下發、更新進度監控、更新日誌查詢等功能。

### 3.10 介面設定模組

玩家端遊戲大廳配置管理。支援大廳背景設定、遊戲分類排序、推薦遊戲設定、公告管理等功能。

## 4. 單機本地後台功能需求

### 4.1 儀表板模組

顯示本機的關鍵 KPI（今日營收、今日下注額、今日派彩額、淨收益、線上玩家數）。提供機台基本資訊、硬體狀態、網路狀態、快速操作按鈕等資訊。

### 4.2 機台管理模組

本地機台管理。支援機台資訊檢視、硬體狀態監控、開分、洗分、重啟、關機、進入測試模式、機台設定等功能。

### 4.3 遊戲管理模組

本地遊戲庫管理。支援遊戲列表檢視、遊戲啟用/停用、遊戲配置、遊戲測試等功能。

### 4.4 交易管理模組

本地交易紀錄管理。支援交易列表檢視（多維度篩選）、交易詳情檢視、交易統計（日/週/月）、交易操作（回滾/確認/導出）等功能。

### 4.5 玩家管理模組

本地玩家資料管理。支援玩家列表檢視、玩家詳情檢視、玩家操作（啟用/停權/重設密碼/調整餘額）、玩家統計等功能。

### 4.6 統計報表模組

本地統計資料收集與呈現。支援日/週/月報表、遊戲排行、玩家排行等多維度統計分析。

### 4.7 系統設定模組

單機本地後台配置管理。支援基本設定、網路設定、遊戲大廳設定、安全設定、備份與恢復等功能。

### 4.8 同步與對帳模組

本地資料與後台資料的同步與驗證。支援同步狀態檢視、手動同步、對帳檢查、同步日誌等功能。

### 4.9 監控與日誌模組

系統狀態監控與日誌管理。支援硬體監控、軟體監控、日誌查詢等功能。

## 5.6 開發時程建議

### 5.6.1 遊戲串接時程評估

| 遊戲商數量 | 額外開發時程 | 說明 |
|------------|--------------|------|
| **5 家以內** | **+2 週** | 標準報價，含框架建置 |
| 6-10 家 | +3 週 | 每家約 2-3 天差異化適配 |
| 10 家以上 | +4 週以上 | 需評估遊戲商類型差異 |

#### 計算方式
- **第一家**：1-2 週（建立串接框架 + 熟悉流程）
- **第二家起**：每家 3-5 天（差異化適配）
- **框架成本**：固定，與遊戲商數量無關

#### 影響時程的變數
1. 遊戲商 API 文件完整性
2. 遊戲商技術支援响应速度
3. 特殊功能需求（風控、彩池等）
4. 測試環境是否由遊戲商提供

### 5.6.2 各版本時程建議

| 版本 | 基礎系統 | + 遊戲串接（5家內） | 總時程 |

| 標準版 | 3-4 週 | +2 週 | **5-6 週** |

> **備註**：以上為單一開發人員估計，實際時程需依團隊規模調整

### 5.6.3 里程碑建議

| 週數 | 里程碑 |
|------|--------|
| Week 1-2 | 基礎框架搭建 + 第一家遊戲商串接 |
| Week 3 | 第二、三家遊戲商串接 |
| Week 4 | 第四、五家遊戲商串接 + 整體測試 |
| Week 5-6 | UAT + 修正 + 上線 |

### 5.1 效能需求

- **集中式後台**：支援 1000+ 台機台的集中管理，API 響應時間 < 200ms，資料庫查詢時間 < 500ms。
- **單機本地後台**：本地操作響應時間 < 100ms，資料庫查詢時間 < 200ms。

### 5.2 可用性需求

- **集中式後台**：系統可用性 ≥ 99.5%，支援高可用部署。
- **單機本地後台**：離線運作時仍可提供完整功能，連線恢復時自動同步資料。

### 5.3 安全性需求

敏感資料加密儲存（AES-256），交易簽名驗證（RSA），基於角色的存取控制（RBAC），詳細的稽核日誌，支援 HTTPS 加密通訊。

### 5.4 可擴展性需求

系統架構支援水平擴展，資料庫支援分片，API 支援版本管理，支援第三方遊戲商接入。

### 5.5 相容性需求

- **集中式後台**：支援 Chrome、Firefox、Safari、Edge 等主流瀏覽器。
- **單機本地後台**：支援 Windows、Linux、macOS 等作業系統。

## 6. 使用案例

### 6.1 機台管理員的使用案例

機台管理員登入單機本地後台，檢視機台狀態，進行開分、洗分、重啟等操作。查詢本機的交易紀錄與統計報表。配置遊戲大廳與系統參數。在連線時自動同步資料到集中式後台。

### 6.2 平台管理員的使用案例

平台管理員登入集中式後台，監控全平台的營運資料。管理所有遊戲機台、遊戲、玩家。下發遊戲更新、配置更新到各機台。查詢全平台的交易統計與對帳資訊。

### 6.3 財務人員的使用案例

財務人員登入集中式後台，查詢全平台的交易紀錄與對帳資訊。生成日/週/月的財務報表。進行交易對帳與異常交易處理。

### 6.4 遊戲商的使用案例

遊戲商通過 API 接入平台，提供遊戲與交易資料。平台通過串接遊戲管理模組進行 API 連線監控與交易對帳。

| `playerId`      | String   | 玩家編號（外鍵）                        |
| `gameId`        | String   | 遊戲編號（外鍵）                        |
| `type`          | Enum     | 交易類型（bet/payout/deposit/withdraw） |
| `amount`        | Decimal  | 交易金額                                |
| `balanceBefore` | Decimal  | 交易前餘額                              |
| `balanceAfter`  | Decimal  | 交易後餘額                              |
| `status`        | Enum     | 交易狀態（success/failed/pending）      |
| `createdAt`     | DateTime | 交易時間                                |

### 8.1 術語表

| 術語             | 定義                                              |
