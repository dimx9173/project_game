# 遊戲平台管理系統 (MVP Phase 1) - 方案 B：標準進階版 (3個月)
## 系統架構 (SA) 與 UI 文字框設計詳細規劃

---

## 1. 系統架構詳細規劃 (標準進階版)

### 1.1 三層架構的代理商權限系統 (RBAC)
* **資料隔離機制**：
  * 中央資料庫的機台 (`machines`) 與玩家 (`users`) 皆綁定 `agent_id`。
  * 代理商登入後，後端 Middleware 會強制將所有 API 查詢加上 `WHERE agent_id = ?`，確保絕對的資料隔離。
  * 代理商可自行向下開設「現場操作員」帳號。

### 1.2 外部遊戲 API 串接架構
* **基礎錢包中心化 (Wallet)**：
  * 遊戲平台具備「中心錢包」功能。當玩家點擊第三方遊戲 (如：PG老虎機 或 知名百家樂) 時，觸發轉帳流程。
  * **轉帳流程**：預扣本地餘額 ➔ 呼叫第三方 Deposit API ➔ 等待 HTTP 200 回傳 ➔ 確認進入遊戲。若第三方 API Timeout，則標記該筆轉帳狀態為 `investigating`，供管理員人工查看。

### 1.3 視覺化戰情室與機台監控 
* **長條圖與圓餅圖渲染**：
  * 採用開源輕量化圖表庫 (如 Recharts/Chart.js)。將原始的交易流水，透過 PostgreSQL 的 `GROUP BY` 聚合指令，產出「每日營收趨勢圖」。
* **1分鐘心跳監測 (Heartbeat)**：
  * 機台本地的 Node.js 服務每 60 秒發送一個包含 `CPU%、RAM%、當前遊戲狀態` 的封包到中央伺服器 `/sys/heartbeat`。
  * 中央透過 Redis (或 DB) 記錄。若超過 3 分鐘未收到，判定離線。
