## 額外遊戲串接擴充報價（超額擴充）

> 各版本皆已內建 3 家遊戲 API 串接，若後續需擴充更多家數，適用以下報價：

| 數量 | 額外費用 | 時程影響 |
|------|----------|----------|
| 額外 1 家（使用既有框架） | +3 萬 | +4 天（含測試） |
| 額外 5 家批次包 | +12 萬 | +3 週（含測試） |
| 10 家以上 | - | 需另行專案評估遊戲商類型差異 |

### 📌 串接工時計算方式（供後續擴充參考）

- **第一家**：1-2 週（建立串接框架 + 熟悉流程）
- **第二家起**：每家 3-5 天（差異化適配）
- **框架成本**：固定，與遊戲商數量無關
- **⚠️ 影響額外串接時程的變數**：
  1. 遊戲商 API 文件完整性
  2. 遊戲商技術支援回應速度
  3. 特殊功能需求（風控、彩池等）
  4. 測試環境是否由遊戲商提供

## 風險評估與建議

### 風險矩陣

| 風險項目 | 發生機率 | 影響程度 | 緩解措施 |
|---------|---------|---------|---------|
| 第三方 API 規格變動 | 高 | 高 | 先做 Mock Server 隔離依賴 |
| 單人開發請假/生病 | 中 | 高 | 預留 10% 時間 buffer |
| 技術難題卡關 | 中 | 中 | 預留 5% 技術攻堅時間 |
| 需求變更 | 低 | 高 | 採用敏捷迭代，每週 review 範圍 |

### 建議開發節奏

| 週次 | MVP 內容 | 標準版內容 | 旗艦版內容 |
|------|---------|-----------|-----------|
| 1-3 | 系統基建、Queue 機制、第三方 API 串接 | - | - |
| 4-6 | 集中式/單機本地 後端與前端 | - | - |
| 7-10 | 測試、調整、部署 | MVP + 測試 | - |
| 11-12 | - | Dashboard、權限模組、測試部署 | 標準版 |
| 13-15 | - | - | OTA 熱更新、爭議對帳 |
| 16-17 | - | - | 硬體監控、測試部署 |

## 升級路徑與增量工時估算

| 升級路徑 | 增量總人天 (MD) | 增量預算估計 (含 Buffer) | 重點擴充項目說明 |
