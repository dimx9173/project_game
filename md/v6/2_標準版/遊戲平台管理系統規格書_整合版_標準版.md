## 1. 文件概述

本文件詳細闡述遊戲平台管理系統的完整軟體需求，包含集中式後台與單機本地後台兩個層次的功能需求、非功能需求、使用案例、資料模型等內容。

### 1.1 文件目的

本文件旨在為開發、測試、部署人員提供清晰的軟體需求指導，確保系統開發符合業務需求。

---

## 2. 系統概述

### 2.1 系統架構

遊戲平台採用雙層後台架構。

- **集中式後台**：負責全局資料監控、遠端機台管理、全平台交易統計、玩家集中管理等功能。
- **單機本地後台**：負責本地機台管理、本地遊戲配置、本地交易結算、本地統計分析等功能。
  兩個後台相互獨立，但在連線時可進行資料同步與對帳。

### 2.2 系統範圍

本系統涵蓋遊戲機台的集中式管理與本地管理兩個層次，支援遊戲、機台、玩家、交易、串接、多人遊戲、使用者權限、版本更新、介面設定等完整功能模組。

---

## 3. 集中式後台功能需求

### 3.1 儀表板模組

實時顯示全平台的關鍵 KPI，提供高度客製化的 Widget 儀表板面板，可自訂圓餅圖、長條圖與熱門遊戲排行分佈。

### 3.2 機台心跳監控與管理模組

遠端機台管理。支援機台以 1 分鐘為單位的即時心跳監控（紅綠燈號）、遠端發送指令操作（開分、洗分、重啟）、以及機台群組權限隔離管理。

### 3.3 遊戲管理模組

全局遊戲庫管理。支援遊戲上下架、分類推薦標籤、以及控制第三方遊戲的供應商開關。

### 3.4 玩家管理模組

集中式玩家資料庫。支援玩家等級與 VIP 標籤管理、中央級距的玩家錢包與流水帳檢視。

### 3.5 錢包交易模組

全平台營收與抽水報表。具備深度的多維交叉篩選器（Filter），可拉出日/週/月營運曲線。

### 3.6 串接遊戲管理模組

負責對外對接 3 家知名的第三方電子遊戲供應商，並運用標準化的 PaymentRouter 介面，讓單機端也能進行第三方 API 錢包扣款。

### 3.7 多人遊戲模組

多人遊戲（如牌桌遊戲）的集中式與本地端基礎管理，標準版內建支援 1 家多人遊戲。
支援牌桌狀態監控與牌局歷史查詢等核心營運功能。

### 3.8 使用者三層權限模組

後台階層化帳號角色管理。區分「超級管理員」、「多帳號」、「店長/操作員」，各多帳號只能觀測所屬旗下機台的報表與儀表板。

### 3.9 介面設定模組

玩家端前台版面配置中心。支援遠端推播大廳排版與活動橫幅 (Banner)。

## 4. 單機本地後台功能需求

### 4.1 儀表板模組

顯示本機的關鍵 KPI（今日營收、今日下注額、今日派彩額、淨收益、線上玩家數）。提供機台基本資訊、硬體狀態、網路狀態、快速操作按鈕等資訊。

### 4.2 機台管理模組

本地機台管理。支援機台資訊檢視、硬體狀態監控、開分、洗分、重啟、關機、進入測試模式、機台設定等功能。

### 4.3 遊戲管理模組

本地遊戲庫管理。支援遊戲列表檢視、遊戲啟用/停用、遊戲配置、遊戲測試等功能。

### 4.4 交易管理模組

本地交易紀錄管理。支援交易列表檢視（多維度篩選）、交易詳情檢視、交易統計（日/週/月）、交易操作（回滾/確認/導出）等功能。

### 4.5 玩家管理模組

本地玩家資料管理。支援玩家列表檢視、玩家詳情檢視、玩家操作（啟用/停權/重設密碼/調整餘額）、玩家統計等功能。

### 4.6 統計報表模組

本地統計資料收集與呈現。支援日/週/月報表、遊戲排行、玩家排行等多維度統計分析。

### 4.7 系統設定模組

單機本地網路防綁架設定。支援中央通訊的憑證與嚴謹的金鑰參數綁定、確保機台不被惡意遷移站點。

### 4.8 雙向佇列同步模組

支援指數退避演算法 (Exponential Backoff) 的背景重試。確保單機在斷線期間產生的帳目，於恢復網路時不會遺漏、一筆不漏地上傳給中央。

### 4.9 監控與日誌模組

系統狀態監控與日誌管理。支援硬體監控、軟體監控、日誌查詢等功能。

## 5. 非功能需求

### 5.1 效能需求

- **集中式後台**：支援 1000+ 台機台的集中管理，API 響應時間 < 200ms，資料庫查詢時間 < 500ms。
- **單機本地後台**：本地操作響應時間 < 100ms，資料庫查詢時間 < 200ms。

### 5.2 可用性需求

- **集中式後台**：系統可用性 ≥ 99.5%，支援高可用部署。
- **單機本地後台**：離線運作時仍可提供完整功能，連線恢復時自動同步資料。

### 5.3 安全性需求

極高標準的身分認證管控，基於三層角色的存取控制（RBAC），並具備密碼與 PIN 碼雙軌認證、本機加密存檔、防竄改的 Ack 回執確認機制，以及交易簽名驗證（HMAC/RSA）。

### 5.4 可擴展性需求

系統架構支援水平擴展，資料庫支援分片，API 支援版本管理，支援第三方遊戲商接入。

### 5.5 相容性需求

- **集中式後台**：支援 Chrome、Firefox、Safari、Edge 等主流瀏覽器。
- **單機本地後台**：支援 Windows、Linux、macOS 等作業系統。

## 6. 使用案例

### 6.1 機台管理員的使用案例

機台管理員登入單機本地後台，檢視機台狀態，進行開分、洗分、重啟等操作。查詢本機的交易紀錄與統計報表。配置遊戲大廳與系統參數。在連線時自動同步資料到集中式後台。

### 6.2 平台管理員與多帳號的使用案例

平台管理員登入集中式總表，檢視所有營業額與 3 家第三方遊戲的抽成。多帳號登入後，畫面自動限縮至他所屬門市的機台 KPI 與玩家名單。

### 6.3 玩家操作串接遊戲的案例

玩家在單機端點選第三方廠商的老虎機，單機端呼叫 PaymentRouter API 進行轉水扣款，並無縫銜接開啟外部遊戲畫面。

## 7. 資料字典

### 7.1 核心資料模型

**機台（Machine）**

| 欄位名稱    | 資料類型 | 說明                                   |
| ----------- | -------- | -------------------------------------- |
| `id`        | String   | 機台編號（主鍵）                       |
| `name`      | String   | 機台名稱                               |
| `ip`        | String   | 機台 IP 位址                           |
| `mac`       | String   | 機台 MAC 位址                          |
| `status`    | Enum     | 機台狀態（online/offline/maintenance） |
| `mode`      | Enum     | 運作模式（network/standalone）         |
| `version`   | String   | 軟體版本                               |
| `createdAt` | DateTime | 建立時間                               |

**遊戲（Game）**

| 欄位名稱       | 資料類型 | 說明                        |
| -------------- | -------- | --------------------------- |
| `id`           | String   | 遊戲編號（主鍵）            |
| `name`         | String   | 遊戲名稱                    |
| `provider`     | String   | 遊戲供應商                  |
| `iconUrl`      | String   | 遊戲圖示 URL                |
| `status`       | Enum     | 遊戲狀態（active/inactive） |
| `displayOrder` | Integer  | 顯示順序                    |
| `createdAt`    | DateTime | 建立時間                    |

**玩家（Player）**

| 欄位名稱      | 資料類型 | 說明                                |
| ------------- | -------- | ----------------------------------- |
| `id`          | String   | 玩家編號（主鍵）                    |
| `username`    | String   | 玩家帳號                            |
| `nickname`    | String   | 玩家暱稱                            |
| `balance`     | Decimal  | 錢包餘額                            |
| `status`      | Enum     | 玩家狀態（active/suspended/frozen） |
| `totalBet`    | Decimal  | 累計下注額                          |
| `totalPayout` | Decimal  | 累計派彩額                          |
| `createdAt`   | DateTime | 註冊時間                            |

**交易（Transaction）**

| 欄位名稱        | 資料類型 | 說明                                    |
| --------------- | -------- | --------------------------------------- |
| `id`            | String   | 交易編號（主鍵）                        |
| `playerId`      | String   | 玩家編號（外鍵）                        |
| `gameId`        | String   | 遊戲編號（外鍵）                        |
| `type`          | Enum     | 交易類型（bet/payout/deposit/withdraw） |
| `amount`        | Decimal  | 交易金額                                |
| `balanceBefore` | Decimal  | 交易前餘額                              |
| `balanceAfter`  | Decimal  | 交易後餘額                              |
| `status`        | Enum     | 交易狀態（success/failed/pending）      |
| `createdAt`     | DateTime | 交易時間                                |

## 8. 附錄

### 8.1 術語表

| 術語             | 定義                                              |
| ---------------- | ------------------------------------------------- |
| **集中式後台**   | 遊戲平台的中央管理系統                            |
| **單機本地後台** | 每台遊戲機台的本地管理系統                        |
| **開分**         | 啟動遊戲機台的遊戲程式                            |
| **洗分**         | 清除玩家資料，重置機台狀態                        |
| **測試模式**     | 機台進入測試狀態，支援虛擬下注                    |
| **交易結算**     | 計算玩家的下注與派彩，更新玩家餘額                |
| **對帳**         | 驗證本地資料與後台資料的一致性                    |
| **KPI**          | 關鍵績效指標（Key Performance Indicator）         |
| **RBAC**         | 基於角色的存取控制（Role-Based Access Control）   |
| **API**          | 應用程式介面（Application Programming Interface） |
| **WebSocket**    | 實時雙向通訊協議                                  |
