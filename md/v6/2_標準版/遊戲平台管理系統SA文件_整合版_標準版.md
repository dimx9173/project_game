## 1. 文件概述

本文件詳細闡述遊戲平台管理系統的整體系統架構，包含集中式後台與單機本地後台兩個層次的架構設計、資料流、通訊機制等內容。

## 2. 系統架構概述

### 2.1 雙層後台架構

遊戲平台採用雙層後台架構設計。

- **上層（集中式後台）**：負責全局資料監控、遠端機台管理、全平台交易統計、玩家集中管理等功能。
- **下層（單機本地後台）**：負責本地機台管理、本地遊戲配置、本地交易結算、本地統計分析等功能。
  兩個後台相互獨立，但在連線時可進行資料同步與對帳。

### 2.2 系統組件

系統包含以下主要組件：

- 集中式後台前端（React + Tailwind CSS）
- 集中式後台後端（Express.js + Node.js）
- 單機本地後台前端（React + Tailwind CSS）
- 單機本地後台後端（Express.js + Node.js）
- 集中式資料庫（PostgreSQL）
- 單機本地資料庫（SQLite）
- 遊戲引擎
- 玩家端應用程式

---

## 3. 集中式後台架構

### 3.1 前端架構

集中式後台前端採用 **React 19 + Tailwind CSS 4**。使用 Wouter 進行路由管理，shadcn/ui 提供統一的 UI 元件，Recharts 用於資料視覺化，React Context 進行狀態管理。支援響應式設計，適配桌面、平板、手機等多種裝置。

### 3.2 後端架構

集中式後台後端採用 **Express.js** 框架，Node.js 執行環境。提供 RESTful API 介面，支援 WebSocket 實時通訊。使用 PostgreSQL 資料庫進行資料持久化。

### 3.3 核心模組

集中式後台包含以下核心模組：

1. **儀表板模組**：全平台的圖表戰情室 (長條圖、圓餅圖)。
2. **進階機台監控模組**：一分鐘心跳 (Heartbeat) 的即時連線紅綠燈。
3. **遊戲管理模組**：本地遊戲庫管理。
4. **玩家管理模組**：集中式玩家管理與錢包檢視。
5. **錢包交易模組**：具有高度 Filter 的交易流水統計。
6. **串接遊戲管理模組**：(核心新增) 遊戲商 API Wallet 串接轉水。
7. **爭議對帳處理模組**：(核心新增) 人工處理斷線與核銷的專區。
8. **使用者權限模組**：(核心新增) 三層式權限(含代理商)。
9. **介面設定模組**：玩家端大廳配置管理。

## 4. 單機本地後台架構

### 4.1 前端架構

單機本地後台前端採用與集中式後台相同的（**React 19 + Tailwind CSS 4**），確保設計風格與使用體驗的一致性。支援本地化部署，可在機台上直接運行。

### 4.2 後端架構

單機本地後台後端採用 **Express.js** 框架，Node.js 執行環境。提供 RESTful API 介面，支援 WebSocket 實時通訊。使用 SQLite 作為本地資料儲存，支援本地快取。

### 4.3 核心模組

單機本地後台包含以下核心模組：

1. **操作與儀表板模組**：安全登入、現場開洗分與小工具。
2. **深層機台監控模組**：(核心新增) CPU、記憶體佔用實時折線圖。
3. **遊戲管理模組**：本地遊戲啟用與第三方 Payment Router 綁定。
4. **交易管理模組**：本地交易與傳輸狀態 (Pending/Done)。
5. **網路與加密設定模組**：綁定中央通訊參數與金鑰。
6. **自動對帳模組**：(核心新增) 自動分析與中央資料庫是否有差異。

## 5. 資料同步與對帳機制

### 5.1 同步流程

當單機連線到集中式後台時，自動觸發資料同步流程。首先檢查離線期間的本地變更，上傳本地交易紀錄到後台。然後下載後台的遊戲更新、配置更新。最後驗證資料一致性，完成同步。

### 5.2 衝突解決

單機帳單若遭遇網路斷線，將於本地佇列中遞延重試直到上傳成功為止，保證資料不遺失。但不設複雜之版本仲裁，一切以單機傳上來之紀錄為準。

### 5.3 對帳機制

系統定期進行對帳檢查，驗證本地資料與後台資料的一致性。對於不一致的項目，記錄並標記為待審核，由管理員手動處理。

## 6. 安全性架構

### 6.1 認證與授權

系統採用帳號密碼認證，使用 JWT Token 進行會話管理。劃分「超級管理員」、「代理商」與「一般操作員」三層權限。代理商可獨立觀看所屬機台之報表。

### 6.2 資料加密

所有敏感資料（玩家帳號、交易紀錄、配置資訊）使用 **AES-256** 加密演算法進行加密儲存。通訊採用 HTTPS 加密協議，確保資料傳輸安全。

### 6.3 交易簽名驗證

所有交易使用 **RSA** 數位簽名進行驗證，確保交易的真實性與完整性，防止交易被竄改。

### 6.4 稽核日誌

記錄所有重要操作的詳細日誌，包括操作者、操作時間、操作內容、操作結果等資訊，支援追蹤與審計。

## 8. 部署架構

### 8.1 集中式後台部署

集中式後台部署在雲端或資料中心。前端靜態資源部署在 CDN。後端服務使用容器化部署（Docker），支援自動擴展。資料庫使用主從複製實現高可用。

### 8.2 單機本地後台部署

單機本地後台與遊戲引擎部署在同一台機器上。支援 Windows、Linux、macOS 等作業系統。使用 Node.js 執行環境，SQLite 作為本地資料庫。

### 8.3 網路拓撲

- 集中式後台與單機本地後台通過 Internet 連線。
- 單機本地後台與遊戲引擎通過本地網路連線。
- 玩家端應用程式與遊戲引擎通過本地網路連線。
