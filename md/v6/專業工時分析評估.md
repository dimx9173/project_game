## 五、風險評估

### 5.1 高風險項目

| 風險項目 | 影響程度 | 緩解措施 |
|----------|----------|----------|
| 遊戲商 API 不穩定 | 高 | 預留緩衝 + Mock 測試 |
| 第一家遊戲串接難度 | 高 | 派資深工程師 + 預留緩衝 |
| 離線同步邏輯複雜 | 中 | 充分測試各種斷線場景 |
| 一人開發風險 | 高 | 建議至少 2 人協作 |

### 5.2 建議 Buffer

| 項目 | 建議比例 |
|------|----------|
| 開發緩衝 | +15% |
| 測試時間 | +10% |
| Buffer 總計 | +25% |

## 八、結論

1. **原始估計偏樂觀**：三版本估計普遍低估 20-30%
2. **遊戲串接風險最高**：第一家遊戲商需預留足夠緩衝
3. **建議修正報價**：總計增加 40 萬預算緩衝
4. **避免一人開發**：建議至少 2 人協作降低風險
