## 3.1 術語對照表

| 術語 | 英文 | 說明 |
|------|------|------|
| 標配 | Standard | 預設包含的功能 |
| 選配 | Optional | 額外付費或客製化的功能 |
| 風控 | Risk Control | 風險控制機制 |
| 對帳 | Reconciliation | 驗證雙方資料一致性 |
| 同步 | Sync | 資料同步 |
| 充值 | Deposit | 玩家儲值 |

# 4. 開發時程建議

## 4.1 遊戲串接時程評估

| 遊戲商數量 | 額外開發時程 | 說明 |
|------------|--------------|------|
| **5 家以內** | **+2 週** | 標準報價，含框架建置 |
| 6-10 家 | +3 週 | 每家約 2-3 天差異化適配 |
| 10 家以上 | +4 週以上 | 需評估遊戲商類型差異 |

### 📌 計算方式

- **第一家**：1-2 週（建立串接框架 + 熟悉流程）
- **第二家起**：每家 3-5 天（差異化適配）
- **框架成本**：固定，與遊戲商數量無關

### ⚠️ 影響時程的變數

1. 遊戲商 API 文件完整性
2. 遊戲商技術支援响应速度
3. 特殊功能需求（風控、彩池等）
4. 測試環境是否由遊戲商提供
