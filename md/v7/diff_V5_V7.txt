# V5 vs V7 功能與規格差異比較

相較於 V5，V7 版本除了完整包含 V6 的技術升級外，更在**實體門市管理、離線資料同步、資安防護策略**以及**底層伺服器規格**上皆有重大革新。

## 1. 實體門市營運功能新增
*   **交接班管理模組**：新增完整的實體門市交接班作業流程記錄功能，包含各班別的開班/洗班總額統計與當班營收結算。
*   **獨立的單機後台介面**：將原本混用的後台功能拆分，實作出專屬「單機本地後台」的精簡化操作介面與路由，移除不必要的全域權限頁面。
*   **金流來源稽核增強**：底層交易追蹤擴增了「機台代碼 (Machine ID)」與「操作員代碼 (Operator ID)」，可精準溯源每筆交易的經手人與發生設備。

## 2. 離線防護與資安核身升級
*   **機台硬體特徵綁定 (自 V6 導入)**：捨棄單純的帳密連線，新機台強制擷取底層硬體特徵（主機板序號、CPU ID 等）產生硬體指紋。所有請求皆需驗證指紋，防堵偽造機台，並支援一鍵遠端撤銷權限。
*   **事件溯源機制 (Event Sourcing)**：玩家錢包餘額的離線同步機制，由直接覆蓋數值改為「增減變動量 (Delta)」計算，徹底解決斷網期間不同設備造成的餘額衝突。
*   **離線斷網定時心跳鎖 (Dead Man's Switch)**：針對已註冊綁定之機台，新增離線連網的強制時間門檻（如 72 小時未連線），超時後系統將自動鎖死所有單機營運功能防範破解。
*   **安全防呆的資料銷毀機制**：明確設定「資料實體銷毀 (Data Minimization)」的前提要件為「成功同步至總台確認」後，才允許執行刪除本地已過期 8 小時之交易記錄，避免離線斷網營收遭誤刪。

## 3. 架構生命週期與搶修應對機能
*   **註冊前後安全分級**：機台在與總管端完成「初次註冊綁定」前，僅可做為無法連網的純單機遊樂設備運作；合規註冊並獲派 Secret Salt 後，才會正式啟用硬體關聯加密與深度管理。
*   **硬體故障急救通道 (Emergency Recovery)**：當實體主機板損毀導致硬體指紋改變時（將導致無法解鎖 DB 營收），平台新增專修的 Temporary Recovery Token 救援簽發流程，建立合規且安全的營收搶救機制。
*   **OTA 佈署機能擴增**：主系統更新發布能力獲得擴充，新增對「遊戲引擎、靜態美術資源包、前端 Web 資源」等大型資料物件的差異化 (Delta) 下載與斷點續傳支援。

## 4. 規格與技術選型演進
在底層資料庫的防護等級與選型上，V7 推進至極高的資安規格與企業級標準：
*   **集中式總台資料庫**：由 V5 的 `MySQL` 全面升級改用 **`PostgreSQL`**，以應對更嚴謹的資料關聯結構、高併發存取並提供更強健的功能。
*   **單機本地資料庫 (銀行級加密防護)**：由 V5 的 `SQLite / MySQL` 明文儲存，全面轉換並強制升級為 **`SQLCipher`**。本地資料庫不再以明碼形式存在，而是配合每台機器獨一無二的硬體指紋動態算出解鎖金鑰進行 AES-256 全庫加密，達成「即便硬碟被惡意拔除也絕對無法讀取內容」的頂級防駭標準。
