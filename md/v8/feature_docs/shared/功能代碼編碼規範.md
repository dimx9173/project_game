## 📋 概述

本文檔定義功能代碼（Function Code）的編碼規則，以及功能文件與功能矩陣的對應方式。

**現有功能矩陣編號**: [Cxx] 集中式功能 / [Lxx] 本地功能 / [Dxx] 部署功能

**文件結構**: 功能說明文件已按 Central/Local 分離存放

## 📁 文件結構

### Central（集中式後台功能）

路徑：`feature_docs/central/[Mxx] 模組名稱/[Cxx] 功能名稱.md`

**適用範圍**: 所有 [Cxx] 開頭的功能代碼

範例：
```
feature_docs/central/[M01] 儀表板/[C01]KPI 監控.md
feature_docs/central/[M02] 機台管理/[C101] 機台新增編輯刪除.md

```

路徑：`feature_docs/local/[Mxx] 模組名稱/[Lxx] 功能名稱.md`

**適用範圍**: 所有 [Lxx] 開頭的功能代碼

範例：
```
feature_docs/local/[M01] 儀表板/[L01] 營收趨勢圖.md
```

路徑：`feature_docs/shared/`

**適用範圍**: 跨模組的共用規範、檢查清單、報告等

檔案清單：

## 🏷️ 功能代碼編碼規則

### 代碼格式

```
[版本]-[類型]-[功能矩陣編號]

範例：
FLG-CEN-C01  = 旗艦版 + 集中式 + 功能矩陣[C01]
FLG-LOC-L05  = 旗艦版 + 本地 + 功能矩陣[L05]
```

### 版本代碼（第1段）

| 代碼 | 版本 | 說明 |
|------|------|------|
| **FLG** | 旗艦版 | 完整功能（集中式 + 本地後台）|
| **STD** | 標準版 | 僅集中式後台 |

### 類型代碼（第2段）

| 代碼 | 類型 | 對應功能矩陣 | 說明 |
|------|------|-------------|------|
| **CEN** | 集中式 | [Cxx] | Central - 集中式後台功能 |
| **LOC** | 本地 | [Lxx] | Local - 本地後台功能 |
| **DEP** | 部署 | [Dxx] | Deployment - 部署與基礎設施 |

### 功能矩陣編號（第3段）

直接使用功能矩陣中的編號：
- [C01] → 代碼中使用 `C01`
- [L05] → 代碼中使用 `L05`
- [D01] → 代碼中使用 `D01`

### 建立新功能時必須

- [ ] **分配功能代碼**（依照規則 FLG-[類型]-[功能矩陣編號]）
- [ ] **對應功能矩陣編號**（[Cxx] / [Lxx]）
- [ ] **在功能文件中標註功能代碼**（使用標準格式）
- [ ] **在功能矩陣文件中標註實現狀態**（已完成/進行中）
- [ ] **更新本功能代碼總表**

```markdown
### 機台註冊

**功能代碼**: `FLG-CEN-C108`

**模組**: 機台管理 - 註冊認證
```
