# 遊戲機台註冊流程說明（完整版）

## 📋 概述

本文檔說明遊戲機台從出廠到正式營運的完整註冊流程，包含集中端金鑰產生、現場註冊、管理員審核、版本更新等步驟。

---

## 🔄 完整註冊流程

### 階段一：集中端產生註冊金鑰

```
┌─────────────────────────────────────────────────────────────┐
│  管理員登入集中式後台                                         │
│     ↓                                                       │
│  進入「機台部署」功能                                        │
│     ↓                                                       │
│  點擊「產生註冊金鑰」                                         │
│     ↓                                                       │
│  系統產生：                                                  │
│    - 12 碼英數字金鑰（例：X7K9M2P5R8T1）                      │
│    - 有效期 72 小時                                          │
│     ↓                                                       │
│  顯示金鑰並提供複製                                          │
│     ↓                                                       │
│  管理員將金鑰交給現場工作人員                                  │
└─────────────────────────────────────────────────────────────┘
```

**金鑰資訊**:
- **長度**: 12 碼英數字
- **有效期**: 72 小時
- **用途**: 一次性註冊驗證
- **產生位置**: 集中式後台「機台部署」功能

### 階段三：本機註冊請求

```
┌─────────────────────────────────────────────────────────────┐
│  機台連線至指定集中端網址                                      │
│     ↓                                                       │
│  發送註冊請求，包含以下資料：                                  │
│    ├─ 註冊金鑰（12碼）                                        │
│    ├─ 機台名稱                                               │
│    ├─ MAC 位址                                               │
│    ├─ 硬體型號                                               │
│    ├─ 目前軟體版本                                           │
│    ├─ 硬體資訊（CPU/記憶體/磁碟）                             │
│    └─ 時間戳記                                               │
│     ↓                                                       │
│  中央伺服器接收請求                                           │
│     ↓                                                       │
│  驗證金鑰有效性（未過期、未使用過）                             │
│     ↓                                                       │
│  金鑰標記為「已使用」                                         │
│     ↓                                                       │
│  產生暫時 instanceId（待審核狀態）                            │
│     ↓                                                       │
│  機台出現在集中端「待審核機台列表」                            │
│     ↓                                                       │
│  機台顯示「等待管理員審核」狀態                                │
└─────────────────────────────────────────────────────────────┘
```

## 📝 關鍵規格

| 項目 | 規格 |
|------|------|
| **註冊金鑰** | 12 碼英數字 |
| **金鑰有效期** | 72 小時 |
| **初始 PIN** | 000000（6位，首次設定用）|
| **本地 PIN** | 7 位數字（維護模式）|
| **值班 PIN** | 8 位數字（12hr 有效）|
| **審核方式** | 管理員手動 Approved |
| **安全通道** | TLS + 金鑰交換 |
| **版本更新** | OTA 自動更新 |

---

## 🔄 狀態流轉

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  待註冊   │ → │ 等待審核  │ → │  待更新   │ → │  已就緒   │
│ (新機台)  │    │ (已提交)  │    │ (版本舊)  │    │ (可營運)  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

---

## ⚠️ 注意事項

1. **金鑰 72hr 有效** - 過期需重新產生
2. **金鑰一次性** - 使用後立即失效
3. **必須審核** - 未經 Approved 無法建立安全通道
4. **版本強制更新** - 舊版本無法進行設定同步
5. **設定同步** - 僅在最新版本時才會執行
