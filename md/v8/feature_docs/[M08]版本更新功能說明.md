## 3.9 版本更新操作 (OTA 更新)

### [C105] 上傳新版本與版本包控制

**操作流程**：

```
1. 點擊「上傳版本」按鈕
2. 選擇版本檔案
3. 填寫版本資訊
4. 系統計算 SHA-256
5. 驗證檔案完整性
6. 完成上傳
```

**輸入欄位**：

| 欄位名稱 | 類型 | 必填 | 說明         |
| -------- | ---- | ---- | ------------ |
| 版本號碼 | 文字 | ✅   | 格式：v8.0.x |
| 版本說明 | 文字 | ✅   | 更新內容描述 |
| 檔案     | 檔案 | ✅   | .zip 格式    |
| 目標機型 | 多選 | ✅   | 適用機型     |
| 強制更新 | 開關 | -    | 是否強制更新 |

**檔案驗證**：

- 自動計算 MD5
- 自動計算 SHA-256
- 驗證檔案大小
- 檢查壓縮完整性

### [C74] 發布版本

**操作流程**：

```
1. 選擇要發布的版本
2. 選擇測試機台 (2-3台)
3. 開始測試派發
4. 監控 24 小時
5. 確認無異常
6. 正式派發 (全部/分批)
```

**派發模式**：

| 模式     | 說明               |
| -------- | ------------------ |
| 測試派發 | 僅派發至指定機台   |
| 全部派發 | 一次派發至所有機台 |
| 分批派發 | 按批次逐步派發     |
| 定時派發 | 設定時間自動派發   |
