## 3.2 遊戲管理操作

### [L11] 遊戲列表檢視

**操作流程**：

```plaintext
1. 進入「遊戲管理」模組
2. 檢視遊戲列表
3. 使用篩選條件搜尋遊戲
4. 點擊遊戲查看詳情
```

**輸入欄位**：

| 欄位名稱 | 類型     | 說明                  |
| -------- | -------- | --------------------- |
| 遊戲編號 | 文字     | 系統自動產生          |
| 遊戲名稱 | 文字     | 遊戲顯示名稱          |
| 遊戲類型 | 下拉選單 | 老虎機/捕魚/棋牌/街機 |
| 供應商   | 下拉選單 | 遊戲所屬供應商        |
| 狀態     | 標籤     | 啟用/停用/維護中      |
| RTP 率   | 數值     | 機台回報率百分比      |

### [C16] 新增遊戲

**操作流程**：

```
1. 點擊「新增遊戲」按鈕
2. 填寫遊戲基本資料
3. 設定遊戲參數
4. 上傳遊戲圖示/預覽圖
5. 設定遊戲分類
6. 完成新增
```

**輸入欄位**：

| 欄位名稱 | 類型     | 必填 | 說明                    |
| -------- | -------- | ---- | ----------------------- |
| 遊戲名稱 | 文字     | ✅   | 自訂名稱，最長 100 字元 |
| 遊戲代碼 | 文字     | ✅   | 供應商提供的遊戲代碼    |
| 遊戲類型 | 下拉選單 | ✅   | 選擇遊戲類型            |
| 供應商   | 下拉選單 | ✅   | 選擇遊戲供應商          |
| 遊戲分類 | 多選     | ✅   | 選擇所屬分類            |
| RTP 率   | 數值     | ✅   | 範圍 85-99%             |
| 最低押注 | 數值     | ✅   | 最小投注額              |
| 最高押注 | 數值     | ✅   | 最大投注額              |
| 語言支援 | 多選     | ✅   | 支援的語言              |
| 遊戲圖示 | 檔案     | ✅   | 建議尺寸 256x256        |
| 遊戲說明 | 文字     | -    | 遊戲描述                |

### [C16] 編輯遊戲

**操作流程**：

```
1. 在遊戲列表點擊「編輯」按鈕
2. 修改遊戲資料
3. 儲存變更
```

**可編輯欄位**：所有新增時的欄位均可編輯（遊戲代碼除外）

### [L09] 遊戲啟用/停用

**操作流程**：

```
1. 在遊戲列表點擊狀態開關
2. 確認變更
3. 系統自動更新並廣播至所有機台
```

**說明**：

- 停用後機台無法進入該遊戲
- 正在遊戲中的機台會收到提示並自動退出
- 可設定維護中狀態，顯示維護公告

### [L10] 遊戲版本管理

**操作流程**：

```
1. 選擇遊戲後點擊「版本管理」
2. 查看版本歷史
3. 上傳新版本
4. 設定版本為正式版
```

**版本欄位**：

| 欄位名稱 | 類型 | 說明                    |
| -------- | ---- | ----------------------- |
| 版本號   | 文字 | 格式：Major.Minor.Patch |
| 上架日期 | 日期 | 版本發布日期            |
| 狀態     | 標籤 | 測試中/正式版/已下架    |
| 更新內容 | 文字 | 版本變更說明            |

### [L27] 遊戲分類管理

**操作流程**：

```
1. 進入「遊戲分類」子頁籤
2. 新增/編輯/刪除分類
3. 設定分類排序
4. 設定分類圖示
```

### [C53] 遊戲商管理

**功能說明**：
根據遊戲供應商 API 進行動態調整，管理供應商連線與遊戲同步。

**操作流程**：

```
1. 進入「遊戲供應商」子頁籤
2. 查看供應商列表
3. 設定供應商參數
4. 同步遊戲列表
5. 監控 API 連線狀態
```

**供應商欄位**：

| 欄位名稱   | 類型     | 說明                       |
| ---------- | -------- | -------------------------- |
| 供應商名稱 | 文字     | 供應商顯示名稱             |
| API 端點   | URL      | 供應商 API 網址            |
| API 金鑰   | 加密文字 | 連線驗證用                 |
| 連線狀態   | 標籤     | 線上/離線/異常             |
| 最後同步   | 日期時間 | 上次遊戲同步時間           |
| 備註       | 文字     | 依照第三方遊戲商的API 調整 |

### 遊戲搜尋與詳情
- **[L12] 遊戲篩選**：允許管理者依據供應商、遊戲類型、啟用狀態等條件，快速篩選出特定遊戲。
- **[L13] 遊戲搜尋**：支援透過遊戲代碼、名稱的關鍵字輸入直接找尋遊戲。
- **[L14] 遊戲詳情**：點擊遊戲後，可查看遊戲的基本資訊、RTP設定、目前運作狀態與詳細規則。

### [L29] 遊戲供應商管理
**功能說明**：設定與管理各家遊戲供應商的基礎資料、錢包幣別設定及授權狀態。

### [L30] 遊戲大廳設定
**功能說明**：管理大廳內的遊戲顯示版位、推薦順序與跑馬燈公告等前台呈現要素。

### 遊戲配置與顯示設定
- **[C25] 遊戲配置**：設定遊戲在平台上的各種靜態參數與圖片。
- **[L31] 遊戲參數設定**：調整該遊戲的賠率、難度、最低起押金額等內部參數（依據供應商 API 開放程度而定）。
- **[L32] 遊戲顯示順序**：透過拖曳或設定權重值，調整遊戲在列表或大廳的排列順序。
- **[L33] 遊戲狀態查詢**：即時取得遊戲目前的連線狀況與可用狀態，確認是否在維修中。

### 遊戲測試與模擬下注
- **[C28] 遊戲測試**：提供內部人員在正式發佈前，進行進入遊戲與流程測試之介面。
- **[L43] 虛擬下注**：透過測試帳號進行下注模擬，確認派發結果及扣款邏輯，不影響真實報表。
- **[L44] 遊戲餘額查詢**：在測試或客訴期間，用以查詢使用者與該遊戲連結之錢包餘額。
- **[L45] 遊戲開分/洗分**：提供給測試專用的額度加值與扣除操作，測試充提流程。
- **[L46] 遊戲派彩**：模擬玩家獲勝後，系統將派彩金額寫回該測試錢包的動作。
