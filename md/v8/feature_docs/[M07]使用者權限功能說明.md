## 3.4 使用者權限操作

### [C70] 帳號新增/編輯

**操作流程**：

```
1. 進入「使用者管理」
2. 點擊「新增帳號」或「編輯」
3. 填寫帳號基本資料
4. 設定權限角色
5. 若為本地端操作人員，需設定/重置 4-6 位數 PIN 碼
6. 儲存變更
```

**輸入欄位**：

| 欄位名稱    | 類型     | 必填 | 說明                         |
| ----------- | -------- | ---- | ---------------------------- |
| 帳號        | 文字     | ✅   | 登入帳號，最長 50 字元       |
| 密碼        | 密碼     | ✅   | 初始密碼（集中式後台登入用） |
| 本地 PIN 碼 | 密碼數值 | ⚠️ | 供本地端登入，4-6位數字      |
| 姓名        | 文字     | ✅   | 使用者姓名                   |
| Email       | 文字     | ✅   | 通知用 Email                 |
| 電話        | 文字     | -    | 聯絡電話                     |
| 角色        | 下拉選單 | ✅   | 選擇權限角色                 |
| 狀態        | 開關     | ✅   | 啟用/停用                    |

### [C70] 帳號編輯/刪除

**操作流程**：

```
1. 在帳號列表點擊「編輯」
2. 修改帳號資料
3. 儲存變更
```

**刪除帳號**：

- 停用帳號：保留資料但禁止登入
- 刪除帳號：移除帳號資料（需確認）

### [C71] 角色權限設定

**操作流程**：

```
1. 進入「角色管理」
2. 新增/編輯角色
3. 設定功能權限
4. 儲存角色
```

**權限類型**：

| 權限 | 說明           |
| ---- | -------------- |
| 檢視 | 可查看功能畫面 |
| 新增 | 可新增資料     |
| 編輯 | 可修改資料     |
| 刪除 | 可刪除資料     |
| 匯出 | 可匯出資料     |
| 管理 | 可設定權限     |

**預設角色**：

| 角色       | 權限範圍              |
| ---------- | --------------------- |
| 系統管理員 | 全部功能              |
| 站長       | 本站點全部功能        |
| 客服       | 查詢/編輯（不含刪除） |
| 稽核       | 僅檢視/匯出           |

| 權限層級 | 代碼 | 說明             |

#### 3.4.4.3 權限制定原則

```
┌─────────────────────────────────────────────────────┐
│              權限制定流程                             │
├─────────────────────────────────────────────────────┤
│  1. 確認帳號用途                                     │
│     └── 內部人員 → 使用者權限模組                     │
│                                                     │
│  2. 選擇角色                                        │
│     └── 根據工作職責選擇預設角色                      │
│                                                     │
│  3. 客製化調整                                       │
│     └── 依需求增減特定權限                           │
│                                                     │
│  4. 審核確認                                        │
│     └── 主管審核後啟用                               │
└─────────────────────────────────────────────────────┘
```

**核心原則**：

- **最小權限** - 只給完成工作需要的最低權限
- **職責分離** - 稽核與操作分開
- **記錄所有** - 所有權限變更都留日誌

#### 3.4.4.4 動態角色自定義（進階）

系統支援動態角色自定義引擎，可細部勾選 API 權限：

**使用場景**：

- 審計員：僅開放交易查詢 API
- 查帳會計：僅開放報表匯出 API
- 維修外包：僅開放機台狀態查看 API

**自定義權限類型**：

- 功能模組權限
- API 端點權限
- 資料範圍權限（特定站點）
- 時間權限（上班時間才能操作）

### [C72] 操作日誌審計

**功能說明**：
記錄所有使用者的操作行為，供稽核與問題追蹤。

**操作流程**：

```
1. 進入「操作日誌」
2. 設定篩選條件
3. 查詢日誌記錄
4. 匯出日誌
```

**日誌欄位**：

| 欄位名稱 | 說明                  |
| -------- | --------------------- |
| 執行時間 | 操作時間              |
| 帳號     | 操作者帳號            |
| 功能     | 作業模組              |
| 動作     | 新增/編輯/刪除/登入等 |
| 來源 IP  | 客戶端 IP             |
| 詳細內容 | 變更前/後資料         |
