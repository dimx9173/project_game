# [C106] 代理錢包管理詳細流程

**功能代碼**: [C106]  
**所屬模組**: [M03]遊戲管理  
**最後更新**: 2026-03-07  
**優先級**: 🔴 高

---

## 📋 概述

代理錢包管理用於管理遊戲供應商的餘額，包含餘額查詢、轉帳、歸集設定、以及限額控制。

---

## 🔄 完整流程圖

### 餘額查詢流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 管理員進入錢包管理頁面                                  │
│     ↓                                                       │
│  2. 選擇遊戲商                                              │
│     - 從遊戲商列表選擇                                      │
│     - 或搜尋遊戲商                                          │
│     ↓                                                       │
│  3. 點擊「查詢餘額」                                        │
│     ↓                                                       │
│  4. 系統呼叫遊戲商 API                                      │
│     - GET {provider_api}/wallet/balance                    │
│     - 附帶 API 金鑰驗證                                       │
│     ↓                                                       │
│  5. 遊戲商回傳餘額資訊                                      │
│     {                                                       │
│       "balance": 50000.00,                                  │
│       "currency": "TWD",                                    │
│       "lastUpdated": "2026-03-07T10:30:00Z"                 │
│     }                                                       │
│     ↓                                                       │
│  6. 系統顯示餘額                                            │
│     - 顯示可用餘額                                          │
│     - 顯示凍結餘額 (如有)                                   │
│     - 顯示最後更新時間                                      │
│     ↓                                                       │
│  7. 記錄查詢日誌                                            │
│     - 記錄查詢時間                                          │
│     - 記錄查詢結果                                          │
└─────────────────────────────────────────────────────────────┘
```

### 轉帳流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 管理員點擊「轉帳」                                      │
│     ↓                                                       │
│  2. 填寫轉帳資訊                                            │
│     - 選擇來源錢包 (本平台)                                 │
│     - 選擇目標錢包 (遊戲商)                                 │
│     - 輸入轉帳金額                                          │
│     - 輸入備註 (可選)                                       │
│     ↓                                                       │
│  3. 系統驗證轉帳請求                                        │
│     - 檢查來源餘額是否足夠                                  │
│     - 檢查轉帳限額                                          │
│     - 檢查操作員權限                                        │
│     ↓                                                       │
│  4. 風控檢查                                                │
│     - 檢查單筆限額                                          │
│     - 檢查單日累計限額                                      │
│     - 檢查異常交易模式                                      │
│     ↓                                                       │
│  5. 建立轉帳訂單                                            │
│     - 產生訂單編號                                          │
│     - 記錄訂單資訊                                          │
│     - 狀態：待處理                                          │
│     ↓                                                       │
│  6. 發送轉帳請求到遊戲商                                    │
│     - POST {provider_api}/wallet/transfer                  │
│     - 等待回應 (timeout: 30 秒)                               │
│     ↓                                                       │
│  7. 遊戲商處理轉帳                                          │
│     - 驗證請求                                              │
│     - 執行轉帳                                              │
│     - 回傳結果                                              │
│     ↓                                                       │
│  8. 系統更新訂單狀態                                        │
│     - 成功：狀態改為「已完成」                              │
│     - 失敗：狀態改為「失敗」，記錄原因                      │
│     ↓                                                       │
│  9. 記錄操作日誌                                            │
│     - 記錄轉帳詳情                                          │
│     - 記錄操作員                                            │
│     ↓                                                       │
│  10. 返回結果                                               │
│      - 顯示轉帳成功/失敗                                    │
│      - 顯示新餘額                                           │
└─────────────────────────────────────────────────────────────┘
```

### 歸集設定流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 管理員進入歸集設定頁面                                  │
│     ↓                                                       │
│  2. 選擇遊戲商                                              │
│     ↓                                                       │
│  3. 設定歸集參數                                            │
│     - 啟用/停用自動歸集                                     │
│     - 設定歸集閾值 (餘額 > X 時觸發)                         │
│     - 設定歸集金額 (固定金額或百分比)                       │
│     - 設定歸集時間 (定時執行)                               │
│     ↓                                                       │
│  4. 系統驗證設定                                            │
│     - 檢查閾值合理性                                        │
│     - 檢查金額合理性                                        │
│     ↓                                                       │
│  5. 儲存設定                                                │
│     - 更新資料庫                                            │
│     - 記錄設定變更                                          │
│     ↓                                                       │
│  6. 啟用定時任務                                            │
│     - 設定定時檢查 (每日/每週)                              │
│     - 符合條件時自動執行歸集                               │
└─────────────────────────────────────────────────────────────┘
```

### 限額控制流程

```
┌─────────────────────────────────────────────────────────────┐
│  轉帳請求                                                   │
│     ↓                                                       │
│  檢查單筆限額                                               │
│  - 最小金額：$100                                           │
│  - 最大金額：$100,000                                       │
│     ↓ 超過                                                   │
│  拒絕交易，顯示錯誤訊息                                      │
│     ↓ 符合                                                   │
│  檢查單日累計限額                                           │
│  - 單日最大轉出：$500,000                                   │
│  - 單日最大轉入：$1,000,000                                 │
│     ↓ 超過                                                   │
│  拒絕交易，顯示錯誤訊息                                      │
│     ↓ 符合                                                   │
│  檢查單月累計限額                                           │
│  - 單月最大轉出：$10,000,000                                │
│     ↓ 超過                                                   │
│  拒絕交易，需要人工審核                                      │
│     ↓ 符合                                                   │
│  允許交易，繼續處理                                          │
└─────────────────────────────────────────────────────────────┘
```

## ⚠️ 異常處理

### 1. 餘額不足

**情境**: 轉帳時來源餘額不足

**處理流程**:
```
1. 系統檢查來源餘額
   ↓
2. 餘額 < 轉帳金額
   ↓
3. 拒絕交易
   - 顯示錯誤訊息：「餘額不足，目前餘額：$XXX」
   - 建議操作：先充值或減少轉帳金額
   ↓
4. 記錄嘗試日誌
```

### 2. API 連線失敗

**情境**: 無法連線到遊戲商 API

**處理流程**:
```
1. 發送 API 請求超時 (30 秒)
   ↓
2. 重試機制:
   - 第 1 次重試：間隔 10 秒
   - 第 2 次重試：間隔 30 秒
   - 第 3 次重試：間隔 60 秒
   ↓
3. 3 次重試均失敗:
   - 標記交易失敗
   - 記錄錯誤事件
   - 通知管理員
   - 建議稍後重試
```

### 3. 限額超過

**情境**: 轉帳金額超過限額

**處理流程**:
```
1. 系統檢查限額
   ↓
2. 超過單筆限額:
   - 拒絕交易
   - 顯示限額資訊
   - 建議分批轉帳
   ↓
3. 超過單日/單月限額:
   - 拒絕交易
   - 顯示已使用限額
   - 建議明日/下月再執行
   - 或申請提高限額 (需審核)
```

### 4. 交易狀態不一致

**情境**: 遊戲商已扣款但本平台未收到確認

**處理流程**:
```
1. 發送轉帳請求後未收到回應
   ↓
2. 查詢交易狀態:
   - GET {provider_api}/transactions/{id}
   ↓
3. 比對結果:
   - 遊戲商已處理：同步狀態，完成交易
   - 遊戲商未處理：重新發送或取消
   ↓
4. 記錄不一致事件
   ↓
5. 必要時人工介入對帳
```

## 🔌 API 調用

### 內部 API

| 順序 | API | 方法 | 說明 |
|------|-----|------|------|
| 1 | `/api/wallet/{providerId}/balance` | GET | 查詢餘額 |
| 2 | `/api/wallet/transfer` | POST | 執行轉帳 |
| 3 | `/api/wallet/auto-collect/settings` | GET/POST | 歸集設定 |
| 4 | `/api/wallet/limits` | GET | 查詢限額 |
| 5 | `/api/wallet/transactions` | GET | 查詢交易紀錄 |

### 遊戲商錢包 API 範例

```http
# 查詢餘額
GET {provider_api}/wallet/balance
Authorization: Bearer {api_key}

Response:
{
  "success": true,
  "balance": 50000.00,
  "frozenBalance": 5000.00,
  "currency": "TWD",
  "lastUpdated": "2026-03-07T10:30:00Z"
}

# 執行轉帳
POST {provider_api}/wallet/transfer
Authorization: Bearer {api_key}
Content-Type: application/json

{
  "amount": 10000,
  "direction": "IN",  // IN: 轉入，OUT: 轉出
  "remark": "週結歸集"
}

Response:
{
  "success": true,
  "transactionId": "TXN-20260307-001",
  "newBalance": 60000.00,
  "timestamp": "2026-03-07T10:35:00Z"
}
```

## 💾 資料庫操作

### 涉及的資料表

| 資料表 | 操作 | 說明 |
|--------|------|------|
| `provider_wallets` | INSERT/UPDATE | 記錄遊戲商錢包資訊 |
| `wallet_transactions` | INSERT | 記錄轉帳交易 |
| `wallet_limits` | INSERT/UPDATE | 記錄限額設定 |
| `auto_collect_settings` | INSERT/UPDATE | 記錄歸集設定 |
| `operation_logs` | INSERT | 記錄操作日誌 |

### 資料表結構

#### provider_wallets

```sql
CREATE TABLE provider_wallets (
    provider_id VARCHAR(50) PRIMARY KEY,
    balance DECIMAL(12,2) NOT NULL DEFAULT 0,
    frozen_balance DECIMAL(12,2) NOT NULL DEFAULT 0,
    currency VARCHAR(10) DEFAULT 'TWD',
    last_sync_at TIMESTAMP,
    status VARCHAR(20) NOT NULL, -- 'ACTIVE', 'INACTIVE', 'SUSPENDED'
    updated_at TIMESTAMP NOT NULL
);
```

#### wallet_transactions

```sql
CREATE TABLE wallet_transactions (
    id VARCHAR(50) PRIMARY KEY,
    provider_id VARCHAR(50) NOT NULL,
    transaction_type VARCHAR(20) NOT NULL, -- 'TRANSFER_IN', 'TRANSFER_OUT'
    amount DECIMAL(12,2) NOT NULL,
    balance_before DECIMAL(12,2),
    balance_after DECIMAL(12,2),
    status VARCHAR(20) NOT NULL, -- 'PENDING', 'COMPLETED', 'FAILED'
    provider_transaction_id VARCHAR(100),
    remark TEXT,
    operator_id VARCHAR(50) NOT NULL,
    created_at TIMESTAMP NOT NULL,
    completed_at TIMESTAMP,
    FOREIGN KEY (provider_id) REFERENCES provider_wallets(provider_id)
);
```

#### wallet_limits

```sql
CREATE TABLE wallet_limits (
    id VARCHAR(50) PRIMARY KEY,
    provider_id VARCHAR(50) NOT NULL,
    single_min DECIMAL(12,2) DEFAULT 100,
    single_max DECIMAL(12,2) DEFAULT 100000,
    daily_max DECIMAL(12,2) DEFAULT 500000,
    monthly_max DECIMAL(12,2) DEFAULT 10000000,
    updated_at TIMESTAMP NOT NULL,
    updated_by VARCHAR(50) NOT NULL,
    FOREIGN KEY (provider_id) REFERENCES provider_wallets(provider_id)
);
```

## 📊 限額設定

### 預設限額

| 限額類型 | 預設值 | 說明 |
|---------|--------|------|
| 單筆最小 | $100 | 單筆轉帳最小金額 |
| 單筆最大 | $100,000 | 單筆轉帳最大金額 |
| 單日最大轉出 | $500,000 | 單日累計轉出上限 |
| 單日最大轉入 | $1,000,000 | 單日累計轉入上限 |
| 單月最大轉出 | $10,000,000 | 單月累計轉出上限 |

### 限額調整流程

```
1. 管理員申請調整限額
   ↓
2. 填寫申請資訊:
   - 申請調整的限額類型
   - 目前限額
   - 申請新限額
   - 調整原因
   ↓
3. 系統驗證申請:
   - 檢查申請人權限
   - 檢查調整幅度 (超過 50% 需高階審核)
   ↓
4. 審核流程:
   - 一般調整：主管審核
   - 大幅調整：高階主管審核
   ↓
5. 審核通過:
   - 更新限額設定
   - 記錄變更日誌
   - 通知申請人
   ↓
6. 審核拒絕:
   - 記錄拒絕原因
   - 通知申請人
```
