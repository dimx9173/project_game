## 操作流程

### 2.1 查看通知列表

```
┌─────────────────────────────────────────────────────────────┐
│  1. 進入儀表板                                              │
│     通知圖示顯示未讀數量                                     │
│     ↓                                                       │
│  2. 點擊通知圖示                                            │
│     展開通知下拉選單                                         │
│     ↓                                                       │
│  3. 瀏覽通知                                                │
│     - 最新通知置頂                                           │
│     - 按時間排序                                            │
│     - 顯示通知等級與類型                                     │
│     ↓                                                       │
│  4. 標記已讀                                                │
│     - 單筆標記：點擊通知                                     │
│     - 全部標記：點擊「全部標記已讀」                          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 通知類型與等級

| 通知類型 | 說明 | 預設等級 | 通知管道 |
|----------|------|----------|----------|
| 系統維護 | 系統例行維護通知 | INFO | 站內 |
| 硬體異常 | 機台硬體問題告警 | ERROR | 站內+簡訊 |
| 網路異常 | 機台離線/連線不穩 | WARN | 站內+推播 |
| 交易異常 | 開洗分失敗/對帳不平 | ERROR | 站內+簡訊 |
| 庫存不足 | 機台餘額不足 | WARN | 站內 |
| 版本更新 | OTA 更新狀態 | INFO | 站內 |
| 權限變更 | 帳號權限調整 | INFO | 站內 |
| 審核提醒 | 待審核項目 | WARN | 站內+推播 |

### 2.3 通知管道設定

```
┌─────────────────────────────────────────────────────────────┐
│  1. 進入「系統設定」->「通知設定」                          │
│     ↓                                                       │
│  2. 選擇通知類型                                            │
│     - 勾選要接收的通知類型                                   │
│     ↓                                                       │
│  3. 選擇通知管道                                            │
│     - 站內通知：儀表板即時彈出                               │
│     - 電子郵件：Email 推播                                  │
│     - 簡訊：SMS 通知 (重要告警)                             │
│     - 推播：Telegram/Discord/Slack Webhook                 │
│     ↓                                                       │
│  4. 設定接收人                                              │
│     - 指定人員或角色                                         │
│     - 群組通知                                               │
│     ↓                                                       │
│  5. 儲存設定                                                │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 通知觸發條件

| 事件 | 觸發條件 | 通知內容 |
|------|----------|----------|
| 機台離線 | 連續 6 次心跳無回應 (>30秒) | 機台名稱、離線時間、可能原因 |
| CPU 過高 | CPU > 90% 持續 1 分鐘 | 機台名稱、CPU 使用率 |
| 交易失敗 | 開分/洗分 API 回傳失敗 | 訂單編號、機台名稱、錯誤原因 |
| 對帳不平 | 中央與本地餘額差異 > 0.01 | 機台名稱、差異金額、差異時間 |
| 庫存不足 | 機台餘額 < 設定門檻 | 機台名稱、目前餘額、門檻值 |

### 2.5 通知歷史與匯出

```
┌─────────────────────────────────────────────────────────────┐
│  1. 進入「監控中心」->「通知歷史」                          │
│     ↓                                                       │
│  2. 篩選條件                                                │
│     - 時間範圍                                               │
│     - 通知類型                                               │
│     - 通知等級                                               │
│     - 機台                                                   │
│     ↓                                                       │
│  3. 查看詳情                                                │
│     - 點擊通知查看完整內容                                   │
│     - 包含處理記錄                                           │
│     ↓                                                       │
│  4. 匯出記錄                                                │
│     - 匯出 CSV/Excel                                        │
│     - 用於稽核或分析                                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.6 數據來源與即時性

- **即時推播**：透過 SignalR WebSocket 即時推送到前端
- **外部通知**：Webhook 異步發送到 Telegram/Discord/SMS
- **儲存周期**：預設保留 90 天，可設定
