## 操作流程

### 2.1 查看硬體狀態

```
┌─────────────────────────────────────────────────────────────┐
│  1. 進入儀表板                                              │
│     預設顯示所有機台硬體狀態概覽                              │
│     ↓                                                       │
│  2. 檢視關鍵指標                                            │
│     - CPU 使用率：顯示即時負載                               │
│     - 記憶體使用：顯示使用量與可用空間                         │
│     - 硬碟空間：顯示剩餘容量                                │
│     ↓                                                       │
│  3. 檢視網路狀態                                            │
│     - 連線狀態：線上/離線                                   │
│     - 延遲時間：Ping 值 (ms)                                 │
│     ↓                                                       │
│  4. 異常標記                                                │
│     - 紅色：超過閾值 (如 CPU >90%)                          │
│     - 黃色：接近閾值 (如 CPU >80%)                          │
│     - 綠色：正常範圍                                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 監控項目與閾值

| 項目 | 正常 | 警告 (黃) | 嚴重 (紅) |
|------|------|-----------|-----------|
| CPU 使用率 | < 70% | 70-90% | > 90% |
| 記憶體使用 | < 70% | 70-85% | > 85% |
| 硬碟空間 | > 20% | 10-20% | < 10% |
| 網路延遲 | < 50ms | 50-100ms | > 100ms |
| 溫度 | < 50°C | 50-60°C | > 60°C |

### 2.3 查看單一機台詳細資訊

```
┌─────────────────────────────────────────────────────────────┐
│  1. 點擊機台卡片或列表中的機台                               │
│     ↓                                                       │
│  2. 展開硬體詳細資訊                                         │
│     - CPU 型號與即時使用率                                   │
│     - 記憶體總量與使用量                                     │
│     - 硬碟使用量與分割區                                     │
│     ↓                                                       │
│  3. 查看網路連線詳情                                        │
│     - IP 位址                                                │
│     - 網路介面狀態                                           │
│     - 流量統計 (上傳/下載)                                   │
│     ↓                                                       │
│  4. 查看歷史趨勢                                            │
│     - 選擇時間範圍 (1小時/24小時/7天)                        │
│     - 查看效能趨勢圖                                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.4 告警處理流程

```
┌─────────────────────────────────────────────────────────────┐
│  1. 收到告警通知                                            │
│     - 儀表板彈出告警提示                                     │
│     - 通知到設定的管道 (Telegram/Discord/SMS)               │
│     ↓                                                       │
│  2. 確認告警內容                                            │
│     - 查看告警等級 (INFO/WARN/ERROR/CRITICAL)              │
│     - 查看涉及的機台與項目                                    │
│     ↓                                                       │
│  3. 進行處理                                                │
│     - 小問題：遠端重啟服務                                   │
│     - 大問題：派工單給工程師前往現場                          │
│     ↓                                                       │
│  4. 標記處理完成                                            │
│     - 記錄處理方式與結果                                    │
│     - 關閉告警                                              │
└─────────────────────────────────────────────────────────────┘
```

### 2.5 數據來源

- **即時資料**：機台每分鐘上報硬體狀態 (CPU/記憶體/磁碟)
- **心跳機制**：機台每 5 秒發送 Heartbeat 至 RabbitMQ
- **網路監控**：機台每 15 秒 Ping 中央伺服器一次
