# [C11] 開分/洗分詳細流程

**功能代碼**: [C11]  
**所屬模組**: [M02]機台管理  
**最後更新**: 2026-03-07  
**優先級**: 🔴 最高優先  
**最後更新**: 2026-03-07 (v1.1 - 增加交易 ROLLBACK 機制)

開分/洗分是機台管理的核心功能，涉及資金安全與營運流程。本流程說明從值班人員登入到完成開分/洗分的完整操作流程。

---

## 🔄 完整流程圖

### 開分流程（充值）

```
┌─────────────────────────────────────────────────────────────┐
│  1. 值班人員登入機台後台                                     │
│     輸入 8 位值班 PIN                                        │
│     ↓                                                       │
│  2. 系統驗證 PIN 有效性                                      │
│     - 檢查 PIN 長度 (8 位)                                    │
│     - 檢查有效期 (12 小時)                                   │
│     - 檢查鎖定狀態 (5 次失敗鎖定)                             │
│     ↓                                                       │
│  3. 選擇機台與開分金額                                       │
│     - 選擇目標機台                                           │
│     - 輸入開分金額 (最小 10 元，最大 50,000 元)                │
│     - 選擇支付方式 (現金/轉帳/其他)                          │
│     ↓                                                       │
│  4. 風控檢查                                                 │
│     - 檢查機台餘額門檻                                       │
│     - 檢查單日累計限額                                       │
│     - 檢查異常交易模式                                       │
│     ↓                                                       │
│  5. 建立開分訂單                                             │
│     - 產生訂單編號 (ORDER-YYYYMMDD-XXXX)                     │
│     - 記錄訂單資訊                                           │
│     - 狀態：待處理                                           │
│     ↓                                                       │
│  6. 發送開分指令到機台                                       │
│     - POST /api/machines/{id}/cash-in                       │
│     - 等待機台確認 (timeout: 30 秒)                           │
│     ↓                                                       │
│  7. 機台確認接收                                             │
│     - 機台更新本地餘額                                       │
│     - 回傳確認訊息                                           │
│     ↓                                                       │
│  8. 同步餘額到本地資料庫                                     │
│     - UPDATE machine_balance SET balance = balance + amount │
│     - 記錄同步時間                                           │
│     ↓                                                       │
│  9. 記錄操作日誌                                             │
│     - INSERT operation_logs                                 │
│     - 記錄操作人員、時間、金額                               │
│     ↓                                                       │
│  10. 返回成功訊息                                            │
│      - 顯示開分成功                                          │
│      - 顯示新餘額                                            │
│      - 可選：列印收據                                        │
└─────────────────────────────────────────────────────────────┘
```

## ⚠️ 異常處理

### 1. 餘額不足

**情境**: 洗分時機台餘額不足

**處理流程**:
```
1. 系統檢查機台餘額
   ↓
2. 餘額 < 洗分金額
   ↓
3. 顯示錯誤訊息：「機台餘額不足，目前餘額：$XXX」
   ↓
4. 建議操作：
   - 先進行開分充值
   - 減少洗分金額
   ↓
5. 記錄嘗試日誌 (不計入失敗次數)
```

### 2. PIN 驗證失敗

**情境**: 值班人員輸入錯誤 PIN

**處理流程**:
```
1. 系統驗證 PIN 失敗
   ↓
2. 記錄失敗次數 (剩餘嘗試次數：5-N)
   ↓
3. 剩餘次數 > 0:
   - 顯示錯誤訊息：「PIN 錯誤，剩餘嘗試次數：X」
   - 允許重新輸入
   ↓
4. 剩餘次數 = 0:
   - 鎖定帳號 (15 分鐘)
   - 顯示鎖定訊息
   - 記錄鎖定事件
   - 通知管理員
```

### 3. 連線中斷

**情境**: 與機台通訊中斷

**處理流程**:
```
1. 發送指令超時 (30 秒)
   ↓
2. 自動重試機制:
   - 第 1 次重試：立即重試
   - 第 2 次重試：間隔 5 秒
   - 第 3 次重試：間隔 10 秒
   ↓
3. 3 次重試均失敗:
   - 訂單狀態改為「失敗」
   - 加入重試佇列 (稍後處理)
   - 通知管理員
   - 顯示錯誤訊息：「與機台連線中斷，請檢查網路」
```

### 4. 超時處理

**情境**: 機台回應超時

**處理流程**:
```
1. 等待機台確認超時 (30 秒)
   ↓
2. 查詢機台狀態:
   - GET /api/machines/{id}/status
   ↓
3. 檢查訂單狀態:
   - 如已處理：同步餘額，完成流程
   - 如未處理：重新發送指令或取消訂單
   ↓
4. 記錄超時事件
```

### 5. 餘額同步失敗

**情境**: 機台餘額與中央不一致

**處理流程**:
```
1. 比對機台餘額與中央紀錄
   ↓
2. 差異 > 容許範圍 (0.01 元):
   - 觸發對帳流程
   - 暫停該機台交易
   - 通知管理員審核
   ↓
3. 差異 <= 容許範圍:
   - 以中央紀錄為準
   - 同步機台餘額
   - 記錄差異日誌
```

## 🔒 交易安全性

### 交易狀態機

```
         ┌──────────┐
         │ PENDING  │
         └────┬─────┘
              │ 開始處理
              ↓
         ┌──────────┐
    ┌───→│PROCESSING│───┐
    │    └──────────┘   │
    │         │         │
    │    成功 │    失敗 │
    │         ↓         ↓
    │    ┌──────────┐ ┌──────────┐
    └───│COMPLETED │ │ FAILED   │
         └──────────┘ └──────────┘
              │         │
              │   已回滾 │
              ↓         ↓
         ┌──────────┐
         │ROLLED_BACK│
         └──────────┘
```

### 交易失敗 ROLLBACK 機制

**情境**: 交易過程中發生錯誤

**處理流程**:
```
1. 檢測到交易失敗
   - 機台回覆失敗
   - 網路超時
   - 資料庫錯誤
   ↓
2. 執行回滾邏輯:
   
   a) 訂單狀態回滾:
      - UPDATE cash_orders SET status = 'ROLLED_BACK'
      - 記錄回滾原因
      
   b) 餘額回滾 (如已更新):
      - 如果是「開分」已更新餘額:
        UPDATE machine_balance 
        SET balance = balance - amount
        WHERE machine_id = ?
        
      - 如果是「洗分」已更新餘額:
        UPDATE machine_balance 
        SET balance = balance + amount
        WHERE machine_id = ?
        
   c) 記錄失敗日誌:
      - INSERT INTO operation_logs (result = 'FAILED', error_message = ?)
      
   d) 發送失敗通知:
      - 通知管理員交易失敗
      - 通知相關人員餘額已回滾
   ↓
3. 返回失敗訊息給客戶端
```

### 交易原子性範例

```sql
-- 開分交易 (帶失敗回滾)
BEGIN TRANSACTION;

BEGIN TRY
    -- 1. 建立訂單
    INSERT INTO cash_orders (id, order_type, machine_id, amount, payment_method, operator_id, status, created_at)
    VALUES ('ORDER-20260307-0001', 'CASH_IN', 'MACHINE001', 1000, 'cash', 'OP001', 'PROCESSING', NOW());

    -- 2. 檢查餘額 (樂觀鎖)
    UPDATE machine_balance 
    SET balance = balance + 1000, 
        last_updated = NOW(), 
        last_operator_id = 'OP001',
        version = version + 1
    WHERE machine_id = 'MACHINE001'
      AND version = @current_version;  -- 檢查版本號

    -- 3. 檢查是否更新成功 (受影響行數為 0 表示失敗)
    IF @@ROWCOUNT = 0
    BEGIN
        -- 版本衝突或其他錯誤，回滾
        ROLLBACK;
        THROW 50001, '餘額更新失敗', 1;
    END

    -- 4. 記錄操作日誌
    INSERT INTO operation_logs (id, operator_id, action, target_id, amount, result, created_at)
    VALUES ('LOG-001', 'OP001', 'CASH_IN', 'MACHINE001', 1000, 'SUCCESS', NOW());

    -- 5. 更新訂單狀態
    UPDATE cash_orders SET status = 'COMPLETED', completed_at = NOW()
    WHERE id = 'ORDER-20260307-0001';

    COMMIT;
    
END TRY
BEGIN CATCH
    -- 發生任何錯誤，執行回滾
    ROLLBACK;
    
    -- 記錄錯誤日誌
    INSERT INTO operation_logs (id, operator_id, action, target_id, amount, result, error_message, created_at)
    VALUES (NEWID(), 'OP001', 'CASH_IN', 'MACHINE001', 1000, 'FAILED', ERROR_MESSAGE(), NOW());
    
    -- 重新拋出錯誤
    THROW;
END CATCH
```

### 並發控制 - 樂觀鎖

```sql
-- 使用 version 欄位實現樂觀鎖
ALTER TABLE machine_balance ADD version INT DEFAULT 1 NOT NULL;

-- 更新時檢查版本
UPDATE machine_balance 
SET balance = balance + @amount,
    version = version + 1,
    last_updated = NOW()
WHERE machine_id = @machine_id
  AND version = @expected_version;  -- 只有版本匹配才更新

-- 檢查更新結果
IF @@ROWCOUNT = 0
BEGIN
    -- 版本衝突！表示有其他交易正在修改
    -- 返回錯誤，讓客戶端重試
    RAISERROR('版本衝突，請重試', 16, 1);
END
```

### 並發控制 - 悲觀鎖 (SELECT FOR UPDATE)

```sql
-- 使用悲觀鎖鎖定餘額記錄
BEGIN TRANSACTION;

-- 鎖定機台餘額記錄 (其他交易必須等待)
SELECT balance, version 
FROM machine_balance 
WITH (UPDLOCK, ROWLOCK)
WHERE machine_id = 'MACHINE001';

-- 檢查餘額是否足夠
IF @balance >= @amount
BEGIN
    -- 更新餘額
    UPDATE machine_balance 
    SET balance = balance - @amount,
        last_updated = NOW()
    WHERE machine_id = 'MACHINE001';
    
    COMMIT;
END
ELSE
BEGIN
    -- 餘額不足
    ROLLBACK;
    RAISERROR('餘額不足', 16, 1);
END
```

## 💾 資料庫操作

### 涉及的資料表

| 資料表 | 操作 | 說明 |
|--------|------|------|
| `cash_orders` | INSERT | 記錄開分/洗分訂單 |
| `machine_balance` | UPDATE | 更新機台餘額 |
| `operation_logs` | INSERT | 記錄操作日誌 |
| `risk_control_logs` | INSERT | 記錄風控檢查結果 |

### 資料表結構

#### cash_orders

```sql
CREATE TABLE cash_orders (
    id VARCHAR(50) PRIMARY KEY,
    order_type VARCHAR(10) NOT NULL, -- 'CASH_IN' or 'CASH_OUT'
    machine_id VARCHAR(50) NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    payment_method VARCHAR(20),
    operator_id VARCHAR(50) NOT NULL,
    status VARCHAR(20) NOT NULL, -- 'PENDING', 'COMPLETED', 'FAILED'
    created_at TIMESTAMP NOT NULL,
    completed_at TIMESTAMP,
    remark TEXT
);
```

#### machine_balance

```sql
CREATE TABLE machine_balance (
    machine_id VARCHAR(50) PRIMARY KEY,
    balance DECIMAL(10,2) NOT NULL,
    last_updated TIMESTAMP NOT NULL,
    last_operator_id VARCHAR(50),
    version INT NOT NULL
);
```

#### operation_logs

```sql
CREATE TABLE operation_logs (
    id VARCHAR(50) PRIMARY KEY,
    operator_id VARCHAR(50) NOT NULL,
    action VARCHAR(50) NOT NULL, -- 'CASH_IN', 'CASH_OUT'
    target_id VARCHAR(50), -- machine_id
    amount DECIMAL(10,2),
    result VARCHAR(20) NOT NULL, -- 'SUCCESS', 'FAILED'
    error_message TEXT,
    created_at TIMESTAMP NOT NULL
);
```

### 交易操作範例

```sql
-- 開分交易
BEGIN TRANSACTION;

-- 1. 建立訂單
INSERT INTO cash_orders (id, order_type, machine_id, amount, payment_method, operator_id, status, created_at)
VALUES ('ORDER-20260307-0001', 'CASH_IN', 'MACHINE001', 1000, 'cash', 'OP001', 'PENDING', NOW());

-- 2. 更新機台餘額
UPDATE machine_balance 
SET balance = balance + 1000, 
    last_updated = NOW(), 
    last_operator_id = 'OP001',
    version = version + 1
WHERE machine_id = 'MACHINE001';

-- 3. 記錄操作日誌
INSERT INTO operation_logs (id, operator_id, action, target_id, amount, result, created_at)
VALUES ('LOG-001', 'OP001', 'CASH_IN', 'MACHINE001', 1000, 'SUCCESS', NOW());

-- 4. 更新訂單狀態
UPDATE cash_orders SET status = 'COMPLETED', completed_at = NOW()
WHERE id = 'ORDER-20260307-0001';

COMMIT;
```

## 📊 風控規則

### 單筆限額

| 交易類型 | 最小金額 | 最大金額 |
|---------|---------|---------|
| 開分 | $10 | $50,000 |
| 洗分 | $10 | $50,000 |

### 單日累計限額

| 交易類型 | 單台機器日上限 | 單操作員日上限 |
|---------|-------------|-------------|
| 開分 | $200,000 | $500,000 |
| 洗分 | $200,000 | $500,000 |

### 異常交易偵測

| 規則 | 閾值 | 處理方式 |
|------|------|---------|
| 短時間多次交易 | 5 次/10 分鐘 | 觸發人工審核 |
| 大額交易 | > $30,000 | 觸發人工審核 |
| 餘額異常波動 | > 50%/小時 | 暫停交易，通知管理員 |
