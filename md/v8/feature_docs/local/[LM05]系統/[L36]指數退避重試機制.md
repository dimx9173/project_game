# [L36] 指數退避重試機制

**功能代碼**: L36  
**所屬模組**: [M05]系統  
**最後更新**: 2026-03-09

---

## 1. 功能說明

指數退避重試機制（Exponential Backoff）用於處理網路暫時不穩定義或伺服器繁忙時的重試策略。其核心思想是隨著重試次數的增加，逐漸拉長兩次重試之間的時間間隔，以避免對伺服器造成雪崩效應。

### 運算規則：
- `WaitTime = InitialDelay * (Multiplier ^ RetryCount)`
- 使用隨機抖動（Jitter）來分散多台機台同時重試的壓力。

---

## 2. 操作流程

### 2.1 自動重試邏輯
1. **偵測失敗**：當 [L35] 同步任務返回連線超時、網路中斷或 503 伺服器忙碌。
2. **計算延遲**：
   - 第 1 次：立即重試。
   - 第 2 次：等待 10 秒。
   - 第 3 次：等待 10 * (2^1) = 20 秒。
   - 第 4 次：等待 10 * (2^2) = 40 秒...依此類推。
3. **執行重試**：在計畫的時間點重新發起 API 請求。
4. **終止保護**：若重試次數達到 5 次仍失敗，該任務將標記為「永久失敗」並暫存於佇列中，直到網路狀態顯著改善或人工干預。

---

## 3. 退避參數設定 (唯讀)

| 參數名稱         | 預設值         | 說明                                 |
| ---------------- | -------------- | ------------------------------------ |
| 初始延遲 (Base)  | 10 秒          | 第一次重試前的等待時間               |
| 指數乘積 (Factor)| 2              | 每次失敗後間隔時間的倍數             |
| 最大重試次數     | 5 次           | 單一任務放棄前的最大嘗試次數         |
| 最大等待上限     | 300 秒 (5min)  | 間隔時間的最高上限                   |
| 抖動係數 (Jitter)| 0.1 (10%)      | 為每次等待時間加入隨機浮動，避免同步 |
