# [L28] 備份與恢復

**功能代碼**: L28  
**所屬模組**: [M05]系統  
**最後更新**: 2026-03-09

---

## 1. 功能說明

本地系統備份與恢復功能旨在確保機台資料的安全性與完整性。在發生硬體故障、軟體錯誤或資料損毀時，管理者可以透過備份檔案快速恢復系統至先前穩定的狀態。

### 主要功能：
- **手動備份**：管理者可隨時觸發系統資料備份。
- **自動備份**：系統根據設定的頻率自動生成備份。
- **資料恢復**：從選定的備份檔案中還原系統設定與交易日誌。

---

## 2. 操作流程

### 2.1 系統備份流程
1. **進入設定頁面**：登入本地後台，進入「系統管理」->「備份與恢復」。
2. **觸發備份**：點擊「立即備份」按鈕。
3. **執行備份**：系統暫停寫入非關鍵資料，進行資料庫快照。
4. **生成檔案**：將資料庫檔案與關鍵配置壓縮為加密備份檔（`.bak`）。
5. **備份完成**：顯示備份成功資訊與備份檔案名稱（含時間戳記）。

### 2.2 資料恢復流程
1. **選擇備份檔案**：從備份清單中選擇欲恢復的檔案，或透過 USB 匯入。
2. **檔案驗證**：系統自動驗證備份檔案的完整性與版本相容性。
3. **確認恢復**：顯示警告資訊（恢復將覆蓋當前資料），管理者輸入 PIN 碼確認。
4. **執行恢復**：系統進入維護模式，重寫資料庫檔案。
5. **重啟驗證**：系統自動重啟，驗證資料一致性後恢復服務。

## 3. 屬性/輸入欄位

| 欄位名稱     | 說明                             | 類型     | 限制條件             |
| ------------ | -------------------------------- | -------- | -------------------- |
| 備份檔名     | 備份檔案的顯示名稱               | String   | 自動生成 (TIMESTAMP) |
| 備份時間     | 該備份檔案生成的精確時間         | DateTime | 唯讀                 |
| 備份大小     | 備份檔案佔用的磁碟空間           | Size     | 唯讀                 |
| 自動備份開關 | 是否啟用系統週期性自動備份       | Boolean  | 開/關                |
| 備份頻率     | 自動備份的時間間隔（如：每日）   | Select   | 每日/每週            |
| 儲存路徑     | 備份檔案存放的資料夾（內部/外部） | Selection| Local / USB          |
