# [L42] 底層通訊與金鑰加密

**功能代碼**: L42  
**所屬模組**: [M05]系統  
**最後更新**: 2026-03-09

---

## 1. 功能說明

本地端與中央伺服器之間的通訊採用多層加密機制，確保資料在傳輸過程中的安全性（Confidentiality）、完整性（Integrity）與不可否認性（Non-repudiation）。

### 主要功能：
- **雙向 TLS 通訊**：基於憑證的傳輸層安全協議。
- **動態金鑰對**：機台首次註冊時自動生成唯一金鑰對。
- **報文加密**：除了傳輸層加密外，關鍵資產報文（如開洗分）進行應用層二次加密。

---

## 2. 操作流程

### 2.1 初始化金鑰流程
1. **硬體指紋收集**：系統啟動初期提取 CPU ID、MAC 地址等硬體標識。
2. **生成金鑰對**：利用本地安全模組（HSM）或軟體加密庫生成 RSA-4096 / ECC 金鑰對。
3. **公鑰上傳**：在機台註冊階段（[L51]），公鑰隨註冊請求上傳至中央伺服器。
4. **憑證簽署**：中央伺服器驗證合法後，返回簽署後的設備憑證。

### 2.2 建立加密通訊流程
1. **握手請求**：本地發起 WebSocket 請求。
2. **身份證明**：本地出示簽署憑證，中央進行 CA 鏈驗證。
3. **會話金鑰協商**：通過 Diffie-Hellman (ECDHE) 協商本次連接的對稱加密會話金鑰。
4. **隧道建立**：建立加密隧道，所有後續資料交換均經過加密處理。

## 3. 屬性/技術清單

| 項目         | 技術標準 / 說明                    | 類型     | 設定方式             |
| ------------ | ---------------------------------- | -------- | -------------------- |
| TLS 版本     | TLS 1.3 (不支援 1.2 以下)          | Config   | 固定 (系統內建)      |
| 加密演算法   | AES-256-GCM (資料) / RSA-4096 (金鑰)| Algorithm| 固定                 |
| 金鑰有效期   | 365 天 (自動更換)                  | Duration | 唯讀 / 自動更新      |
| 憑證狀態     | 有效 / 已過期 / 已吊銷            | Status   | 自動監控             |
| 通訊協議     | Secure WebSocket (WSS)             | Protocol | 固定                 |
| 心跳偵測頻率 | 30 秒                              | Interval | 可配置 (預設 30s)    |
