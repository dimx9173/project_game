## 功能說明

### 爭議對帳處理說明

**功能定義**：用於處理機台交易異常、或系統與遊戲商之間的交易爭議，包含建立爭議單、調查、裁決與補救。

#### 爭議單建立
1. 選擇爭議類型 (如：deposit_not_arrived, payout_error)。
2. 關聯原始交易編號。
3. 填寫問題描述與期望處理方式。

#### 調查與裁決
- **狀態追蹤**：待處理 -> 調查中 -> 裁決中 -> 結案。
- **補救措施**：餘額調整、補發交易、沖銷交易、派彩修正。

#### 補救操作邏輯
```javascript
async function executeRemedy(disputeId, authCode) {
  // 1. 驗證主管授權碼
  // 2. 根據裁決結果執行對應 API (如 /machines/:id/balance)
  // 3. 更新爭議單狀態為 closed
  // 4. 寫入稽核日誌
}
```

## 歷史記錄功能

- 完整保留所有爭議單處理歷程
- 可追蹤每個狀態的變更時間
- 記錄所有操作人員
- 保留相關通聯記錄

**歷史記錄欄位**：

| 欄位       | 說明           |
| ---------- | -------------- |
| 爭議單編號 | 爭議單唯一識別 |
| 機台編號   | 發生問題的機台 |
| 爭議類型   | 問題類型       |
| 建立時間   | 爭議單建立時間 |
| 處理時間   | 總處理時長     |
| 裁決結果   | 最終裁決       |
| 處理人     | 最後處理人員   |
| 結案時間   | 結案時間       |

**篩選與查詢**：

| 篩選條件 | 類型       | 說明         |
| -------- | ---------- | ------------ |
| 日期區間 | 日期picker | 建立日期範圍 |
| 爭議類型 | 多選       | 問題類型篩選 |
| 狀態     | 多選       | 爭議單狀態   |
| 緊急程度 | 多選       | 緊急程度     |
| 處理人   | 多選       | 負責人員     |
| 機台編號 | 文字       | 機台編號搜尋 |

**統計報表**：

| 指標         | 說明                   |
| ------------ | ---------------------- |
| 爭議總數     | 期間內建立的所有爭議單 |
| 爭議成立率   | 成立數 / 總數          |
| 平均處理時間 | 從建立到結案平均天數   |
| 熱門爭議類型 | 各類型數量排行         |
| 處理人績效   | 各處理人處理數量與時效 |

**匯出功能**：

- 支援匯出歷史記錄為 CSV 或 Excel
- 支援自訂欄位匯出

## 錯帳對帳管理機制

### 錯帳類型

| 類型               | 說明       |
| ------------------ | ---------- |
| missing_deposit    | 轉帳未到帳 |
| missing_withdrawal | 轉出未確認 |
| payout_error       | 派彩錯誤   |
| system_error       | 系統錯誤   |

### 處理流程

```
1. 偵測錯帳
   ├── 自動偵測 (對帳發現)
   └── 人工回報 (客戶投訴)

2. 調查原因
   ├── 檢查日誌
   ├── 聯繫遊戲商
   └── 調取交易紀錄

3. 處理方式
   ├── 補發點數
   ├── 退款
   └── 沖銷

4. 產生憑證
   ├── PDF 憑證
   ├── 寄送財務
   └── 歸檔
```

- 每日自動對帳 ( UTC 00:00 )
- 發現差異立即告警
- 支援人工對帳
