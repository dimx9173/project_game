## 3.8 遊戲商(串接遊戲)管理操作

### [C103] 新增遊戲商

**操作流程**：

```
1. 點擊「新增遊戲商」按鈕
2. 填寫遊戲商基本資料
3. 設定 API 連線參數
4. 設定金流路由
5. 測試連線
6. 完成設定
```

**輸入欄位**：

| 欄位名稱   | 類型 | 必填 | 說明             |
| ---------- | ---- | ---- | ---------------- |
| 遊戲商代碼 | 文字 | ✅   | 唯一識別碼       |
| 遊戲商名稱 | 文字 | ✅   | 完整名稱         |
| 遊戲類型   | 多選 | ✅   | 老虎機/捕魚/棋牌 |
| API 網址   | URL  | ✅   | API 端點         |
| API 金鑰   | 密碼 | ✅   | 驗證用           |
| 清算帳戶   | 文字 | ✅   | 對帳用帳戶       |
| 備註       | 文字 | -    | 選填             |

### API 與同步作業
- **[C54] API 連線狀態監控**：即時或定時檢測與第三方遊戲供應商 API 的連線延遲與健康度。
- **[C55] 遊戲同步管理**：主動或排程觸發向供應商端同步最新遊戲清單與設定。
- **[C56] API 日誌查詢**：留存與供應商之間的 Request/Response 詳細日誌，方便除錯與客訴追蹤。

### [C104] 餘額查詢與轉帳

**操作流程**：

```
1. 選擇遊戲商
2. 查詢當前餘額
3. 執行轉入/轉出
4. 確認交易
5. 記錄日誌
```

**轉帳類型**：

| 類型 | 說明                 |
| ---- | -------------------- |
| 轉入 | 從中央錢包轉入遊戲商 |
| 轉出 | 從遊戲商轉回中央錢包 |
| 對帳 | 同步餘額資料         |

---

### [C103] 移除遊戲商

**操作流程**：

```
1. 停用遊戲商連線
2. 完成所有待處理交易
3. 執行下架
4. 保留歷史資料
```
