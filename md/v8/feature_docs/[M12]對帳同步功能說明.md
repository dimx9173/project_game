## 3.11 錯帳對帳操作

### [L20] 錯帳類型

| 類型代碼           | 說明       |
| ------------------ | ---------- |
| missing_deposit    | 轉帳未到帳 |

### [L20] 錯帳處理流程

**操作流程**：

```plaintext
1. 偵測錯帳 (自動/人工)
2. 建立錯帳單
3. 調查原因
4. 選擇處理方式
5. 執行補發/退款/沖銷

7. 結案
```

**處理方式**：

| 方式 | 說明         |
| ---- | ------------ |
| 補發 | 重新執行交易 |
| 退款 | 退還款項     |
| 沖銷 | 反向沖銷     |

## 3.12 爭議對帳處理專區

### [L20] 功能說明

**什麼是爭議對帳處理專區？**

爭議對帳處理專區是專門處理交易爭議的區塊，當對機台交易結果有異議時（例如：充值未到帳、派彩錯誤、重複扣款等），管理員可以在此建立爭議單、进行调查、做出裁決並執行補救措施。

**爭議類型**：

| 類型代碼            | 說明       | 嚴重程度 |
| ------------------- | ---------- | -------- |
| deposit_not_arrived | 充值未到帳 | 高       |
| duplicate_charge    | 重複扣款   | 高       |
| payout_error        | 派彩錯誤   | 高       |
| withdrawal_failed   | 提現失敗   | 中       |
| balance_discrepancy | 餘額不符   | 中       |
| system_error        | 系統錯誤   | 高       |
| fraud_suspicion     | 疑似洗錢   | 高       |

### [L20] 爭議單建立

**操作流程**：

```plaintext
1. 進入「爭議對帳處理專區」
2. 點擊「建立爭議單」
3. 選擇爭議類型
4. 填寫爭議詳情
5. 關聯相關交易
6. 提交爭議單
```

**輸入欄位**：

| 欄位名稱     | 類型     | 必填         | 說明                    |
| ------------ | -------- | ------------ | ----------------------- |
| 爭議單編號   | 文字     | 系統自動產生 | 格式：DSP-YYYYMMDD-XXXX |
| 爭議類型     | 下拉選單 | ✅           | 選擇爭議類型            |
| 機台編號     | 文字     | ✅           | 發生問題的機台編號      |
| 關聯交易編號 | 文字     | ✅           | 有問題的交易編號        |
| 問題描述     | 文字     | ✅           | 詳細描述問題情況        |
| 期望處理方式 | 下拉選單 | ✅           | 補發/退款/沖銷/其他     |
| 緊急程度     | 下拉選單 | ✅           | 一般/緊急/立即處理      |
| 附件上傳     | 檔案     | -            | 相關截圖或證據          |
| 負責人       | 文字     | 系統自動帶入 | 建立人帳號              |

**緊急程度說明**：

| 等級     | 回覆時間 | 說明         |
| -------- | -------- | ------------ |
| 一般     | 24小時內 | 一般性問題   |
| 緊急     | 4小時內  | 影響遊戲體驗 |
| 立即處理 | 1小時內  | 嚴重影響營運 |

### [L20] 調查流程

**操作流程**：

```plaintext
1. 審視爭議單內容
2. 調閱相關交易紀錄
3. 查詢系統日誌
4. 比對多方資料
5. 撰寫調查報告
```

**調查項目**：

| 調查項目     | 說明             | 資料來源   |
| ------------ | ---------------- | ---------- |
| 交易紀錄     | 調閱完整交易流水 | 交易資料庫 |
| 現金交易紀錄 | 核對現金收付記錄 | 現金日誌   |
| 機台操作日誌 | 確認機台操作時間 | 系統日誌   |
| 機台交易紀錄 | 核對機台本地資料 | 本地資料庫 |

**調查狀態**：

| 狀態   | 說明           |
| ------ | -------------- |
| 待處理 | 新建立待處理   |
| 调查中 | 正在調查中     |
| 待覆核 | 調查完成待覆核 |
| 裁決中 | 正在做出裁決   |
| 處理中 | 正在執行補救   |
| 已結案 | 處理完成       |
| 已駁回 | 駁回爭議       |

### [L20] 裁決與補救

**裁決選項**：

| 裁決結果    | 說明                            |
| ----------- | ------------------------------- |
| 成立-補發   | 確認問題，補發款項              |
| 成立-退款   | 確認問題，退還款項              |
| 成立-沖銷   | 確認問題，執行沖銷              |
|             | 成立-其他確認問題，其他補救方式 |
| 不成立-駁回 | 駁回爭議申請                    |
| 證據不足    | 需要更多證據                    |

**補救操作**：

| 操作類型 | 說明             | 需主管授權 |
| -------- | ---------------- | ---------- |
| 餘額調整 | 直接調整機台餘額 | 是         |
| 補發交易 | 重新執行交易     | 是         |
| 沖銷交易 | 反向沖銷交易     | 是         |
| 派彩修正 | 修正派彩結果     | 是         |
| 優惠補償 | 發放優惠券/彩金  | 是         |

**裁決輸入欄位**：

| 欄位名稱 | 類型     | 必填 | 說明         |
| -------- | -------- | ---- | ------------ |
| 裁決結果 | 下拉選單 | ✅   | 選擇裁決結果 |
| 裁決理由 | 文字     | ✅   | 詳細裁決說明 |
| 補救金額 | 數字     | -    | 補救金額     |
| 補救方式 | 下拉選單 | -    | 選擇補救方式 |
| 授權碼   | 數字     | ✅   | 主管授權碼   |
| 備註     | 文字     | -    | 補充說明     |

## 3.17 對帳與同步中心操作

### [C93-C95/L34-L37] 對帳與同步中心總覽

**功能說明**：
對帳與同步中心是用於監控與管理集中式後台與單機本地後台之間資料流的統一介面。它整合了同步狀態監控、手動同步觸發、對帳檢查以及衝突處理，確保系統在離線恢復後的帳務與資料一致性。

**操作流程**：

```plaintext
1. 進入「對帳與同步中心」
2. 在「狀態總覽區」查看當前各節點連線與同步狀態
3. 如有需要，點擊「手動同步」立即上傳本地交易或下載中央更新
4. 系統自動進行對帳檢查，若有異常則顯示在「異常與衝突審核區」
5. 管理員針對衝突項目進行人工審核或沖銷處理
6. 在「同步日誌與佇列」檢視歷史同步紀錄與當前傳輸中資料
```

**介面區塊分配與功能說明**：

| 介面區塊                   | 整合項目                                                         | 說明                                                                                                                          |
| -------------------------- | ---------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **狀態總覽區**       | [L34] 同步狀態檢視``[C93] 手動同步``[C94] 對帳檢查 | 顯示整體連線狀態（如綠燈/紅燈）、最後同步時間、對帳結果與佇列筆數。以圖表或儀表板形式呈現，並提供明顯的「立刻手動同步」按鈕。 |
| **異常與衝突審核區** | [L37] 衝突處理與人工審核                                         | 條列式列出帳單不一致或版本衝突的項目。無異常時顯示「目前無異常」。提供相關處理策略選項（如以時間戳為準或進行人工覆寫沖銷）。  |
| **同步日誌與佇列**   | [C95] 同步日誌``[L35] 雙向佇列與交易補償                  | 歷史同步活動紀錄表（成功/失敗），與目前卡在本地未完成上傳的交易佇列詳情，便於 IT/管理員追蹤。                                 |

> **說明**：[L36] 指數退避重試機制 與 [L42] 底層通訊加密屬自動執行的後端操作，在日常 UI 中不需專屬專區呈現，若有設定需求，已統一分類進 **[C88] 網路設定** 或 **[C90] 系統參數設定**。
