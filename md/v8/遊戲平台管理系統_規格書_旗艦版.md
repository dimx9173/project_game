## 1. 文件概述

本文件詳細闡述遊戲平台管理系統的完整軟體需求，包含集中式後台與單機本地後台兩個層次的功能需求、非功能需求、使用案例、資料模型等內容。

### 1.1 文件目的

本文件旨在為開發、測試、部署人員提供清晰的軟體需求指導，確保系統開發符合業務需求。

## 2. 系統概述

### 2.1 系統架構

> **中央集權架構**：單機本地後台為雲端快取端點，資料由集中式統一管理，本地無法直接修改。

遊戲平台採用**中央集權架構**設計。

- **集中式後台**：負責全局資料監控、**機台註冊認證**、全平台交易統計、**所有資料統一管理**等功能。
- **單機本地後台**：為雲端快取端點，**僅能查看快取資料**，無法直接修改。交易資料先存入本地佇列，連線時上傳至集中式處理。
- **必須連線**：單機本地後台**必須連線才能開始遊戲**，離線後僅能查看快取資料（不支援匿名下注）。

### 2.2 系統範圍

本系統涵蓋遊戲機台的集中式管理與單機本地後台兩個層次，支援遊戲、機台、交易、串接、多人遊戲、使用者權限、版本更新、介面設定等完整功能模組。

### 2.3 識別碼生成規則

#### 2.3.1 機台 instanceId 生成

> **設計原則**：機台必須向集中式註冊才能啟用，由集中式產生  並綁定 MAC。

| 項目 | 說明 |
| ---- | ---- |
| **生成方式** | 集中式統一產生（UUID v4） |
| **綁定機制** | instanceId 與 MAC 位址綁定 |
| **唯一性** | 全平台唯一，確保機台識別 |
| **註冊流程** | 首次啟用時向集中式註冊，取得 instanceId |

### 2.5 技術架構

本系統採用 **C# .NET 8** 技術棧：

- **前端**：Angular 17+ + Angular Material + Bootstrap 5
- **後端**：ASP.NET Core 8 Web API
- **集中式資料庫**：PostgreSQL 15+
- **本地資料庫**：SQLite
- **通訊**：HTTP/HTTPS + WebSocket

## 3. 集中式後台功能需求

### 3.1 儀表板模組

實時顯示全平台的關鍵 KPI，提供高度客製化的 Widget 儀表板面板，可自訂圓餅圖、長條圖與熱門遊戲排行分佈。

### 3.2 深度機台管理與圖表監控

遠端機台管理。支援機台以 1 分鐘為單位的即時心跳、CPU 與記憶體佔用的資源監測。包含遠端指令發送、極嚴格的分類與分區檢視。

### 3.3 遊戲庫與版本管理模組

全局遊戲庫管理。支援遊戲上下架、分類推薦標籤、控制第三方遊戲的供應商開關，以及全台軟體版本版號維護。

### 3.4 錢包交易與多維度報表模組

全平台營收與抽水報表。具備深度的多維交叉篩選器（Filter），可拉出日/週/月/年，甚至依據多帳號與機台的立體營運曲線與交叉比對圖表。

#### 3.4.1 功能說明

- **報表總覽與趨勢圖**
  - 提供即時的營收總覽看板（今日營收、總下注、總派彩、公司盈利）。
  - 支援折線圖與長條圖呈現營收趨勢。
- **多維度營收查詢**
  - 支援依據：**時間帶**（自訂區間）、**店家/代理商**、**機台編號**、**機台標籤**、**遊戲商**、**單一遊戲**等多維度交叉篩選。
  - 支援標籤多選篩選（AND 邏輯），例如：篩選「台北門市」且「VIP專用」的機台。
  - 提供詳細的數據表列：總投幣、總開分、總押注、總得分、總洗分。
- **報表匯出功能**
  - 支援將查詢結果匯出為 CSV 或 Excel 檔案，供財務或稽核人員存查。

#### 3.5.2 業務規則

| 規則編號 | 規則說明                            |
| -------- | ----------------------------------- |
| REP-001  | 營收報表數據最少需保留 5 年以上紀錄 |
| REP-002  | 報表查詢時間區間最大限制為 1 年     |
| REP-003  | 僅具有「財務」或「最高管理員」權限者可存取與匯出 |

### 3.5 串接遊戲管理模組（根據遊戲廠商API進行動態調整）

遊戲商 API 串接管理。支援遊戲商列表檢視、API 連線狀態監控、遊戲同步管理、交易對帳、API 日誌查詢等功能。

#### 3.5.3 功能說明

- **遊戲商列表管理**

  - 支援新增、編輯、刪除遊戲商資料
  - 顯示遊戲商名稱、API 端點、連線狀態、金鑰指紋
  - 支援遊戲商分類（電子、棋牌、捕魚、彩票等）
- **API 連線監控**

  - 即時顯示各遊戲商的 API 連線狀態（線上/離線/異常）
  - 心跳檢測間隔：30 秒
  - 連續 3 次心跳失敗標記為離線
- **遊戲同步管理**

  - 自動同步遊戲商提供的遊戲列表
  - 支援遊戲名稱、類型、 RTP 、ICON URL 等資訊同步
  - 手動觸發立即同步功能
- **交易對帳**

  - 每日定時對帳（可配置）
  - 比對本地交易記錄與遊戲商回傳資料
  - 標記差異項目並產出對帳報告
- **API 日誌查詢**

  - 記錄所有 API 請求與回應
  - 支援依時間、遊戲商、遊戲、狀態篩選
  - 日誌保留期限：90 天

### 3.6 多人遊戲模組 （根據遊戲廠商API進行動態調整）

多人遊戲（如百家樂、龍虎、骰寶）的集中式管理。支援牌桌列表檢視、牌桌狀態監控、牌桌配置管理、牌局歷史查詢、荷官操作紀錄等功能。

#### 3.6.1 功能說明

- **牌桌列表檢視**

  - 顯示所有多人遊戲牌桌的基本資訊
  - 支援遊戲類型（百家樂、龍虎、骰寶）、牌桌狀態、線上機台數篩選
  - 即時更新牌桌狀態（空閒/進行中/已關閉）
- **牌桌狀態監控**

  - 即時顯示各牌桌的遊戲進度
  - 顯示當前牌靴餘牌數、下注金額、參與機台數
  - 異常狀態告警（連續錯誤、異常下注）
- **牌桌配置管理**

  - 牌桌基本設定：最低/最高投注額、開放時間、限流機台數
  - 莊家抽水比例設定
  - 牌靴設定：使用副數、每副牌張數、換靴規則
- **牌局歷史查詢**

  - 查詢歷史牌局記錄
  - 顯示每局結果（莊/閒/和）、點數、派彩結果
  - 支援依日期、牌桌、遊戲類型篩選
  - 匯出功能（CSV/Excel）
- **荷官操作紀錄**

  - 記錄荷官的開牌、結算、補正等操作
  - 支援依日期、荷官、牌桌篩選
  - 異常操作標記與稽核

| MP-002   | 牌局歷史保留期限：1 年           |

### 3.7 靜態與動態 RBAC 模組

最高權限之動態角色自定義引擎。業主可手動勾選高達上百個細部 API 權限，指派給特定的「審計員」、「查帳會計」或「維修外包」。

### 3.8 全球 OTA 無感熱更新發布中心

包含腳本上傳驗證、MD5 雜湊審查、排程更新（預計深夜）、與失敗自動還原機制。機台端不受干擾的背景靜默熱更新。

### 3.9 爭議對帳處理專區

專為處理網路斷線、封包遺失、人工現場補發獎勵的核銷特區。提供雙面對帳與會計差異除錯，嚴格控管每一筆人工干預的資金流向。

### 3.10 前端版面廣播模組

機台端前台版面配置中心。支援遠端推播大廳排版與活動橫幅 (Banner)。

## 4. 單機本地後台功能需求

### 4.1 儀表板模組

顯示本機的關鍵 KPI（今日營收、今日下注額、今日派彩額、淨收益、線上連線數）。提供機台基本資訊、硬體狀態、網路狀態、快速操作按鈕等資訊。

### 4.2 機台管理模組

本地機台管理。支援機台資訊檢視、硬體狀態監控、開分、洗分、重啟、關機、進入測試模式、機台設定等功能。

### 4.3 遊戲管理模組

本地遊戲庫管理。支援遊戲列表檢視、遊戲啟用/停用、遊戲配置、遊戲測試等功能。

### 4.4 交易管理模組

本地交易紀錄管理。支援交易列表檢視（多維度篩選）、交易詳情檢視、交易統計（日/週/月）、交易操作（回滾/確認/導出）等功能。

### 4.5 統計報表模組

本地統計資料收集與呈現。支援日/週/月報表、遊戲排行、機台營收排行等多維度統計分析。

### 4.6 系統設定模組

單機本地後台配置管理。支援基本設定、遊戲大廳設定、安全設定、備份與恢復等功能。

#### 4.6.1 功能說明

##### 基本設定

- **機台名稱**：設定本地機台的顯示名稱
- **機台編號**：唯讀，系統自動產生之 UUID
- **時區設定**：設定本地時區，預設為 UTC+8
- **語言設定**：支援繁體中文、簡體中文、英文
- **日期格式**：YYYY-MM-DD / DD/MM/YYYY / MM-DD-YYYY
- **時間格式**：12小時制 / 24小時制


##### 遊戲大廳設定

- **大廳標題**：機台端顯示的標題名稱
- **大廳背景圖**：背景圖片 URL 或本地路徑
- **遊戲分類**：定義遊戲分類與顯示順序
- **熱門遊戲**：設定顯示於首頁的熱門遊戲
- **最新遊戲**：設定顯示於首頁的最新遊戲
- **公告內容**：跑馬燈與首頁公告內容

##### 安全設定

**管理員登入安全**
- **密碼最小長度**：8 位元以上
- **密碼複雜度要求**：需包含大寫字母、小寫字母、數字（可選特殊符號）
- **登入驗證**：集中式後台登入需通過 Email 安全驗證碼 (6 位數)
- **登入失敗鎖定**：失敗 5 次自動鎖定帳號
- **密碼重置**：
  - 集中式後台：由超級管理員重設或透過註冊 Email 重置
  - 本地後台：由集中式後台管理員重設 PIN 碼


**風控規則**
- **單筆最高存款**：可設定金額上限
- **單筆最高提款**：可設定金額上限
- **單日最高提款次數**：可設定次數上限
- **異常行為偵測**：可選啟用
- **大額交易自動審核**：可選啟用

**API 安全**
- **API Key 前綴**：可自訂（例：pk_live_）
- **API 請求速率限制**：可設定次/分鐘
- **API 請求日誌**：可選啟用
- **允許的 HTTP 方法**：GET / POST / PUT / DELETE
- **CORS 允許來源**：可設定多個網域

##### 備份與恢復

- **自動備份**：每日自動備份資料庫
- **備份保留天數**：保留備份的天數，預設 7 天
- **手動備份**：立即建立備份
- **還原**：從備份檔案還原系統
- **匯出**：匯出資料為 JSON/CSV 格式
- **匯入**：從 JSON/CSV 檔案匯入資料

#### 4.8.3 雲端遙控機制

> **中央集權架構**：單機本地後台為雲端快取端點，資料由集中式統一管理，本地無法直接修改。

集中管理可對單機本地後台進行**雲端遙控**，本地端僅能查看快取資料，無法直接修改。

##### 功能說明

- **雲端遙控**：集中管理可遠端控制本地機台（開分/洗分/重啟等），需本地在線
- **操作回饋**：遠端送出指令後需收到結果回應，才會存至集中DB
- **操作日誌**：所有遠端操作均記錄至使用者操作日誌
- **本地快取下載**：本地僅能下載雲端快取，無法直接寫入資料表
- **交易佇列上傳**：本地交易先存入佇列，連線時上傳至雲端

##### 運作流程

1. **集中端發起**：管理員在集中式後台發起遠端操作請求
2. **本地驗證**：單機本地後台驗證在線狀態，回傳可執行狀態
3. **執行操作**：單機本地後台執行操作並記錄操作日誌
4. **回傳結果**：單機本地後台回傳執行結果（成功/失敗）
5. **儲存集中**：集中管理儲存結果至集中DB，記錄完整操作日誌

##### 適用範圍

> **本地端權限**：⚠️ 唯讀 = 雲端快取 | ❌ 不支援 = 雲端統一管理

| 功能模組 | 本地操作 | 雲端遙控 | 說明 |
| -------- | :------: | :------: | ---- |
| 機台管理 | ⚠️ 唯讀 | ✅ | 開分/洗分/重啟由雲端遙控 |
| 遊戲管理 | ⚠️ 唯讀 | ❌ | 雲端統一管理 |
| 交易紀錄 | 📝 佇列 | - | 本地暫存→上傳雲端 |
| 系統設定 | ⚙️ 局部 | - | 僅基本參數 |
| 介面設定 | ⚠️ 唯讀 | ❌ | 雲端統一管理 |
| 監控中心 | ⚠️ 唯讀 | - | 僅查看 |
| 遊戲供應商 | ❌ | ❌ | 集中式統一管理 |
| 串接遊戲 | ❌ | ❌ | 集中式統一管理 |
| 多人遊戲 | ❌ | ❌ | 集中式統一管理 |

##### 業務規則

| 規則編號 | 規則說明 |
| -------- | -------- |
| RR-001 | 單機本地後台需在線才能接受雲端遙控 |
| RR-002 | 每次雲端遙控需記錄操作者、操作時間、操作內容、結果 |
| RR-003 | 雲端遙控失敗需回傳錯誤訊息供集中端記錄 |
| RR-004 | 本地端資料為雲端快取，所有修改由雲端統一管理 |
| RR-005 | 遊戲供應商、串接遊戲、多人遊戲由集中式統一管理 |

### 4.8 底層通訊與金鑰加密模組

單機本地網路防綁架設定。支援中央通訊的動態憑證核發、雙面驗證 (Mutual TLS)、與硬體特徵綁定的安全檢測。

### 4.9 雙向佇列與交易補償機制

> **中央集權架構**：本地端僅能上傳交易資料，無法直接修改資料表。

支援指數退避演算法 (Exponential Backoff) 與補償機制。採用**雲端統一處理原則**：

- **雲端管理原則**：所有資料修改由集中式統一管理，本地僅做快取
- **交易佇列**：本地交易先存入佇列，連線時上傳至雲端處理
- **自動解決**：重複交易（UUID 去重）
- **人工介入**：餘額衝突、帳務異常 → 進入「爭議對帳處理專區」

⚠️ **核心原則**：**絕不憑空銷毀任何記帳斷層**，所有帳務衝突保留本地記錄，由人工裁決。

### 4.10 監控與日誌模組

系統狀態監控與日誌管理。支援硬體監控、軟體監控、日誌查詢等功能。

## 5. 非功能需求

### 5.1 效能需求

- **集中式後台**：支援 1000+ 台機台的集中管理，API 響應時間 < 200ms，資料庫查詢時間 < 500ms。
- **單機本地後台**：本地操作響應時間 < 100ms，資料庫查詢時間 < 200ms。

### 5.2 可用性需求

- **集中式後台**：系統可用性 ≥ 99.5%，支援高可用部署。
- **單機本地後台**：本地端為雲端快取端點，**離線時僅能查看快取資料，無法開始新遊戲（因交易需即時上傳）**。連線恢復時自動同步資料。

### 5.3 企業級防護與稽核需求

實施極端嚴格的資安標準（AES-256-GCM + RSA）、交易簽名驗證（HMAC-SHA256）。每個單一調用包含完整的 HTTP Request, IP, 裝置特徵等記錄做為稽核依據。

### 5.4 可服務性與自動擴充架構

系統架構支援水平擴展，資料庫支援分片，API 支援版本管理，支援第三方遊戲商接入。

### 5.5 相容性需求

- **集中式後台**：支援 Chrome、Firefox、Safari、Edge 等主流瀏覽器。
- **單機本地後台**：支援 Windows、Linux、macOS 等作業系統。

## 7. 使用案例

### 6.1 機台管理員的使用案例

> **中央集權架構**：單機本地後台為雲端快取端點，無法直接修改資料。

機台管理員登入單機本地後台，**僅能查看快取資料**。開分、洗分、重啟等操作由集中式遙控執行。查詢本機的交易紀錄與統計報表（來自雲端快取）。在連線時自動同步資料到集中式後台。

### 6.2 平台管理員的使用案例

平台管理員登入集中式後台，監控全平台的營運資料。管理所有遊戲機台、遊戲。下發遊戲更新、配置更新到各機台。查詢全平台的交易統計與對帳資訊。

### 6.3 財務人員的使用案例

財務人員登入集中式後台，查詢全平台的交易紀錄與對帳資訊。生成日/週/月的財務報表。進行交易對帳與異常交易處理。

### 6.4 遊戲商的使用案例

遊戲商通過 API 接入平台，提供遊戲與交易資料。平台通過串接遊戲管理模組進行 API 連線監控與交易對帳。

### 6.5 高階硬體的機台監控

駐點工程師登入單機後台，看見 CPU 因為更新下載突然飆漲到了 90%，但他知道這是背景 OTA 的正常現象，並安心地讓系統執行熱更新而不需要讓機台停機。

### 6.6 動態會計審核的情境

財務人員登入集中式後台的 `[爭議對帳處理專區]`，系統自動標記了一筆因昨天雷擊重啟導致「中央與單機不對等」的 1500 分交易。會計人員按下「承認單機數據」後，系統補上人工核銷戳記，結束爭議。

### 6.7 超級擴充第三方行情

工程師接到指令掛載第 4 間國外的捕魚機公司（超越原始的 3 家預設）。因此他不用修改 V8 主架構與核心流水帳邏輯，只要實作標準的遊戲商 API 介面，即刻就讓所有台灣的機台都能跑國外的 API 遊戲。
