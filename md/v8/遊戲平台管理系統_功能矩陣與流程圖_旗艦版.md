# 1. 功能需求矩陣

> **📌 遊戲管理模組關係說明**
>
> | 模組                         | 說明                                                | 比喻       |
> | ---------------------------- | --------------------------------------------------- | ---------- |
> | **遊戲管理** (C12-C17) | 管理「遊戲本身」- 哪些遊戲可以玩、上架/下架         | 餐廳菜單   |
> | **串接遊戲** (C29-C32) | 管理「供應商 API」- 連接到供應商、可能單人+多人遊戲 | 餐廳供應商 |
> | **多人遊戲** (C33-C43) | 管理「真人牌桌」- 百家樂/龍虎/骰寶 牌桌管理         | 餐廳座位   |
>
> - 串接遊戲管理連接到供應商後，供應商可能提供「單人遊戲」或「多人遊戲」
> - 多人遊戲的遊戲來自外部供應商（非系統自行決定）

### 2.5.1 本地被集中接管功能

本地後台可離線運作，連線後自動同步資料到集中式後台。以下為功能歸屬說明：

#### 中央可接管的功能

| 類別 | 功能 | 說明 |
|------|------|------|
| **機台管理** | 遠端開分/洗分 | 中央直接操控機台 |
| | 遠端重啟 | 中央發送指令 |
| | 機台配置下發 | 中央推播設定到機台 |
| **遊戲管理** | 遊戲列表同步 | 中央統一管理遊戲 |
| | 遊戲啟用/停用 | 中央遙控 |
| **交易管理** | 交易資料上傳 | 離線交易自動同步 |
| | 餘額同步 | 中央與本地餘額對帳 |
| **版本更新** | OTA 遠端更新 | 中央派發更新到機台 |

#### 本地仍可設定的功能（系統設定）

| 類別 | 功能 | 說明 |
|------|------|------|
| **基本設定** | 機台名稱 | 本地識別用 |
| **網路設定** | 連線參數 | 本地網路配置 |
| **遊戲大廳設定** | 本地顯示配置 | 大廳本地顯示設定 |
| **安全設定** | PIN 碼 | 本地開分權限 |
| | 金鑰加密 | 本地通訊加密 |
| **備份與恢復** | 本地資料備份 | 本地資料保護 |

#### 同步類型與流程

```
┌─────────────────────────────────────────────────────────────┐
│                    連線建立時                                │
├─────────────────────────────────────────────────────────────┤
│  1. 檢查離線期間本地變更                                   │
│     ↓                                                      │
│  2. 上傳本地交易紀錄到中央                                  │
│     ↓                                                      │
│  3. 下載中央的遊戲更新/設定更新                             │
│     ↓                                                      │
│  4. 驗證資料一致性                                         │
│     ↓                                                      │
│  5. 同步完成                                               │
└─────────────────────────────────────────────────────────────┘
```

| 同步類型 | 觸發時機 | 說明 |
|----------|----------|------|
| **自動同步** | 連線建立時 | 背景定時同步 |
| **手動同步** | 管理員觸發 | 立即同步 |
| **強制同步** | 衝突時 | 忽略衝突強制同步 |

#### 離線運作能力

本地後台可**離線運作**，網路恢復後自動同步，確保營運不中斷。離線期間的交易會暫存於本地佇列，連線後自動上傳。

#### 3.5.4.3 權限制定原則

```
┌─────────────────────────────────────────────────────┐
│              權限制定流程                             │
├─────────────────────────────────────────────────────┤
│  1. 確認帳號用途                                     │
│     └── 內部人員 → 使用者權限模組                     │
│                                                     │
│  2. 選擇角色                                        │
│     └── 根據工作職責選擇預設角色                      │
│                                                     │
│  3. 客製化調整                                       │
│     └── 依需求增減特定權限                           │
│                                                     │
│  4. 審核確認                                        │
│     └── 主管審核後啟用                               │
└─────────────────────────────────────────────────────┘
```

**核心原則**：
- **最小權限** - 只給完成工作需要的最低權限
- **職責分離** - 稽核與操作分開
- **記錄所有** - 所有權限變更都留日誌

## 3.14 PaymentRouter 抽象層

### 3.14.1 功能說明

**什麼是 PaymentRouter？**

PaymentRouter 是系統的支付通道抽象層，負責統一管理多個支付通道（銀行卡、虛擬貨幣、第三方支付、錢包等），提供支付路由、通道切換、金額分流等功能。它就像一個智慧型的支付調度中心，能根據通道的可用性、費率、額度等因素，自動或手動選擇最適合的支付路徑。

**核心功能**：

- 多通道統一管理
- 智慧路由選路
- 通道可用性監控
- 金額分流與限額控制
- 故障自動切換

### 3.14.2 支付通道設定

**操作流程**：

```
1. 進入「PaymentRouter」模組
2. 點擊「新增通道」
3. 填寫通道參數
4. 設定通道權重
5. 測試連線
6. 啟用通道
```

**輸入欄位**：

| 欄位名稱 | 類型 | 必填 | 說明 |
| -------- | ---- | ---- | ---- |
| 通道代碼 | 文字 | ✅ | 唯一識別碼，如：bank_icbc |
| 通道名稱 | 文字 | ✅ | 顯示名稱，如：工商銀行 |
| 通道類型 | 下拉選單 | ✅ | 銀行/虛擬貨幣/第三方支付/錢包 |
| API 端點 | URL | ✅ | 支付閘道網址 |
| API 金鑰 | 密碼 | ✅ | 驗證用金鑰 |
| 商戶號 | 文字 | ✅ | 商戶識別號 |
| 手續費率 | 百分比 | ✅ | 交易手續費率（%） |
| 單筆最低限額 | 數字 | ✅ | 最低交易金額 |
| 單筆最高限額 | 數字 | ✅ | 最高交易金額 |
| 單日限額 | 數字 | ✅ | 單日累計限額 |
| 狀態 | 開關 | ✅ | 啟用/停用 |

**通道類型說明**：

| 類型 | 代碼 | 適用場景 |
| ---- | ---- | -------- |
| 銀行卡 | bank | 大額充值/兌換 |
| 虛擬貨幣 | crypto | 數位貨幣交易 |
| 第三方支付 | third_party | 支付寶/Line Pay等 |
| 錢包餘額 | wallet | 帳戶餘額交易 |

### 3.14.3 路由規則設定

**操作流程**：

```
1. 進入「路由規則」
2. 點擊「新增規則」
3. 設定規則條件
4. 設定目標通道
5. 設定規則優先級
6. 儲存規則
```

**規則類型**：

| 類型 | 說明 | 範例 |
| ---- | ---- | ---- |
| 金額範圍 | 根據金額選擇通道 | 0-1000用通道A，1000以上用通道B |
| 通道可用性 | 通道狀態自動切換 | 主通道失敗時切換到備用通道 |
| 負載平衡 | 流量分散到多個通道 | 多通道輪詢或權重分配 |
| 費率優先 | 選擇最低費率通道 | 自動選擇手續費最低的通道 |
| 白名單 | 特定玩家只能用特定通道 | VIP玩家專用通道 |

**規則條件欄位**：

| 欄位名稱 | 類型 | 說明 |
| -------- | ---- | ---- |
| 規則名稱 | 文字 | 規則識別名稱 |
| 優先級 | 數字 | 數字越小越優先（1-100） |
| 適用交易類型 | 多選 | 充值/兌換 |
| 金額範圍 | 區間 | 最小~最大金額 |
| 玩家等級 | 多選 | 適用的VIP等級 |
| 通道可用性 | 條件 | 通道狀態條件 |
| 目標通道 | 多選 | 符合條件時使用的通道 |
| 權重分配 | 數字 | 流量分配比例 |

**路由邏輯流程**：

```
┌─────────────────────────────────────────┐
│           收到支付請求                   │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│     規則1：金額範圍匹配？                │
│     是 → 選擇目標通道                   │
│     否 → 繼續規則2                      │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│     規則2：玩家等級匹配？                │
│     是 → 選擇目標通道                   │
│     否 → 繼續規則3                      │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│     規則3：通道可用性檢查               │
│     可用 → 執行交易                    │
│     不可用 → 自動切換備用通道           │
└─────────────────────────────────────────┘
```

### 3.14.4 通道切換邏輯

**切換類型**：

| 類型 | 觸發條件 | 行為 |
| ---- | -------- | ---- |
| 手動切換 | 管理員手動操作 | 立即切換到指定通道 |
| 自動切換 | 通道異常、超時 | 自動切換到備用通道 |
| 負載切換 | 通道流量達標 | 自動分流到其他通道 |
| 費率切換 | 費率變動 | 切換到更低費率通道 |

**故障檢測項目**：

| 檢測項目 | 閾值 | 動作 |
| -------- | ---- | ---- |
| API 回應時間 | >10秒 | 標記為響應慢 |
| 連續失敗次數 | >3次 | 標記為不可用 |
| 錯誤率 | >10% | 觸發警告 |
| 通道餘額 | <最低餘額 | 觸發充值提醒 |

**自動切換流程**：

```
1. 偵測到通道異常
2. 記錄異常事件
3. 發送告警通知
4. 自動切換到備用通道
5. 記錄切換日誌
6. 持續監控原通道
7. 原通道恢復後切回
```

## 3.15 爭議對帳處理專區

### 3.15.1 功能說明

**什麼是爭議對帳處理專區？**

爭議對帳處理專區是專門處理交易爭議的區塊，當玩家對交易結果有異議時（例如：充值未到帳、派彩錯誤、重複扣款等），管理員可以在此建立爭議單、进行调查、做出裁決並執行補救措施。

**爭議類型**：

| 類型代碼 | 說明 | 嚴重程度 |
| -------- | ---- | -------- |
| deposit_not_arrived | 充值未到帳 | 高 |
| duplicate_charge | 重複扣款 | 高 |
| payout_error | 派彩錯誤 | 高 |
| withdrawal_failed | 提現失敗 | 中 |
| balance_discrepancy | 餘額不符 | 中 |
| system_error | 系統錯誤 | 高 |
| fraud_suspicion | 疑似洗錢 | 高 |

### 3.14.2 爭議單建立

**操作流程**：

```
1. 進入「爭議對帳處理專區」
```
