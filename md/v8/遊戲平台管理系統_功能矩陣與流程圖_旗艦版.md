# 1. 功能需求矩陣

> **📌 遊戲管理模組關係說明**
>
> | 模組                         | 說明                                                | 比喻       |
> | ---------------------------- | --------------------------------------------------- | ---------- |
> | **遊戲管理** (C16-C17) | 管理「遊戲本身」- 哪些遊戲可以玩、上架/下架         | 餐廳菜單   |
> | **串接遊戲** (C53-C56) | 管理「供應商 API」- 連接到供應商、單人/多人遊戲 | 餐廳供應商 |
> | **多人遊戲** (C57-C67) | 管理「真人牌桌」- 百家樂/龍虎/骰寶 牌桌管理  | 餐廳座位   |
>
> ⚠️ **多人遊戲斷線保護**：必須保持連線方可進行遊戲交易，不支援離線下注。
>
> - 串接遊戲管理連接到供應商後，供應商可能提供「單人遊戲」或「多人遊戲」
> - 多人遊戲的遊戲來自外部供應商（非系統自行決定）

## 1.1 系統功能需求矩陣（集中式 vs 本地）

> **說明**：本地端（單機本地後台）為雲端快取端點，資料由集中式統一管理，本地無法直接修改。所有本地操作透過雲端 API 執行，離線時僅能查看快取資料，無法開始新遊戲。
>
> **本地端權限**：
>
> - ⚠️ 唯讀：資料來自雲端快取，僅供檢視
> - 📝 佇列：本地暫存，連線時上傳至雲端
> - ⚙️ 局部：僅本地網路參數等基本設定

### 2.5.1 本地被集中接管功能

本地後台可離線運作，連線後自動同步資料到集中式後台。以下為功能歸屬說明：

#### 中央可接管的功能

| 類別               | 功能          | 說明               |
| ------------------ | ------------- | ------------------ |
| **機台管理** | 遠端開分/洗分 | 中央直接操控機台   |
|                    | 遠端重啟      | 中央發送指令       |
|                    | 機台配置下發  | 中央推播設定到機台 |
| **遊戲管理** | 遊戲列表同步  | 中央統一管理遊戲   |
|                    | [C21] 遊戲啟用/停用 | 中央遙控           |
| **交易管理** | 交易資料上傳  | 離線交易自動同步   |
|                    | 餘額同步      | 中央與本地餘額對帳 |
| **版本更新** | OTA 遠端更新  | 中央派發更新到機台 |

#### 本地仍可設定的功能（系統設定）

| 類別                   | 功能         | 說明             |
| ---------------------- | ------------ | ---------------- |
| **基本設定**     | 機台名稱     | 本地識別用       |
| **網路設定**     | 連線參數     | 本地網路配置     |
| **遊戲大廳設定** | 本地顯示配置 | 大廳本地顯示設定 |
| **安全設定**     | PIN 碼       | 本地開分權限     |
|                        | 金鑰加密     | 本地通訊加密     |
| **備份與恢復**   | 本地資料備份 | 本地資料保護     |

#### 同步類型與流程

```
┌─────────────────────────────────────────────────────────────┐
│                    連線建立時                                │
├─────────────────────────────────────────────────────────────┤
│  1. 檢查離線期間本地變更                                   │
│     ↓                                                      │
│  2. 上傳本地交易紀錄到中央                                  │
│     ↓                                                      │
│  3. 下載中央的遊戲更新/設定更新                             │
│     ↓                                                      │
│  4. 驗證資料一致性                                         │
│     ↓                                                      │
│  5. 同步完成                                               │
└─────────────────────────────────────────────────────────────┘
```

| 同步類型           | 觸發時機   | 說明             |
| ------------------ | ---------- | ---------------- |
| **自動同步** | 連線建立時 | 背景定時同步     |
| **手動同步** | 管理員觸發 | 立即同步         |
| **強制同步** | 衝突時     | 忽略衝突強制同步 |

#### 離線運作能力

本地後台可**離線運作**，網路恢復後自動同步，確保營運不中斷。離線期間的交易會暫存於本地佇列，連線後自動上傳。

#### 3.4.4.3 權限制定原則

```
┌─────────────────────────────────────────────────────┐
│              權限制定流程                             │
├─────────────────────────────────────────────────────┤
│  1. 確認帳號用途                                     │
│     └── 內部人員 → 使用者權限模組                     │
│                                                     │
│  2. 選擇角色                                        │
│     └── 根據工作職責選擇預設角色                      │
│                                                     │
│  3. 客製化調整                                       │
│     └── 依需求增減特定權限                           │
│                                                     │
│  4. 審核確認                                        │
│     └── 主管審核後啟用                               │
└─────────────────────────────────────────────────────┘
```

**核心原則**：

- **最小權限** - 只給完成工作需要的最低權限
- **職責分離** - 稽核與操作分開
- **記錄所有** - 所有權限變更都留日誌

## 3.13 爭議對帳處理專區

### [L20] 功能說明

**什麼是爭議對帳處理專區？**

爭議對帳處理專區是專門處理交易爭議的區塊，當玩家對交易結果有異議時（例如：充值未到帳、派彩錯誤、重複扣款等），管理員可以在此建立爭議單、进行调查、做出裁決並執行補救措施。

**爭議類型**：

| 類型代碼            | 說明       | 嚴重程度 |
| ------------------- | ---------- | -------- |
| deposit_not_arrived | 充值未到帳 | 高       |
| duplicate_charge    | 重複扣款   | 高       |
| payout_error        | 派彩錯誤   | 高       |
| withdrawal_failed   | 提現失敗   | 中       |
| balance_discrepancy | 餘額不符   | 中       |
| system_error        | 系統錯誤   | 高       |
| fraud_suspicion     | 疑似洗錢   | 高       |

### [L20] 爭議單建立

**操作流程**：

```
1. 進入「爭議對帳處理專區」
2. 點擊「建立爭議單」
3. 選擇爭議類型
4. 填寫爭議詳情
5. 關聯相關交易
6. 提交爭議單
```

**輸入欄位**：

| 欄位名稱     | 類型     | 必填         | 說明                    |

| 爭議類型     | 下拉選單 | ✅           | 選擇爭議類型            |
| 玩家帳號     | 文字     | ✅           | 發生問題的玩家帳號      |
