# 1. 功能需求矩陣

> **📌 遊戲管理模組關係說明**
>
> | 模組                         | 說明                                            | 比喻       |
> | ---------------------------- | ----------------------------------------------- | ---------- |
> | **遊戲管理** (C16-C17) | 管理「遊戲本身」- 哪些遊戲可以玩、上架/下架     | 餐廳菜單   |
> | **串接遊戲** (C53-C56) | 管理「供應商 API」- 連接到供應商、單人/多人遊戲 | 餐廳供應商 |
> | **多人遊戲** (C57-C67) | 管理「真人牌桌」- 百家樂/龍虎/骰寶 牌桌管理     | 餐廳座位   |
>
> ⚠️ **多人遊戲斷線保護**：必須保持連線方可進行遊戲交易，不支援離線下注。
>
> - 串接遊戲管理連接到供應商後，供應商可能提供「單人遊戲」或「多人遊戲」
> - 多人遊戲的遊戲來自外部供應商（非系統自行決定）

本章節匯整系統核心業務的運作流程，分為「交易與遊戲」、「機台管理與註冊維護」、「系統同步與維護」三大類。所有圖表統一使用 Mermaid 依「集中式後台 (Central)」與「單機本地後台 (Local)」之互動繪製。

## 2.1 交易與遊戲流程

### 2.1.1 機台開分與洗分流程 (充值與兌換) [L45 / L46]

整合現場管理員透過本地機台申請，並交由集中式後台處理風控與金流串接的完整流程。

```mermaid
sequenceDiagram
    autonumber
    participant Player as 玩家
    participant Admin as 現場值班人員
    participant Local as 單機本地後台
    participant Central as 集中式後台
    participant GameAPI as 遊戲 API

    Note over Player, GameAPI: 【開分流程 (充值)】
    Player->>Admin: 交付現金要求開分
    Admin->>Local: 使用 8PIN 碼登入
    Admin->>Local: 點選遊戲並輸入開分金額
    Local->>Central: 傳送開分訂單 (含本地端資訊)
    Central->>GameAPI: 呼叫 API 增加遊戲點數
    GameAPI-->>Central: 回傳點數增加成功
    Central-->>Central: 將訂單紀錄存入資料庫 (DB)
    Central->>Local: 回傳開分成功結果
    Local->>Admin: 顯示開分完成
    Central->>Player: (玩家於遊戲端自行確認充值成功)

    Note over Player, GameAPI: 【洗分流程 (兌換)】
    Player->>Admin: 要求洗分結算
    Admin->>Local: 使用 8PIN 碼登入
    Admin->>Local: 點選遊戲並輸入洗分金額
    Local->>Central: 傳送洗分訂單 (含本地端資訊)
    Central->>GameAPI: 呼叫 API 扣減遊戲點數
    GameAPI-->>Central: 回傳點數扣減成功
    Central-->>Central: 將訂單紀錄存入資料庫 (DB)
    Central->>Local: 回傳洗分成功結果
    Local->>Admin: 顯示洗分完成
    Central->>Player: (玩家於遊戲端自行確認扣款成功)
    Admin->>Player: 交付現金給玩家
```

### 2.1.2 遊戲下注與派彩流程 (匿名機台模式) [L43-L46]

本流程說明未登入玩家的機台如何透過本地快取進行下注，並於背景非同步將紀錄上傳至集中式後台。

```mermaid
sequenceDiagram
    autonumber
    participant Device as 機台 (匿名端)
    participant Local as 單機本地後台
    participant Central as 集中式後台
    participant GameAPI as 遊戲商 API
 
    Device->>Local: 選擇並載入連線遊戲
    Device->>Local: 玩家點擊發起下注
    Local-->>Local: 預先扣減本地快取餘額
    Local->>Central: [異步背景發送] 交易下注紀錄
    Local->>GameAPI: API 轉發下注請求
    GameAPI-->>Local: 回傳遊戲結果 (輸贏狀態)
    Local-->>Local: 計算派彩與更新本地餘額
    Local->>Central: [異步背景發送] 派彩資料紀錄
    Local->>Device: 畫面顯示最終遊戲結果
```

## 2.2 機台管理與註冊維護流程

### 2.2.1 新機台註冊與綁定 (Device Pairing) [C108 / L51]

機台首次架設時由安裝人員註冊綁定雲端，並經由管理員審核、版本更新與設定同步後，正式啟用並核發永久通訊金鑰。

```mermaid
sequenceDiagram
    autonumber
    participant Admin as 平台管理員
    participant Central as 集中式後台
    participant Local as 單機本地後台
    participant Man as 現場安裝人員

    Admin->>Central: 產生 12 碼註冊金鑰 (72hr 有效)
    Central-->>Admin: 系統顯示註冊金鑰
    Admin->>Man: 交付註冊金鑰
  
    Man->>Local: 機台開機，輸入初始 PIN (000000)
    Man->>Local: 輸入集中端網址/IP、註冊金鑰與機台名稱
  
    Local->>Central: 發送註冊請求 (含金鑰、MAC、硬體與版本資訊)
    Central-->>Central: 驗證金鑰有效性、產生暫時 ID 並加入待審核
    Central->>Local: 回傳狀態：等待審核
  
    Admin->>Central: 於待審核列表確認資訊，選擇門市後點擊 Approved
    Central-->>Central: 產生正式 ID，綁定 MAC 並建立安全通訊通道
    Central->>Local: 回傳狀態：已審核，建立連線與比對版本
  
    Local-->>Central: 版本比對
    Central->>Admin: 列表顯示「需更新」提示 (若版本過舊)
    Admin->>Central: 點擊「執行更新」
  
    Central->>Local: OTA 下發最新版本更新包
    Local-->>Local: 下載、安裝並自動重啟
```

## 附錄：功能矩陣欄位說明

### 欄位定義

| 欄位               | 說明                                     | 圖示                                                          |
| ------------------ | ---------------------------------------- | ------------------------------------------------------------- |
| **中央後台** | 功能是否在中央後台（集中式後台）中提供   | ✅ = 有此功能 ❌ = 無此功能                                   |
| **遠端控制** | 功能是否支援從中央後台遠端控制本地機台   | ✅ = 可遠端控制 ❌ = 不可遠端控制 - = 不適用                  |
| **本機設定** | 功能是否可在本地機台直接設定（不連線時） | ✅ = 完全支援 ❌ = 不支援 ⚙️ = 部分支援/本地設定 - = 不適用 |

### 圖示說明

| 圖示 | 含義                                                          |
| ---- | ------------------------------------------------------------- |
| ✅   | **支援** - 功能完全支援此模式                           |
| ❌   | **不支援** - 功能不支援此模式                           |
| ⚙️ | **部分支援** - 功能部分支援（如本地可檢視但需雲端啟用） |
| -    | **不適用** - 此模式不適用於該功能                       |

### 範例說明

| 功能                 | 中央後台 | 遠端控制 | 本機設定 | 說明                       |
| -------------------- | :------: | :------: | :------: | -------------------------- |
| 遊戲開分/洗分 (L45)  |    ✅    |    ✅    |    ❌    | 測試帳號開洗分功能         |
| 機台設定 (C14)       |    ✅    |    ✅    |    ✅    | 中央可設定，也可本地設定   |
| 遊戲配置 (C25)       |    ✅    |    ✅    |   ⚙️   | 中央可配置，本地僅部分設定 |
| 機台編輯/刪除 (C101) |    ✅    |    ❌    |    ❌    | 僅中央後台可編輯/刪除機台  |

### 設計原則

1. **中央後台為主** - 大部分功能集中在中央後台統一管理
2. **遠端控制** - 需要即時連線的操作（如開洗分）支援遠端
3. **本機設定** - 網路不穩定時，部分功能可在本地運作
