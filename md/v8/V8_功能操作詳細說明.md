## 2. 機台管理操作

### 2.1 新增機台

**操作流程**：

```
1. 點擊「新增機台」按鈕
2. 填寫機台基本資料
3. 設定機台參數
4. 驗證機台連線
5. 完成新增
```

**輸入欄位**：

| 欄位名稱 | 類型 | 必填 | 說明 |
|----------|------|------|------|
| 機台編號 | 文字 | ✅ | 唯一識別碼，格式：MCH-XXXX |
| 機台名稱 | 文字 | ✅ | 自訂名稱，最長 50 字元 |
| 機台型號 | 下拉選單 | ✅ | 選擇硬體型號 |
| 所屬站點 | 下拉選單 | ✅ | 選擇所屬營業站點 |
| IP 位址 | IP | ✅ | 機台網路位置 |
| 連接埠 | 數字 | ✅ | 預設 8080 |
| 遊戲類型 | 多選 | ✅ | 老虎機/捕魚/棋牌 |
| 備註 | 文字 | - | 選填備註說明 |

**畫面 Mockup**：
![機台列表](./assets/mockup_mvp_machine_list.html)

---

### 2.2 編輯機台

**操作流程**：

```
1. 點擊機台列表中的編輯按鈕
2. 修改機台資料
3. 儲存變更
4. 系統記錄操作日誌
```

**可編輯欄位**：

| 欄位名稱 | 說明 |
|----------|------|
| 機台名稱 | 修改顯示名稱 |
| 所屬站點 | 變更所屬站點 |
| IP 位址 | 更新網路設定 |
| 機台狀態 | 線上/離線/維護 |
| 備註 | 更新備註資訊 |

---

### 2.3 刪除機台

**操作流程**：

```
1. 選取要刪除的機台
2. 點擊刪除按鈕
3. 確認刪除對話框
4. 執行軟刪除（保留資料）
```

**刪除規則**：
- 僅可刪除離線狀態機台
- 刪除前需確認無進行中交易
- 系統保留刪除紀錄 90 天

---

### 2.4 遠端指令

**支援指令**：

| 指令名稱 | 代碼 | 說明 |
|----------|------|------|
| 重新啟動 | restart | 遠端重啟機台系統 |
| 關機 | shutdown | 遠端關閉機台 |
| 開分 | cash_in | 設定機台開分分數 |
| 洗分 | cash_out | 設定機台洗分分數 |
| 強制同步 | force_sync | 立即同步資料 |

## 3. 玩家管理操作

### 3.1 新增玩家

**操作流程**：

```
1. 點擊「新增玩家」按鈕
2. 填寫玩家基本資料
3. 設定玩家權限
4. 建立錢包帳戶
5. 完成註冊
```

**輸入欄位**：

| 欄位名稱 | 類型 | 必填 | 說明 |
|----------|------|------|------|
| 玩家帳號 | 文字 | ✅ | 唯一識別碼，6-20 字元 |
| 玩家暱稱 | 文字 | ✅ | 顯示名稱 |
| 手機號碼 | 電話 | ✅ | 驗證用 |
| 電子郵件 | Email | - | 選填 |
| VIP 等級 | 下拉選單 | ✅ | 設定會員等級 |
| 代理商 | 下拉選單 | ✅ | 所屬代理商 |
| 備註 | 文字 | - | 選填 |

---

### 3.2 餘額調整

**操作流程**：

```
1. 搜尋並選取玩家
2. 點擊「調整餘額」按鈕
3. 選擇調整類型
4. 輸入金額與原因
5. 系統驗證與執行
6. 產生交易紀錄
```

**調整類型**：

| 類型 | 代碼 | 說明 |
|------|------|------|
| 存款 | deposit | 玩家儲值 |
| 取款 | withdraw | 玩家提款 |
| 調整 | adjust | 人工調整 |

**驗證機制**：
- 需輸入管理 PIN 碼
- 大於 $10,000 需主管覆核
- 產生數位簽章憑證

**畫面 Mockup**：
![玩家列表](./assets/mockup_mvp_player_list.html)

---

### 3.3 移除玩家

**操作流程**：

```
1. 搜尋目標玩家
2. 檢查帳戶狀態（餘額必須為 0）
3. 執行停用而非刪除
4. 保留交易紀錄
```

**停用規則**：
- 餘額必須為 0
- 無進行中交易
- 保留資料供查帳使用

## 4. 交易管理操作

### 4.1 交易查詢

**操作流程**：

```
1. 進入交易列表頁面
2. 設定篩選條件
3. 執行查詢
4. 查看交易詳情
5. 匯出報表
```

**篩選條件**：

| 條件 | 類型 | 說明 |
|------|------|------|
| 日期區間 | 日期picker | 起始日期 ~ 結束日期 |
| 玩家帳號 | 文字 | 精確或模糊搜尋 |
| 機台編號 | 文字 | 所屬機台 |
| 交易類型 | 多選 | 存款/取款/調整/派彩 |
| 狀態 | 多選 | 成功/失敗/處理中 |
| 金額區間 | 數字範圍 | 最小 ~ 最大金額 |

**時間範圍快速篩選**：
- 今日
- 昨日
- 本週
- 上週
- 本月
- 上月
- 自訂區間

**畫面 Mockup**：
![交易列表](./assets/mockup_mvp_data_list.html)

---

### 4.2 交易沖銷

**操作流程**：

```
1. 選取要沖銷的交易
2. 點擊「沖銷」按鈕
3. 填寫沖銷原因
4. 系統驗證沖銷資格
5. 執行沖銷
6. 產生反向交易
7. 記錄沖銷人員
```

**沖銷規則**：
- 僅可沖銷 24 小時內交易
- 需主管授權
- 自動產生沖銷憑證（PDF）

**輸入欄位**：

| 欄位名稱 | 類型 | 必填 | 說明 |
|----------|------|------|------|
| 沖銷原因 | 下拉選單 | ✅ | 選擇沖銷原因 |
| 詳細說明 | 文字 | ✅ | 補充說明 |
| 授權碼 | 數字 | ✅ | 主管授權碼 |

---

### 4.3 交易爭議處理

**操作流程**：

```
1. 建立爭議單
2. 調閱相關資料
3. 進行調查
4. 做出裁決
5. 執行補救措施
6. 結案存檔
```

## 7. OTA 更新操作

### 7.1 上傳新版本

**操作流程**：

```
1. 點擊「上傳版本」按鈕
2. 選擇版本檔案
3. 填寫版本資訊
4. 系統計算 SHA-256
5. 驗證檔案完整性
6. 完成上傳
```

**輸入欄位**：

| 欄位名稱 | 類型 | 必填 | 說明 |
|----------|------|------|------|
| 版本號碼 | 文字 | ✅ | 格式：v8.0.x |
| 版本說明 | 文字 | ✅ | 更新內容描述 |
| 檔案 | 檔案 | ✅ | .zip 格式 |
| 目標機型 | 多選 | ✅ | 適用機型 |
| 強制更新 | 開關 | - | 是否強制更新 |

**檔案驗證**：
- 自動計算 MD5
- 自動計算 SHA-256
- 驗證檔案大小
- 檢查壓縮完整性

**畫面 Mockup**：
![OTA 更新](./assets/mockup_flg_ota.html)

---

### 7.2 發布版本

**操作流程**：

```
1. 選擇要發布的版本
2. 選擇測試機台 (2-3台)
3. 開始測試派發
4. 監控 24 小時
5. 確認無異常
6. 正式派發 (全部/分批)
```

**派發模式**：

| 模式 | 說明 |
|------|------|
| 測試派發 | 僅派發至指定機台 |
| 全部派發 | 一次派發至所有機台 |
| 分批派發 | 按批次逐步派發 |
| 定時派發 | 設定時間自動派發 |

---

### 7.3 版本管理

**功能說明**：
- 版本列表檢視
- 派發進度監控
- 失敗自動還原
- 版本歷史記錄

## 8. 監控中心操作

## 9. 錯帳對帳操作

### 9.1 錯帳類型

| 類型代碼 | 說明 |
|----------|------|
| missing_deposit | 轉帳未到帳 |
| missing_withdrawal | 轉出未確認 |
| payout_error | 派彩錯誤 |
| system_error | 系統錯誤 |

### 9.2 錯帳處理流程

**操作流程**：

```
1. 偵測錯帳 (自動/人工)
2. 建立錯帳單
3. 調查原因
4. 選擇處理方式
5. 執行補發/退款/沖銷
6. 產生憑證 (PDF)
7. 結案
```

**處理方式**：

| 方式 | 說明 |
|------|------|
| 補發 | 重新執行交易 |
| 退款 | 退還款項 |
| 沖銷 | 反向沖銷 |

**畫面 Mockup**：
![錯帳對帳](./assets/mockup_flg_reconciliation.html)

---

### 9.3 錢包自動對帳

**對帳時間**：每日 UTC 00:00

**對帳項目**：
- 中央錢包 vs 遊戲商錢包
- 交易紀錄一致性
- 餘額驗證

**告警規則**：
- 錯帳金額 > $1000 發送通知
