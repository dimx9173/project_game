## LM05 [M11] 本地系統與認證 (Local System Settings & Auth)

### US-LOCAL-01：機台首次初始化註冊與 MVP 基礎設定
**角色**：現場安裝工程師
**描述**：當機台首次安裝時，我想要輸入中央伺服器資訊並進行註冊，以便讓機台納入中央管理；同時我需要設定 MVP 基礎網路參數。
*   **驗收標準 (AC)**：
    1.  支援輸入中央 API 位址 (HTTPS URL) 與 API Key 進行伺服器綁定。
    2.  系統自動提取機台 MAC 位址、CPU/RAM 資訊作為註冊識別碼。
    3.  註冊成功後，機台需能安全保存中央核發的 Token / UUID (`InstanceId`)。
    4.  支援提供精簡版 (MVP) 網路設定介面，快速完成上線。
*   **對應 Mockup**：`mockup_local_register.html`, `mockup_mvp_local_setup.html`

### US-LOCAL-02：管理員 PIN 碼登入與本地安全控管
**角色**：店端管理員 / 維修工程師
**描述**：當我要操作本地端後台時，我想要透過 4-6 位 PIN 碼進行身分驗證，並在離線狀態下也能有限度登入，以便保護敏感資料。
*   **驗收標準 (AC)**：
    1.  提供獨立的純數字觸控友好型 PIN 碼輸入介面。
    2.  PIN 碼驗證需發送至中央伺服器；若處於無法連線狀態，則依據本地加密快取驗證。
    3.  提供鎖定機制：連續輸入錯誤 5 次鎖定 15 分鐘。
*   **對應 Mockup**：`mockup_local_login.html`

### US-LOCAL-03：本地系統參數與硬體設定
**角色**：技術支持工程師
**描述**：我想要在本地調整硬體特定參數（如週邊設備、讀卡機、螢幕解析度），以便適配不同的實體機殼。
*   **驗收標準 (AC)**：
    1.  提供進階的本地環境與系統設定介面。
    2.  設定變更後，需於下一次心跳包 (Heartbeat) 中回報給中央。
*   **對應 Mockup**：`mockup_local_system_settings.html`

## LM04 [M04] 戰情與本地即時統計 (Local Dashboard)

### US-LOCAL-04：本地營運數據與儀表板總覽
**角色**：店端管理員
**描述**：我想要在首頁看到今日整機的營收、總下注額、派彩額，以及機台效能，以便一目了然機台狀態。
*   **驗收標準 (AC)**：
    1.  儀表板需明確標示目前所有數值為「本地快取」，可能與中央有秒差。
    2.  圖表顯示連線品質（綠、黃、紅燈）與即時硬體資源損耗（CPU / 記憶體）。
    3.  允許在儀表板頁面直接點擊快捷指令（如重啟服務）。
*   **對應 Mockup**：`mockup_local_dashboard.html`

## LM02 [M02] 本地機台與遊戲管理 (Local Machine & Game)

### US-LOCAL-05：本機帳號與開洗分作業 (Local Cash Handling)
**角色**：現場維運主管
**描述**：當玩家於本機前需要人工上下分時，我想要透過後台進行開分 (Cash-in) 或洗分 (Cash-out)，以便即時服務玩家。
*   **驗收標準 (AC)**：
    1.  提供金額輸入介面並支援快選按鈕（+1000, +5000）。
    2.  開洗分操作必須寫入本地交易佇列，等待同步至中央；同時必須留下操作人員紀錄。
*   **對應 Mockup**：`mockup_local_cash.html`

### US-LOCAL-06：單機遊戲列表管理與本機資訊監控
**角色**：店端管理員
**描述**：我想要在本機能夠查看當前安裝的遊戲清單、版號，並監視本機被分配的配置參數。
*   **驗收標準 (AC)**：
    1.  以唯讀 (Read-only) 方式展示中央指派給該機台的遊戲列表與圖標。
    2.  顯示單機詳細參數，包含機台 ID、營業時間與標籤，且禁止於本地修改上述資訊（需由中央控管）。
*   **對應 Mockup**：`mockup_local_game.html`, `mockup_local_machine.html`

## LM12 [M12] 交易佇列與同步引擎 (Local Sync & Queue)

### US-LOCAL-07：本地雙向佇列與斷線同步狀態監視
**角色**：系統 / 店端管理員
**描述**：我想要清楚看到機台與中央的同步狀態，當網路斷線時，需要確保交易能存在本地，連線後自動回傳。
*   **驗收標準 (AC)**：
    1.  提供專屬頁面顯示「Pending Uploads (待上傳)」與「Pending Downloads (待下載)」的筆數與狀態。
    2.  **離線安全機制**：當處於斷線狀態且佇列超過警戒值時，必須於前端警示。
    3.  提供手動「立即同步」按鍵，以及錯誤日誌 (Sync Logs) 檢視。
*   **對應 Mockup**：`mockup_local_sync.html`
