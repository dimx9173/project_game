## 1. 文件概述

本文件詳細闡述遊戲平台管理系統的整體系統架構，包含集中式後台與單機本地後台兩個層次的架構設計、資料流、通訊機制等內容。

## 2. 系統架構概述

### 2.1 雙層後台架構

遊戲平台採用雙層後台架構設計。

- **上層（集中式後台）**：負責全局資料監控、遠端機台管理、全平台交易統計、玩家集中管理等功能。
- **下層（單機本地後台）**：負責本地機台管理、本地遊戲配置、本地交易結算、本地統計分析等功能。
  兩個後台相互獨立，但在連線時可進行資料同步與對帳。

### 2.2 系統組件

系統包含以下主要組件：

- 集中式後台前端（React + Tailwind CSS）
- 集中式後台後端（Express.js + Node.js）
- 單機本地後台前端（React + Tailwind CSS）
- 單機本地後台後端（Express.js + Node.js）
- 集中式資料庫（PostgreSQL）
- 單機本地資料庫（SQLite）
- 遊戲引擎
- 玩家端應用程式

## 3. 集中式後台架構

### 3.1 前端架構

集中式後台前端採用 **React 19 + Tailwind CSS 4**。使用 Wouter 進行路由管理，shadcn/ui 提供統一的 UI 元件，Recharts 用於資料視覺化，React Context 進行狀態管理。支援響應式設計，適配桌面、平板、手機等多種裝置。

### 3.2 後端架構

集中式後台後端採用 **Express.js** 框架，Node.js 執行環境。提供 RESTful API 介面，支援 WebSocket 實時通訊。使用 PostgreSQL 資料庫進行資料持久化。

### 3.3 核心模組

集中式後台包含 10 個核心模組：

1. **儀表板模組**：全平台 KPI 監控。
2. **機台管理模組**：遠端機台管理。
3. **遊戲管理模組**：全局遊戲庫管理。
4. **玩家管理模組**：集中式玩家管理。
5. **錢包交易模組**：全平台交易統計。
6. **串接遊戲管理模組**：遊戲商 API 管理。
7. **多人遊戲模組**：多人遊戲管理。
8. **使用者權限模組**：帳號與角色管理。
9. **版本更新模組**：應用程式版本管理。
10. **介面設定模組**：玩家端配置管理。

## 4. 單機本地後台架構

### 4.1 前端架構

單機本地後台前端採用與集中式後台相同的（**React 19 + Tailwind CSS 4**），確保設計風格與使用體驗的一致性。支援本地化部署，可在機台上直接運行。

### 4.2 後端架構

單機本地後台後端採用 **Express.js** 框架，Node.js 執行環境。提供 RESTful API 介面，支援 WebSocket 實時通訊。使用 SQLite 作為本地資料儲存，支援本地快取。

### 4.3 核心模組

單機本地後台包含 9 個核心模組：

1. **儀表板模組**：本機 KPI 監控。
2. **機台管理模組**：本地機台管理。
3. **遊戲管理模組**：本地遊戲庫管理。
4. **交易管理模組**：本地交易統計。
5. **玩家管理模組**：本地玩家管理。
6. **統計報表模組**：本地統計分析。
7. **系統設定模組**：本地配置管理。
8. **同步與對帳模組**：資料同步驗證。
9. **監控與日誌模組**：系統監控與日誌管理。

## 5. 資料同步與對帳機制

### 5.1 同步流程

當單機連線到集中式後台時，自動觸發資料同步流程。首先檢查離線期間的本地變更，上傳本地交易紀錄到後台。然後下載後台的遊戲更新、配置更新。最後驗證資料一致性，完成同步。

### 5.2 衝突解決

當機台離線期間進行的操作與後台操作衝突時，系統採用時間戳優先、版本號優先、人工審核、交易回滾等策略進行解決，確保資料最終一致。

### 5.3 對帳機制

系統定期進行對帳檢查，驗證本地資料與後台資料的一致性。對於不一致的項目，記錄並標記為待審核，由管理員手動處理。

## 6. 安全性架構

### 6.1 認證與授權

系統採用帳號密碼認證，使用 JWT Token 進行會話管理。實施基於角色的存取控制（RBAC），不同角色的使用者擁有不同的操作權限。

### 6.2 資料加密

所有敏感資料（玩家帳號、交易紀錄、配置資訊）使用 **AES-256** 加密演算法進行加密儲存。通訊採用 HTTPS 加密協議，確保資料傳輸安全。

### 6.3 交易簽名驗證

所有交易使用 **HMAC/RSA** 數位簽名進行驗證，確保交易的真實性與完整性，防止交易被竄改。

### 6.4 稽核日誌

記錄所有重要操作的詳細日誌，包括操作者、操作時間、操作內容、操作結果等資訊，支援追蹤與審計。

## 8. 部署架構

### 8.1 集中式後台部署

集中式後台部署在雲端或資料中心。前端與後端服務皆使用容器化部署（Docker），支援自動擴展。資料庫使用主從複製實現高可用。

### 8.2 單機本地後台部署

單機本地後台與遊戲引擎部署在同一台機器上。支援 Windows、Linux、macOS 等作業系統。使用 Node.js 執行環境，SQLite 作為本地資料庫。

### 8.3 網路拓撲

- 集中式後台與單機本地後台通過 Internet 連線。
- 單機本地後台與遊戲引擎通過本地網路連線。
- 玩家端應用程式與遊戲引擎通過本地網路連線。

## 9. 業務流程圖

### 9.1 玩家註冊與登入流程

```text
玩家 → 本地後台：註冊請求
本地後台 → 玩家：驗證手機/驗證碼
玩家 → 本地後台：填寫基本資料
本地後台：建立本地帳號
[離線模式]：直接登入
[連線模式]：本地後台 → 集中式後台：同步到中央
集中式後台：建立中央帳號
本地後台 → 玩家：登入成功
```

### 9.2 充值流程

```text
玩家 → 本地後台：選擇充值金額 + 支付方式
本地後台 → 集中式後台：風控檢查
集中式後台 → 支付通道：建立充值訂單
本地後台 → 玩家：調起支付介面
支付通道 → 玩家：完成支付
支付通道 → 集中式後台：支付回調
集中式後台：驗證並更新餘額
集中式後台 → 本地後台：同步餘額
本地後台 → 玩家：充值成功通知
```

### 9.3 遊戲下注與派彩流程

```text
玩家 → 本地後台：選擇遊戲
本地後台 → 玩家：載入遊戲
玩家 → 本地後台：下注
本地後台：扣減本地餘額
本地後台 → 集中式後台：同步下注
集中式後台 → 遊戲商 API：轉發下注
遊戲商 API → 集中式後台：遊戲結果
集中式後台：計算派彩
集中式後台 → 本地後台：更新餘額
本地後台 → 集中式後台：同步派彩
本地後台 → 玩家：顯示遊戲結果
```

### 9.4 兌換流程

```text
玩家 → 本地後台：申請兌換
本地後台 → 集中式後台：風控檢查
集中式後台：驗證餘額足夠
集中式後台 → 本地後台：扣減本地餘額
本地後台 → 集中式後台：建立兌換訂單
本地後台 → 玩家：審核中通知
集中式後台：[自動/人工審核]
集中式後台 → 支付通道：發起支付
支付通道 → 集中式後台：支付回調
集中式後台 → 本地後台：同步狀態
本地後台 → 玩家：兌換成功通知
```

### 9.5 資料同步與對帳流程

```text
本地後台 → 集中式後台：連線觸發同步
本地後台 ← 集中式後台：交換版本號
本地後台 → 集中式後台：上傳本地變更
本地後台 ← 集中式後台：衝突檢測
[有衝突]：解決策略（時間戳優先/版本號優先/人工審核）
本地後台 ← 集中式後台：下載中央變更
本地後台 → 集中式後台：套用並確認
本地後台 ← 集中式後台：對帳檢查
本地後台 ← 集中式後台：產生對帳報告
[有不一致]：標記待審核
```

### 9.6 機台遠端管理流程

```text
平台管理員 → 集中式後台：下達指令（開分/洗分/重啟）
集中式 → 本地後台：驗證權限
本地後台：執行指令
本地後台 → 集中式後台：回傳結果
集中式後台 → 平台管理員：顯示執行結果
集中式後台：記錄日誌
```
