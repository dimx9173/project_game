# Mockup 驗收標準

**版本：** 1.0  
**日期：** 2026-03-06  
**適用範圍：** `mockup/central/*.html` + `mockup/local/*.html`

---

## 1. 驗收等級定義

| 等級 | 定義 | 標準 |
|------|------|------|
| 🟢 **通過** | 完全符合驗收標準 | 0 個 Critical 問題 |
| 🟡 **有條件通過** | 輕微問題不影響功能 | ≤ 3 個 Minor 問題 |
| 🔴 **需修正** | 存在影響使用的問題 | ≥ 1 個 Critical 問題 |

---

## 2. 驗收檢查清單

### 2.1 結構驗收 (Structural Acceptance)

| # | 檢查項目 | Critical? | 通過標準 |
|---|---------|-----------|---------|
| 1.1 | HTML 基本結構 | ✅ 是 | 包含 `<!DOCTYPE html>`, `<meta name="viewport">`, `<title>` |
| 1.2 | Sidebar 導航 | ✅ 是 | 登入/密碼重設頁面除外，其他均需 `sidebar.js` |
| 1.3 | CSS/JS 引用 | ✅ 是 | 所有外部資源可正確載入 |
| 1.4 | 頁面標題正確性 | ✅ 是 | `<title>` 正確反映頁面功能 |

### 2.2 功能驗收 (Functional Acceptance)

| # | 檢查項目 | Critical? | 通過標準 |
|---|---------|-----------|---------|
| 2.1 | 功能映射完整性 | ✅ 是 | 每個功能代碼 (C01-C106, L01-L46) 有對應 Mockup |
| 2.2 | 互動元素存在 | ✅ 是 | 按鈕/連結可點擊 (有 onclick 或 href) |
| 2.3 | 表單元素完整性 | ✅ 是 | 必要欄位有 `required` 屬性 |
| 2.4 | 狀態切換元素 | 🟡 否 | Toggle/Tab 有狀態變化樣式 |

### 2.3 UI/UX 驗收

| # | 檢查項目 | Critical? | 通過標準 |
|---|---------|-----------|---------|
| 3.1 | 色彩主題一致 | 🟡 否 | 使用統一的紫色主題 (#8b5cf6) |
| 3.2 | 玻璃擬態風格 | 🟡 否 | 卡片使用 `glass-card` class |
| 3.3 | 字體系統 | 🟡 否 | 中文使用 Noto Sans TC |
| 3.4 | 圓角一致性 | 🟡 否 | 卡片統一使用 `rounded-2xl` |
| 3.5 | 互動回饋 | 🟡 否 | 有 hover / active 效果 |
| 3.6 | 確認對話框 | ✅ 是 | 刪除/停用操作有確認 Modal |
| 3.7 | 錯誤提示 UI | ✅ 是 | 表單驗證有錯誤訊息顯示 |
| 3.8 | 空狀態處理 | 🟡 否 | 無資料時顯示友善提示 |

### 2.4 響應式驗收 (Responsive Acceptance)

| # | 檢查項目 | Critical? | 通過標準 |
|---|---------|-----------|---------|
| 4.1 | 桌面版布局 | ✅ 是 | 1440px 正常顯示 |
| 4.2 | 平板布局 | 🟡 否 | 768px 布局正確 |
| 4.3 | 手機布局 | ✅ 是 | 375px 無水平溢版 |
| 4.4 | 表格響應式 | ✅ 是 | 小螢幕可橫向滾動或卡片化 |
| 4.5 | Sidebar 響應式 | 🟡 否 | 中央後台有漢堡選單或收合功能 |

### 2.5 無障礙驗收 (Accessibility Acceptance)

| # | 檢查項目 | Critical? | 通過標準 |
|---|---------|-----------|---------|
| 5.1 | 色彩對比度 | ✅ 是 | WCAG AA (4.5:1) 以上 |
| 5.2 | ARIA 標籤 | 🟡 否 | 主要元素有 role / aria-label |
| 5.3 | 鍵盤導航 | 🟡 否 | 可聚焦元素有 focus 樣式 |
| 5.4 | 表單標籤 | ✅ 是 | label 與 input 正確關聯 |
| 5.5 | 圖片 alt | 🟡 否 | 有含義的 alt 文字 |

### 2.6 命名驗收 (Naming Acceptance)

| # | 檢查項目 | Critical? | 通過標準 |
|---|---------|-----------|---------|
| 6.1 | 檔案命名規範 | 🟡 否 | 符合 `mockup_{type}_{module}_{function}.html` |
| 6.2 | 前綴統一 | 🟡 否 | 使用 `central_` 或 `local_` 而非 `mvp_` / `std_` |

## 3. 問題優先級定義

| 等級 | 縮寫 | 定義 | 範例 |
|------|------|------|------|
| 🔴 **Critical** | C | 阻礙開發或影響使用者操作 | 缺少確認對話框、對比度不足無法閱讀 |
| 🟡 **Minor** | M | 輕微問題但不影響功能 | 命名不一致、圓角不統一 |
| 🟢 **Info** | I | 優化建議 | 動畫效果、空狀態提示 |

## 4. 驗收流程

```
┌─────────────────┐
│  1. 結構檢查    │ ──→ 自動化掃描
└────────┬────────┘
         ▼
┌─────────────────┐
│  2. 功能檢查    │ ──→ 對照功能矩陣
└────────┬────────┘
         ▼
┌─────────────────┐
│  3. UI/UX 檢查  │ ──→ 手動抽檢
└────────┬────────┘
         ▼
┌─────────────────┐
│  4. 響應式檢查  │ ──→ 多尺寸截圖
└────────┬────────┘
         ▼
┌─────────────────┐
│  5. 無障礙檢查  │ ──→ DevTools 審查
└────────┬────────┘
         ▼
┌─────────────────┐
│  6. 問題彙總    │ ──→ 產生報告
└─────────────────┘
```

## 5. 驗收決定標準

| 結果 | Critical 數 | Minor 數 | 行動 |
|------|------------|---------|------|
| ✅ **通過** | 0 | ≤ 5 | 允許進入開發 |
| ⚠️ **有條件通過** | 0 | 6-15 | 記錄問題，允許進入開發 |
| ❌ **需修正** | ≥ 1 | 任意 | 修正後重新驗收 |
