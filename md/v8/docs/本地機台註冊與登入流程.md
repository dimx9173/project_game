# 本地機台註冊與登入流程說明

## 📋 概述

本文檔說明單機本地後台的註冊與登入機制，釐清中央帳號與本地 PIN 碼的使用場景。

---

## 🔄 註冊流程

### 首次開機註冊

```
┌─────────────────────────────────────────────────────────────┐
│  機台首次開機                                                │
│     ↓                                                       │
│  顯示「機台註冊」畫面                                         │
│     ↓                                                       │
│  輸入「集中式後台產生的註冊密碼」                              │
│     ↓                                                       │
│  連線至中央伺服器驗證                                         │
│     ↓                                                       │
│  註冊成功 → 綁定該機台至中央帳號                              │
└─────────────────────────────────────────────────────────────┘
```

### 註冊密碼來源

| 項目 | 說明 |
|------|------|
| **產生方** | 集中式後台 |
| **產生時機** | 管理員在集中式後台「機台部署」功能中新增機台時 |
| **密碼形式** | 一次性註冊密碼（類似啟用碼） |
| **有效期限** | 單次使用，註冊後失效 |

---

## 🔐 登入機制

### 雙重登入方式

| 情境 | 登入方式 | 用途 |
|------|---------|------|
| **機台連線時** | 中央帳號 + PIN 碼 | 正常營運登入 |
| **機台離線時** | 本地 PIN 碼 | 緊急調整設定 |

---

## 👤 連線時登入（正常情況）

```
┌─────────────────────────────────────────────────────────────┐
│  機台已連線至中央伺服器                                       │
│     ↓                                                       │
│  顯示「中央帳號登入」畫面                                      │
│     ↓                                                       │
│  輸入：                                                      │
│    - 帳號（Email）                                           │
│    - 密碼 / 或選擇 PIN 碼登入                                 │
│     ↓                                                       │
│  中央伺服器驗證                                               │
│     ↓                                                       │
│  登入成功 → 進入本地後台（依該帳號權限顯示功能）               │
└─────────────────────────────────────────────────────────────┘
```

### 連線時登入特點

- ✅ 使用集中式帳號體系
- ✅ 權限由中央統一管理
- ✅ 可執行所有授權功能
- ✅ 操作記錄同步至中央

---

## 🔌 離線時登入（網路中斷）

```
┌─────────────────────────────────────────────────────────────┐
│  機台離線（無法連接中央伺服器）                                │
│     ↓                                                       │
│  顯示「本地登入」畫面                                         │
│     ↓                                                       │
│  輸入「本地 PIN 碼」                                          │
│     ↓                                                       │
│  本地驗證（不經過中央）                                        │
│     ↓                                                       │
│  登入成功 → 進入本地後台（僅限基礎設定功能）                   │
└─────────────────────────────────────────────────────────────┘
```

### 離線時登入特點

- ✅ 僅驗證本地 PIN 碼
- ✅ **無需中央帳號**
- ⚠️ 功能受限（僅能調整本地設定）
- ⚠️ 無法執行涉及雲端的操作

---

## 📝 PIN 碼說明

### 兩種 PIN 碼比較

| 類型 | 中央帳號 PIN | 本地 PIN |
|------|-------------|---------|
| **設定位置** | 集中式後台「使用者管理」 | 本地後台「本機設定」 |
| **使用時機** | 連線時登入 | 離線時登入 |
| **驗證位置** | 中央伺服器 | 本地機台 |
| **用途** | 身份識別 + 權限控管 | 離線緊急登入 |
| **重設方式** | 集中式後台管理員重設 | 連線時可由中央重設 |

### 本地 PIN 碼設定

```
本機設定頁面
   └── 安全設定區塊
        └── 本地 PIN 碼
             ├── 變更 PIN 碼（需連線時才能變更）
             └── PIN 碼規則：4-6 位數字
```

## 🎯 使用場景範例

### 場景一：新機台部署

1. 管理員在集中式後台新增機台「旗艦店 A-01」
2. 系統產生註冊密碼：`REG-ABC123XYZ`
3. 駐點工程師開機，輸入註冊密碼
4. 機台註冊成功，綁定至中央帳號體系
5. 正常使用中央帳號登入

### 場景二：網路中斷緊急處理

1. 機台網路異常，無法連接中央
2. 顯示「本地登入」選項
3. 駐點工程師輸入本地 PIN 碼：`1234`
4. 登入成功，進入本地後台
5. 調整網路設定 / 檢查離線佇列狀態
6. 網路恢復後，自動同步資料

### 場景三：忘記本地 PIN

1. 機台離線，工程師忘記本地 PIN
2. 必須先恢復網路連線
3. 使用中央帳號登入
4. 進入「本機設定」→「安全設定」
5. 重設本地 PIN 碼
6. 下次離線時即可使用新 PIN 登入

---

## ⚠️ 安全注意事項

1. **本地 PIN 碼僅供緊急使用**，日常營運應使用中央帳號登入
2. **本地 PIN 錯誤 5 次鎖定**，需連線後由中央解鎖
3. **本地操作記錄**會在連線後上傳至中央稽核
4. **敏感操作**（如開分/洗分）離線時無法執行
