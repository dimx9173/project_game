# 本地機台雙軌制 PIN 登入與授權流程

## 📋 概述

本文檔說明單機本地後台的**雙軌制 PIN 系統**，確保帳務安全與操作可追溯。

---

## 🔐 兩種 PIN 系統

| PIN 類型 | 長度 | 產生方式 | 有效期 | 使用時機 | 功能權限 |
|---------|------|---------|--------|---------|---------|
| **值班人員 PIN** | **8 位** | 集中式後台產生 | **12 小時** | 連線時 | ✅ 開分/洗分 + 所有功能 |
| **本地機台 PIN** | **7 位** | 機台設定 | 永久 | 離線時 | ⚠️ 僅設定/瀏覽，**❌ 不可開洗分** |

---

## 🟢 模式一：線上認證（值班人員 PIN）

### 產生流程

```
值班人員上班前
    ↓
登入集中式後台
    ↓
點擊「產生值班 PIN」
    ↓
系統產生 8 位 PIN（12hr 有效）
    ↓
顯示在螢幕 + 發送至員工 Email
```

### 登入流程

```
機台連線狀態
    ↓
輸入 8 位值班 PIN
    ↓
連線驗證 PIN 有效性
    ↓
識別員工身份（王小明, EMP-001）
    ↓
「🔵 線上認證模式」登入成功
    ↓
可執行：開分/洗分 + 所有功能
```

### 開分/洗分流程

```
輸入金額
    ↓
確認操作員：王小明 (EMP-001)
    ↓
再次輸入 8 位 PIN 確認
    ↓
執行開分/洗分
    ↓
交易記錄：時間 + 金額 + 員工編號 + PIN 驗證記錄
    ↓
即時同步至中央伺服器
```

---

## ⚪ 模式二：本地維護（機台 PIN）

### 用途

- 機台離線時的緊急維護
- 調整機台設定（網路、顯示等）
- 瀏覽本地快取資料

### 登入流程

```
機台離線狀態
    ↓
輸入 7 位本地機台 PIN
    ↓
本地驗證（不連線）
    ↓
「⚪ 本地維護模式」登入成功
    ↓
僅可：機台設定、資料瀏覽
    ↓
❌ 開分/洗分功能鎖定（防止帳務無法追蹤）
```

### 為何禁止開洗分？

| 風險 | 說明 |
|------|------|
| 身份不明 | 無法識別是誰執行的操作 |
| 帳務缺口 | 離線交易無法即時同步，可能產生差異 |
| 責任不清 | 發生爭議時無法追蹤責任人 |

## 📝 PIN 管理規則

### 值班人員 PIN（8位）

| 項目 | 規則 |
|------|------|
| **產生** | 集中式後台手動產生 |
| **通知** | 螢幕顯示 + Email 發送 |
| **有效期** | 12 小時 |
| **重設** | 隨時可產生新的，舊的立即失效 |
| **驗證** | 必須連線至中央伺服器 |
| **記錄** | 所有操作記錄員工身份 |

### 本地機台 PIN（7位）

| 項目 | 規則 |
|------|------|
| **設定** | 首次開機時設定（預設 0000000）|
| **變更** | 本機設定頁面修改 |
| **驗證** | 本地驗證，無需連線 |
| **用途** | 僅限機台維護 |
| **限制** | 禁止開分/洗分 |
| **重設** | 連線後可由中央重設 |

## 🔄 交班流程

```
員工 A 下班
    ↓
在機台點擊「交班」
    ↓
系統註銷目前 8 位 PIN
    ↓
員工 B 上班
    ↓
在集中式後台產生新的 8 位 PIN
    ↓
員工 B 使用新 PIN 登入機台
```

---

## ⚠️ 安全注意事項

1. **值班 PIN 不得分享** - 每人專屬，12hr 自動失效
2. **本地 PIN 僅供維護** - 禁止用於帳務操作
3. **開分/洗分必須連線** - 確保身份識別與即時同步
4. **交班必須註銷舊 PIN** - 防止重複使用
5. **PIN 錯誤 5 次鎖定** - 需管理員解鎖
