# 遊戲機台註冊流程說明

## 📋 概述

本文檔說明遊戲機台從出廠到正式營運的完整註冊流程，確保機台身份識別與平台管理。

---

## 🎯 設計原則

> **機台必須向集中式註冊才能啟用，由集中式產生 instanceId 並綁定 MAC。**

| 項目 | 說明 |
|------|------|
| **instanceId** | 集中式統一產生（UUID v4） |
| **綁定機制** | instanceId 與機台 MAC 位址綁定 |
| **唯一性** | 全平台唯一，確保機台識別 |
| **註冊時機** | 首次啟用時向集中式註冊，取得 instanceId |

---

## 🔄 完整註冊流程

### 階段一：出廠預設

```
┌─────────────────────────────────────────────────────────────┐
│  機台製造完成                                                │
│     ↓                                                       │
│  安裝系統映像檔                                               │
│     ↓                                                       │
│  預設本地 PIN：0000000（7位）                                │
│     ↓                                                       │
│  出廠 → 運送至門市                                           │
└─────────────────────────────────────────────────────────────┘
```

**出廠狀態**：
- 系統已安裝，但未註冊
- 本地 PIN 為預設值 0000000
- 無法執行開分/洗分（需註冊後才能開通）

---

### 階段二：首次開機設定

```
┌─────────────────────────────────────────────────────────────┐
│  機台送達門市，首次開機                                       │
│     ↓                                                       │
│  輸入預設 PIN：0000000                                       │
│     ↓                                                       │
│  進入本地設定頁面                                             │
│     ↓                                                       │
│  設定項目：                                                   │
│    - 機台名稱（例：旗艦店 A-01）                              │
│    - 中央伺服器位址                                          │
│    - 本地機台 PIN（7位，取代預設 0000000）                    │
│     ↓                                                       │
│  設定完成，機台嘗試連線中央                                    │
└─────────────────────────────────────────────────────────────┘
```

---

### 階段三：集中式註冊

```
┌─────────────────────────────────────────────────────────────┐
│  機台連線至中央伺服器                                         │
│     ↓                                                       │
│  發送註冊請求：                                               │
│    - 機台名稱                                                │
│    - MAC 位址                                                │
│    - 硬體型號                                                │
│    - 門市資訊（從連線來源識別）                                │
│     ↓                                                       │
│  中央伺服器驗證                                               │
│     ↓                                                       │
│  產生 instanceId（UUID v4）                                   │
│     ↓                                                       │
│  綁定 instanceId ↔ MAC 位址                                  │
│     ↓                                                       │
│  註冊成功，回傳 instanceId 給機台                              │
│     ↓                                                       │
│  機台儲存 instanceId 至本地資料庫                              │
└─────────────────────────────────────────────────────────────┘
```

---

### 階段四：機台啟用

```
┌─────────────────────────────────────────────────────────────┐
│  註冊完成                                                     │
│     ↓                                                       │
│  機台狀態：已註冊，等待值班人員登入                            │
│     ↓                                                       │
│  值班人員在集中式後台申請值班 PIN（8位，12hr有效）             │
│     ↓                                                       │
│  值班人員使用 8 位 PIN 登入機台                                │
│     ↓                                                       │
│  身份識別：王小明 (EMP-001)                                   │
│     ↓                                                       │
│  🎉 機台正式啟用，可執行開分/洗分                              │
└─────────────────────────────────────────────────────────────┘
```

## 🔐 安全機制

### 1. MAC 位址綁定

- 每個 instanceId 與機台 MAC 位址唯一綁定
- 防止機台被複製或非法轉移

### 2. 註冊驗證

- 中央伺服器驗證 MAC 位址合法性
- 檢查是否為已授權的硬體設備

### 3. 離線保護

- 未註冊機台無法執行開分/洗分
- 僅能進行基本設定

## 🔄 重新註冊情境

### 情境一：機台搬遷

```
機台從 A 門市搬至 B 門市
    ↓
在集中式後台「機台管理」中
    ↓
更新機台所屬門市資訊
    ↓
無需重新註冊，instanceId 不變
```

### 情境二：硬體故障更換

```
機台主機板損壞，MAC 位址改變
    ↓
聯繫技術支援
    ↓
驗證機台所有權
    ↓
解除舊 MAC 綁定
    ↓
重新註冊（產生新的 instanceId）
```

### 情境三：系統重置

```
機台系統重灌
    ↓
首次開機輸入預設 PIN：0000000
    ↓
設定本地 PIN（可設定為之前的 PIN）
    ↓
連線中央
    ↓
自動識別已註冊（透過 MAC）
    ↓
回復 instanceId，無需重新註冊
```

## ⚠️ 注意事項

1. **首次開機必須連線** - 才能自動向中央註冊
2. **預設 PIN 僅限首次** - 設定新 PIN 後 0000000 失效
3. **MAC 位址唯一** - 不可更改，否則視為不同機台
4. **註冊後才能營運** - 未註冊機台無法執行開分/洗分
