# Mockup 測試計畫

**版本：** 1.0  
**日期：** 2026-03-06  
**測試人員：** 測試員 🧪  
**測試範圍：** `mockup/central/*.html` (74 個) + `mockup/local/*.html` (5 個)

---

## 1. 測試目標

確保 Mockup 符合功能規格、UI/UX 設計規範，並為開發團隊提供品質合格的設計稿。

---

## 2. 測試策略

### 2.1 功能測試 (Functional Testing)

| 測試項目 | 測試方法 | 通過標準 |
|---------|---------|---------|
| 頁面結構完整性 | 檢查 HTML 基本結構 (`<!DOCTYPE html>`, `<meta viewport>`, `<title>`) | 所有頁面包含必要結構 |
| 互動元素可點擊性 | 檢查按鈕、連結、表單元素可點擊 | 所有可見元素綁定事件 |
| 表單驗證 | 檢查 input 類型與 required 屬性 | 符合資料類型驗證 |
| 狀態切換 | 檢查 toggle、tab 元素狀態變化 | 狀態正確反映 |
| 功能映射 | 對照功能矩陣文件 | 每個功能代碼有對應 Mockup |

### 2.2 相容性測試 (Compatibility Testing)

| 測試項目 | 測試方法 | 通過標準 |
|---------|---------|---------|
| 瀏覽器一致性 | Chrome / Firefox / Safari 渲染對比 | 視覺差異 < 5% |
| 裝置尺寸響應 | 320px / 768px / 1024px / 1440px 檢視 | 布局正確無破版 |
| 橫向滾動檢測 | 小螢幕表格檢視 | 無水平溢出版 |

### 2.3 UI/UX 測試

| 測試項目 | 測試方法 | 通過標準 |
|---------|---------|---------|
| 視覺一致性 | 檢查色彩/字體/圓角/陰影 | 符合設計系統規範 |
| 色彩對比度 | 抽檢文字與背景對比度 | WCAG AA (4.5:1) |
| 互動回饋 | hover / active 效果存在 | 視覺變化明顯 |
| 空狀態處理 | 模擬無資料情境 | 顯示友善提示 |

### 2.4 無障礙測試 (Accessibility)

| 測試項目 | 測試方法 | 通過標準 |
|---------|---------|---------|
| ARIA 標籤 | 檢查 role / aria-label | 主要元素有標籤 |
| 鍵盤導航 | Tab 鍵導航測試 | 可聚焦元素有 focus 樣式 |
| 表單標籤 | 檢查 label 與 input 關聯 | 所有 input 有關聯 label |
| 圖片 alt | 檢查 img 標籤 alt 屬性 | 有含義的 alt 文字 |

## 3. 測試重點 (基於審查發現)

### 🔴 高風險項目

| # | 問題 | 受影響檔案 | 測試重點 |
|---|------|-----------|---------|
| 1 | 缺少刪除/停用確認對話框 | `mockup_user_permission.html`, `mockup_mvp_machine_list.html` | 檢查是否有確認 Modal |
| 2 | 表單驗證錯誤提示缺失 | 所有表單頁面 | 檢查錯誤訊息 UI |
| 3 | 色彩對比度不足 | `mockup_game_*.html` 等 | 抽檢 text-gray-400 對比度 |
| 4 | 行動版優化缺失 | `mockup/central/*` | 檢查 sidebar 響應式 |
| 5 | 表格小螢幕溢版 | 所有列表頁 | 檢查滾動或卡片化 |

### 🟡 中風險項目

| # | 問題 | 受影響檔案 | 測試重點 |
|---|------|-----------|---------|
| 1 | 功能重複頁面 | `mockup_lobby_config/settings.html`, `mockup_flg_monitor/system_monitor.html` | 確認功能區分 |
| 2 | 命名規範 | `mockup_mvp_*.html`, `mockup_std_*.html` | 檢查命名一致性 |
| 3 | 命名空間不統一 | 早期 mockup | 檢查顏色 class 命名 |

## 4. 測試環境

- **瀏覽器：** Chrome 120+, Firefox 121+, Safari 17+
- **裝置：** Desktop (1440px), Tablet (768px), Mobile (375px)
- **工具：** Playwright / Chrome DevTools

---

## 5. 測試排程

| 階段 | 內容 | 預估時間 |
|------|------|---------|
| Stage 1 | 頁面結構 + 功能映射自動化掃描 | 30 分鐘 |
| Stage 2 | UI/UX 手動抽檢 (高風險項目) | 1 小時 |
| Stage 3 | 響應式 / 無障礙抽檢 | 1 小時 |
| Stage 4 | 問題彙總與回報 | 30 分鐘 |
