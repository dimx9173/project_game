## Slide 3: 🚀 專案背景：為何要有 V8 旗艦版？

* **現狀與挑戰：**
  * 規模擴張：分散部署與管理難度激增。
  * 數據孤島問題：現狀核帳困難，資訊不即時。
  * 維護成本極高：依賴人工現場處理，效率低下。
* **核心目標：**
  * **中央集權化管理**
  * **數據導向營運**
  * **全自動維護流程**

## Slide 4: 🎯 客戶核心需求與預期結果

* **客戶要什麼？**

---

## Slide 5: 🏗️ 核心架構選型：企業級 .NET 8

* **Backend (後端)：** ASP.NET Core 8 Web API
  * 高效能、跨平台、極致安全。
* **Frontend (前端)：** Blazor Server + Radzen
  * 實時通訊、一致的高級感 UI。
* **DB Strategy (資料庫)：**
  * 中央：PostgreSQL (大規模數據)。
  * 在地：SQLite (輕量、邊緣計算)。

---

## Slide 6: 🛡️ 三大「護城河」核心模組

1. **OTA 無感更新 (OTA Silent Update)**
   * 全球派發、MD5 校驗、自動回滾。
2. **爭議帳務處理 (Dispute Resolution)**
   * 非同步事務隊列、自動預警流程。
3. **動態 RBAC (Dynamic RBAC)**
   * 中央授權、細粒度審核日志、PIN 碼管控。

---

## Slide 7: 👥 團隊分工與職責

* **後端：** API 實作、核心同步服務 (`SyncService`)、加密安全。
* **前端：** 統一 UI 控制器、實時監控儀表板、OTA 看板。
* **QA：** 破壞性斷線測試、報表多維度精準核對。
